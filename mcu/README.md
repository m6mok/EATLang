# mcu/ — кросс-компиляция под микроконтроллер (трек 2)

Демонстрация: эмулятор MOS 6502 (`examples/mos6502/`), собранный под
ARM Cortex-M3, выполняется на плате QEMU mps2-an385 и печатает в UART
байт-в-байт то же, что интерпретатор на хосте (`make verify_arm`).

## Как это устроено

Канонический `.ll` компилятора платформонезависим — вся зависимость
от платформы сосредоточена в шиме шести аксиом ОС. На хосте это
`src/eatc/runtime.c` (libc), на МК — `mcu/runtime_mcu.c`:

| Аксиома | Хост (libc) | МК (mps2-an385) |
| --- | --- | --- |
| `write_byte` / `write_span` | stdout | CMSDK UART0 (0x40004000) |
| `write_err_byte` | stderr | тот же UART0: канал один |
| `read_byte` | stdin, EOF | прошитый массив `eat_input` |
| `exit` | `exit(code)` | полухостинг SYS_EXIT_EXTENDED |
| `trap` / `trap_code` | stderr + exit(1) | UART + полухостинг exit(1) |

Ключевые решения:

- **вход зашивается в образ** (`mcu/embed_input.py` → `eat_input[]`):
  у UART нет понятия «конец потока», а данные прошивки — честная
  модель входа для МК; программа без входа получает `Err(Eof)` сразу;
- **режим trap-кодов** (`--trap-codes`): в прошивке вместо текстов —
  числа, таблица `; trap <код>: <сообщение>` остаётся на хосте
  (`build/arm/*.trapmap`);
- **сборка через clang по .ll**: `eatc build` пишет `.ll` (verified:
  доказанные проверки сняты), `clang --target=thumbv7m-none-eabi`
  перекладывает его в объектный код, `ld.lld` линкует по
  `mcu/mps2_an385.ld`; llvmlite не трогаем;
- в `runtime_mcu.c` же — стартап (таблица векторов, перенос `.data`,
  обнуление `.bss`) и EABI-хелперы памяти (`__aeabi_memcpy` и т.п.),
  которые LLVM зовёт на freestanding-цели;
- таблицы раскрутки ARM EHABI (`.ARM.exidx`) отброшены в
  linker-скрипте: исключений нет, trap завершает программу.

## Команды

```sh
brew install lld qemu    # ld.lld (линковка ELF) и qemu-system-arm
make arm_mos6502         # прошивка build/arm/Mos6502.elf + размеры
make run_arm_mos6502     # запуск в QEMU (mps2-an385, Cortex-M3)
make verify_arm          # вывод QEMU == интерпретатор, байт-в-байт
```

## Числа (2026-07-13, mul13x11.rom)

- Флеш: **10.7 КБ** (`.text` 10560 + `.rodata` 152 + вектора 8 Б;
  сравнить: хост-бинарник Mos6502 — 69.7 КБ Mach-O). Строковых
  глобалов почти нет — режим trap-кодов.
- RAM: `.bss` 4 Б (курсор входа), `.data` 0; всё остальное — стек.
- Стек: отчёт §8 даёт худшую цепочку **≤ 262 594 Б** (кадр `main`
  131 466 Б — `Cpu` с 64К памяти эмулятора живёт в кадре, плюс копия
  из `cpu_new`). Стек растёт от вершины 4 МБ RAM платы — запас
  ×15; для реального МК отчёт §8 — готовая нижняя граница выбора
  чипа (нужно ≥ 320 КБ SRAM, например Cortex-M7/H7-класс).

## Ограничения демонстрации

- Плата фиксирована (mps2-an385), UART без прерываний, опрос TXFULL.
- Ввод только прошитый: интерактивного UART-чтения нет — для него
  нужен протокол конца потока (решение по SPEC, если понадобится).
- На реальном железе `bkpt 0xAB` (полухостинг) без отладчика уходит
  в вечный цикл `for(;;)` — замените на сброс/сон по месту.
