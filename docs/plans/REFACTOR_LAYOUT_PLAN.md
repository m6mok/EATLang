## 1. Итог одной строкой

Разложить 28 файлов `selfhost/` по 5 папкам-фазам (`lex/ parse/ check/
ir/ verify/`, `Rt.eat` в корне), свернуть ~800 сырых обращений к пулу
AST `self.p.nX[i / 65536][i % 65536]` в методы-аксессоры (образец уже
живёт в `Verify.eat`) и убрать три локальных дубля в `lib/` (`Http.hexv`,
разбор цифр в `Json`/`Http`) переиспользованием `lib/Hex`, `lib/Ascii`.

## 2. Почему язык к этому приспособлен

Ничего в семантике не меняется — это гигиена исходника, продолжение
линии REFACTOR_SELFHOST (потолок строк, конструкторы фаз, `extend`).
Папки материализуют группировку по фазам, которую уже задают префиксы
имён (`Check*`, `Ir*`, `Verify*`). Аксессоры пула — тот же приём, что
`Verify.eat` применил к своим обращениям: доменный метод вместо ручной
двухуровневой индексации страничного массива `[[u32; 65536]; 2]`.
Дедуп `lib/` — прямое следствие модульной библиотеки: то, что уже есть
в `Hex`/`Ascii`, не должно жить копией в `Json`/`Http`.

## 3. Диагноз (2026-07-17)

| Симптом | Причина | Лекарство |

## 6. Архитектура / механизм

### 6.1. Раскладка (этап 1)

`Rt.eat` остаётся в `selfhost/` (нулевой модуль рантайма; на него
жёстко смотрит `src/eatc/driver.py:27` — не двигаем, продакшн-драйвер
не трогаем). Остальное — по фазам, каждый `*Main.eat` в папке своей
фазы:

```text
selfhost/
  Rt.eat
  lex/     Tok  Lexer  LexMain
  parse/   Ast  Parser  ParserExpr  ParseMain
  check/   Check  CheckConst  CheckBody  CheckDump  CheckFold
           SigMain  TypedMain
  ir/      Ir  IrEmit  IrExpr  IrStmt  IrMain  IrCodesMain  IrOptMain
  verify/  Verify  VerifyExpr  VerifyRel  VerifyFlow  VerifyDump
           VerifyMain
```

`SigMain`/`TypedMain` — точки входа фазы Check (эмитят sig/typed из
Check), поэтому в `check/`. Перенос — `git mv` (сохраняет историю).

**Обязательные правки (иначе сборка не находит файлы):**
1. `Makefile` — переменные `RT`, `SELFHOST_LEXER|PARSER|SIG|TYPED|IR|
   IR_CODES|IR_OPT|VERIFY` (строки 13, 207–247). Правила гоняют через
   переменные — их не трогать.
2. `tests/gate/verify_all_file.sh:8` — жёстко зашитый фильтр
   `selfhost/Verify.eat|selfhost/VerifyMain.eat` → новые пути
   `selfhost/verify/Verify.eat|selfhost/verify/VerifyMain.eat`
   (**критично**: иначе верификатор попробует верифицировать сам себя).
3. `tests/bench/bench.py` — списки файлов (строки 36, 460–464,
   547–561).
4. `tools/gen_verifymain.py` — пути `Verify.eat`/`Check.eat`/
   `VerifyMain.eat` (строки 24–27).

Устойчиво к переносу (рекурсивные `find`/`rglob` — править НЕ надо):
`Makefile` `find … selfhost …` (313, 361, 390), `tests/check_style.py`
`rglob` + `parts[0] == "selfhost"` (потолок 2500 строк продолжит
действовать на подпапки), `tests/gate/selfhost_file.sh`/`opt_file.sh`
(фильтров по путям нет). Импорт-блоков `from "selfhost/..."` нигде нет.

**Доккарты (сборку не ломают, обновить синхронно):** карта файлов
`docs/plans/SELFHOST.md` (таблица + текст), `AGENTS.md:79`, `README.md`,
`docs/MODIFYING.md`, `docs/PROCESSES.md`, `docs/POWER_OF_10.md`,
`docs/GUIDE.md`, `tests/bench/README.md`; строчные ссылки вида

не load-bearing; комментарии-ссылки в `src/eatc/*.py` — пояснительные.

### 6.2. Аксессоры пула (этап 2)

Образец — `selfhost/Verify.eat:561-622`:

```eat
func nk(var self, n: u32) -> u32
    requires n < 131072
    ensures true
{ return self.c.p.nk[n / 65536][n % 65536] }
```

- На `Check` (держит парсер полем `p`): методы читают `self.p.nX[…]`;
  имя метода `nX` не конфликтует (пул — на поле `p`, не на `Check`).
  Вызовы `self.p.nX[i / 65536][i % 65536]` → `self.nX(i)`.
- На `Ir` (держит Check полем `c`, парсер — `self.c.p`): методы читают
  `self.c.p.nX[…]` — идентично `Verify`. Вызовы `self.c.p.nX[…]` →
  `self.nX(i)`.
- Набор полей пула: `nk nl nc nss nsl nx ny nxh nfc nlc nns` (объявление
  `Parser.eat:33-44`) — вводить аксессоры по факту обращений каждой
  группы (в Ir встречаются не все).
- Методы кладутся в `extend Check { … }` / `extend Ir { … }` рядом с
  инфраструктурой фазы (или в базовый файл фазы).

**Перф-ворота (обязательны).** Check и Ir — горячий путь компилятора.
Выигрыш безопасен только при инлайнинге аксессоров; иначе — регресс.
`make bench` до/после, критерий — без регрессии сверх шума. Если
аксессоры не промотируются — либо инлайн-хинты (кандидат 2,
[OPTIMIZATIONS_PLAN](OPTIMIZATIONS_PLAN.md)), либо этап откатывается.
Вывод компилятора (IR/дампы тестовых программ) не меняется — рефактор
внутренний; но весь гейт готовности перегоняется (исходники selfhost —
вход фикспойнта и самоприменения).

### 6.3. Дедуп lib (этапы 3–4)

- **Этап 3.** `lib/Http.eat`: `import { hex_val } from "lib/Hex.eat"`
  (Http уже импортирует из `Fmt` — добавить связку), удалить метод
  `hexv` (`:634-648`, не экспортируется — `lib.sig` не меняется),
  заменить `self.hexv(b)` (`:692`) на `hex_val(b)`.
- **Этап 4.** `lib/Json.eat`, `lib/Http.eat`: `import { is_digit,
  digit_value } from "lib/Ascii.eat"`, заменить сырые `b >= 48 and
  b <= 57` → `is_digit(b)` и `b - 48` → `digit_value(b)` (Json
  `:211,397,420,433`, Http `:404`). Сверить семантику границ
  `digit_value` с локальной. Горячий парс-путь — `make bench` (http,
  json) без регрессии; при регрессе оставить инлайн в горячих точках.

## 7. Что реализуем НЕ

- **Папки в `lib/`** — файлы `lib/` подключаются `import from "lib/…"`;
  их канонические пути вшиваются в `#module` и в снапшот
  `tests/sig/lib.sig`, перенос сломал бы паритет и все примеры. Вне
  рамок.
- **Папки в `src/eatc/`** (Python) — решение пользователя: не зеркалим
  раскладку, `src/eatc/` остаётся плоским (правка Python-импортов — не
  оправданный churn ради косметики).
- **Свёртка visitor-машин выражений** (Check/Ir/Verify) — дублирование
  структурно необходимо: тела расходятся (проверка типов / сериализация
  IR / накопление фактов), общий обход не вынести без first-class
  функций и изменил бы вывод IR. Разведка подтвердила — не трогаем.
- **Централизация таблиц «код→имя»** (Tok/Ast) — уже централизованы,
  вывод дампов байт-в-байт; массив вместо `elif` требует кучи.
- **Вынос строк ошибок в константы** — повторы локальны (в основном
  `CheckBody`), выигрыш мал, а тексты зеркалятся в Python байт-в-байт.
- **`Rt.eat` в подпапку** — нулевой модуль, жёсткая ссылка драйвера.

## 9. Паритет

- Этапы 1–2 меняют только `selfhost/` (+ сборку/доки): зеркалить
  нечего — эталон Python и self-hosted компилируют одни исходники, а
  весь гейт готовности (включая `verify_bootstrap`,
  `verify_selfhost_verify_all`) перегоняется целиком.
- Этапы 3–4 меняют только `lib/` — в Python не зеркалится;
  `tests/sig/lib.sig` не должен измениться (изменения внутренние,
  экспорт не трогаем) — проверяется `make verify_sig`.

## 11. Инварианты

- Power of 10 не трогаются: перенос файлов и аксессоры —
  синтаксическая гигиена, семантика тождественна; ни кучи, ни рекурсии,
  ни новых обходов. `requires n < 131072` на аксессорах — та же
  доказуемая граница, что у ручной индексации.
- Все дампы/IR: Python-эталон == self-hosted байт-в-байт на каждом
  этапе; частично зелёное не коммитится.
- Производительность selfhost и lib не деградирует: `make bench`
  (+`bench_crosslang` для lib) сверяется на этапах 2–4.
- Раскладка полей каждой структуры остаётся в одном блоке `struct`
  (аксессоры — методы, не поля).
