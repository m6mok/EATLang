## 1. Итог одной строкой

Fixed-point ложится **одним `lib/`-модулем без единой правки
компилятора**. Паритет `бинарник == интерпретатор == self-hosted`
получается даром: модуль написан на EATLang, оба компилятора уже умеют
все его конструкции, а `verify_selfhost` сверяет дампы автоматически
(§8). Из-за отсутствия дженериков каждый формат — отдельный тип с
рукописным дублированием API, поэтому «максимум вариантов» реализуем
**поэтапно**: сперва два взаимодополняющих формата (бинарный и
десятичный), остальные — по факту спроса.

## 2. Что такое fixed-point (и чем это НЕ float)

Число хранится как обычное целое `raw`, а «реальное» значение =
`raw / SCALE`, где `SCALE` фиксирован (точка не «плавает», как у float).
Пример Q16.16 (`SCALE = 65536`): `raw = 98304` → `1.5`. Вся арифметика —
целочисленная:

- сложение/вычитание — над `raw` напрямую (`a.v + b.v`);
- умножение — `raw_a * raw_b / SCALE` (промежуток шире, чтобы не
  переполниться, см. §7);
- печать — целая часть `raw / SCALE`, дробная `raw % SCALE`, разложенная
  в десятичные цифры.

Float'ов в языке нет ([Struct.eat](../../examples/struct/Struct.eat):
«Float'ов в v1 нет»); fixed-point даёт дроби, оставаясь в целочисленном,
тотальном, trap-на-переполнении мире — ровно то, чем считают дроби на МК
без FPU (трек 2).

## 3. Почему правок языка не нужно (в отличие от async)

Всё выражается существующими средствами:

- `struct Fix { v: i32 }` — значение на стеке, by-value, без кучи;
- методы с `self` и контрактами — как `Point` в
  [Struct.eat](../../examples/struct/Struct.eat);
- касты `i64(x)`/`i32(x)` — для широкого промежутка умножения;
- знаковое `/` и `%`, деление на ноль и переполнение → **trap** с
  координатами (детерминизм, а не NaN/inf).

Следствие: **решения пользователя по языку не требуется** (инвариант 2
AGENTS.md не затрагивается). Решения нужны только по составу и именам
модуля (§11).

## 4. Пространство форматов

Из-за trap-на-переполнении осуществимость `mul` зависит от ширины
промежутка. Таблица кандидатов:

| Формат | База | SCALE | Диапазон | Шаг | mul | Вердикт |
| --- | --- | --- | --- | --- | --- | --- |
| **Q16.16** | `i32` | 65536 | ±32768 | ~1.5e-5 | `i64(a)*i64(b)/SCALE`, точно, промежуток ≤4.6e18 < i64max | **v1 (бинарный)** |
| **Deci6** | `i64` | 1_000_000 | ±9.2e12 | 1e-6 | `a*b/SCALE` в i64; **trap при больших** (произведение raw > i64max) | **v1 (десятичный)** |
| Q24.8 | `i32` | 256 | ±8.4e6 | ~4e-3 | `i64`-промежуток, точно | будущее (широкий бинарный) |
| Q8.8 | `i32` | 256 | ±8.4e6 | ~4e-3 | как Q24.8 | низкая ценность |
| Q32.32 | `i64` | 2^32 | ±2e9 | ~2.3e-10 | через `I128` из [lib/U128.eat](../../lib/U128.eat): произведение raw не переполняется, trap только на выходе из 128 бит | **✅ реализован 2026-07-17** (тип `Q32`, активация U128_PLAN §9.6) |

- **v1 — два формата**: `Q16.16` (компактный бинарный, степень двойки,
  идеален для МК) и `Deci6` (десятичный, тривиальная и точная печать
  «денег»/измерений). Они закрывают «разные потребности»: бинарный vs
  десятичный, компактный (одно слово) vs широкий (диапазон).
- **Q32.32 отклонён честно**: без 128-битного промежутка (в языке нет)
  умножение переполняется уже на `1.0 × 1.0`. Не заявлять как вариант.
- Остальные форматы — тот же скелет API с другим `SCALE`; добавляются по
  факту второго клиента (принцип из
  [MODULES_PLAN §12](../MODULES_PLAN.md): «не на упреждение»).

## 5. Представление и инварианты

```text
struct Fix  { v: i32 }    # Q16.16:  real = v / 65536
struct Deci { v: i64 }    # Deci6:   real = v / 1_000_000
const FIX_SCALE:  i32 = 65536
const DECI_SCALE: i64 = 1000000
```

- поля by-value, никаких указателей → «алиасинг невозможен по
  построению»;
- `SCALE` вынесены `const` (годятся в контракты; при желании — отдельный
  `lib/Fixed`-const или инлайн-литералы, §11);
- каждый метод возвращает **новое** значение (`self` неизменяем).

## 6. API (полный, две группы имён — по типу)

Предлагается **методы** для операций (цепочки `a.mul(b).add(c)`, имена
не конфликтуют между типами) + **свободные конструкторы** (у них нет
получателя). Альтернатива — только свободные функции с префиксами
(`fix_add`/`deci_add`) из-за глобальной уникальности имён; методы
читаются лучше (решение — §11).

Ниже — для `Fix` (для `Deci` — зеркально, с `i64`):

**Конструкторы (свободные функции)**
- `fix(n: i32) -> Fix` — из целого (`n * SCALE`)
- `fix_ratio(num: i32, den: i32) -> Fix` — из дроби (`num*SCALE/den`)

**Арифметика (методы)**
- `add(self, o) -> Fix`, `sub(self, o) -> Fix`, `neg(self) -> Fix`
- `mul(self, o) -> Fix` — через `i64`-промежуток (§7)
- `div(self, o) -> Fix` — `requires` знаменатель ≠ 0

**Сравнения и утилиты (методы)**
- `eq/lt/le(self, o) -> bool`, `abs(self) -> Fix`
- `min/max(self, o) -> Fix`, `clamp(self, lo, hi) -> Fix`
  (контракты в духе [Num.eat](../../lib/Num.eat))

**Округление (методы) → возвращают целое или Fix**
- `to_int(self) -> i32` — усечение к нулю
- `round(self) -> i32` — к ближайшему (полушаг прибавляется по знаку)
- `floor(self) -> Fix`, `ceil(self) -> Fix`

**Форматирование**
- `repr(self, digits: u32) -> str<32>` — «-3.14159» с `digits` дробными
  знаками; целая часть — цифровым циклом (или переиспользовать
  [Fmt.fmt_i32](../../lib/Fmt.eat), первый lib→lib импорт), дробная —
  `frac * 10^digits / SCALE`, ведущие нули добиваются.

Итого ~16 функций/методов на тип × 2 типа + тесты.

## 7. Технические тонкости (пин перед реализацией)

1. **Умножение — через широкий промежуток.** Q16.16:
   `i32( i64(a.v) * i64(b.v) / 65536 )`. Промежуток `i32×i32 ≤ 4.6e18`
   в `i64` не переполняется; финальный `i32(...)` **trap'ит**, если
   реальный результат вне ±32768 — это корректно (детерминизм).
2. **Деление — тоже широкий промежуток.** `i32( i64(a.v) * 65536 / b.v )`
   (иначе `a.v * SCALE` переполнит `i32`). `requires b.v != 0` — тогда
   верификатор снимает проверку деления на ноль.
3. **Знаковое `/` и `%` — ЗАПИНЕНО 2026-07-17 на эталоне**
   (`eatc run` и бинарник дают одно и то же): деление **усекает к
   нулю**, знак остатка — **по делимому** (C-семантика, LLVM
   `sdiv`/`srem`):
   `(-7)/2 = -3`, `(-7)%2 = -1`; `7/(-2) = -3`, `7%(-2) = 1`;
   `(-7)/(-2) = 3`, `(-7)%(-2) = -1`.
   Формулы округления (S = SCALE, все деления — языковые):
   - `to_int`: `v / S` — усечение к нулю даром;
   - `floor`: `q = v / S; if v % S < 0 { q = q - 1 }` — к −∞;
   - `ceil`: `q = v / S; if v % S > 0 { q = q + 1 }` — к +∞;
   - `round` (к ближайшему, половина — от нуля):
     `if v >= 0 { (v + S/2) / S } else { (v - S/2) / S }`
     (`v ± S/2` у края диапазона переполняется → trap, штатно).
   Тесты покрыть обеими сторонами нуля.
4. **Deci6 — ограниченный диапазон mul/div.** `a.v*b.v` в `i64`
   переполняется при значениях с произведением raw > 9.2e18; это trap.
   Задокументировать в шапке модуля как штатную границу формата (как
   «u64 в горячих циклах M0», GUIDE §3).
5. **repr и ёмкость str.** `str<32>` хватает: знак + до 13 цифр целой +
   точка + до 6 дробных. Сборка через интерполяцию в цикле — O(n²) при
   n ≤ 32, приемлемо (та же оговорка, что у `read_line`,
   [MODULES_PLAN §11](../MODULES_PLAN.md)).

## 8. Встраивание в гейт (`make check verify ...`)

- **Паритет — автоматом.** `verify_selfhost` перебирает `find examples
  lib selfhost tests -name '*.eat'`; новый `lib/Fixed.eat` попадает в
  сверку `lex/parse/sig/typed/ir` эталон↔self-hosted без правок Makefile.
  Поскольку модуль использует только уже поддержанные конструкции, дампы
  совпадут байт-в-байт.
- **verify_sig.** Добавить в [SigProbe.eat](../../tests/sig/SigProbe.eat)
  импорт одного имени из `Fixed` (напр. `fix`), затем `make regen_sig` и
  закоммитить diff `tests/sig/lib.sig` (осознанное изменение интерфейса).
- **Тесты модуля.** `eatc check` требует `main`, поэтому `test`-блоки
  `Fixed` исполняются только через клиента в стриме. Варианты: (а)
  добавить использование `Fix`/`Deci` в
  [examples/modules/Main.eat](../../examples/modules/Main.eat) (он
  собирается в `make verify`, что запускает `check` → тесты всех
  импортированных модулей); либо (б) отдельный маленький пример-витрина.
  Решение — §11.
- **Полный гейт** (`make check verify verify_suite verify_selfhost
  verify_bootstrap verify_trapcodes verify_sig`) должен быть зелёным
  целиком (AGENTS.md инвариант 3).

## 9. Power of 10 — как соблюдается

- нет кучи: значения — `struct` фиксированного размера на стеке;
- нет рекурсии: методы не зовут себя, граф — DAG;
- циклы с границей: разложение цифр — `for _ in 0..digits`/`0..20`
  с константной верхней границей (как [Fmt.eat](../../lib/Fmt.eat));
- результат не отбрасывается: методы чистые, значения используются;
- контракты: у методов с параметрами — `requires`, с возвратом —
  `ensures` (минимум `requires true`/`ensures true` явно).

Ни одного обхода правил не вводится.
