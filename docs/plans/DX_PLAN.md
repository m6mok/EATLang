## 1. Итог одной строкой

Писать на EATLang неудобно не из-за Power of 10, а из-за **недостающего
библиотечного слоя** (сетевая механика HTTP переписывается в каждом
примере заново) и **отсутствия точечного сахара** над уже существующими
`match`/контрактами; лечится в два независимых трека — **L1: библиотека
без правок языка** (главный выигрыш для HTTP) и **L2: сахар над языком**
(`?`, дефолтные контракты), где каждый пункт — отдельное решение
пользователя с зеркалом в selfhost.

## 2. Почему язык к этому приспособлен (и где конфликт)

- **L1 не трогает ни одного правила.** Всё, что переписывают примеры
  (событийный цикл, запись курсором, чтение тела, разбор порта,
  ответы-хелперы, сборка JSON) — уже написанные тотальные функции с
  bounded-циклами; их надо **вынести в `lib/`**, а не изобрести. Это
  чистое переиспользование, риск Power of 10 нулевой.
- **L2 — чистый поверхностный сахар.** `?` разворачивается в
  существующий исчерпывающий `match` ([GENERICS_PLAN §7.1](GENERICS_PLAN.md));
  дефолтный контракт `requires`/`ensures` — синтез узла `true`, который
  верификатор и так обрабатывает. IR по построению идентичен → паритет
  почти бесплатен, тотальность и bounded-модель не затронуты.
- **Честный конфликт — только один.** Дефолтные контракты снимают
  конвенцию «отказ от контракта виден на ревью»
  ([examples/functions/Functions.eat:2-6](../../examples/functions/Functions.eat#L2)).
  Это не нарушение правила 5 (контракт `true` — слабейший, ничего не
  доказывает и раньше), а отказ от **стилевой** нормы видимости. Решение
  пользователя (§10, п. L2-b).
- **Жёсткая рамка HTTP-фреймворка (не конфликт, а данность):**
  единственный `loop` обязан жить в `main` (SPEC §209, [HTTP_PLAN §6.2](HTTP_PLAN.md)),
  указателей на функции нет (правило 9). Значит `lib/Server.eat` владеет
  **всей механикой соединения**, но сам цикл и диспетч-`match` по
  маршрутам остаются в коде пользователя. Роутер-как-таблица-колбэков
  (`router.get("/x", handler)`) невозможен и не проектируется.

## 3. Каталог болей (диагноз по всем примерам)

Собран прямым чтением 6 HTTP-примеров, витрины `all`, `struct`, `json`,
`mos6502`, обеих библиотек и снапшота `lib.sig`.

### 3.1 HTTP: дублирование сетевой механики

`lib/Http.eat` не содержит **ни одного** обращения к сокету — строит
байты в `Resp.buf` и останавливается. Каждый пример переписывает:

| Что дублируется | Где (путь:строка) | Объём |
| --- | --- | --- |
| Событийный цикл `main()` (accept→дренаж→диспетч→close, `SERVE_CONNS`) | [Hello.eat:105](../../examples/http/Hello.eat#L105), [Router.eat:235](../../examples/http/Router.eat#L235), [Todo.eat:759](../../examples/http/todo/Todo.eat#L759), Api/Echo/Pool | ~60 строк ×6, дословно |
| `send_resp` (запись курсором) | [Hello.eat:89](../../examples/http/Hello.eat#L89), [Router.eat:219](../../examples/http/Router.eat#L219), [Todo.eat:708](../../examples/http/todo/Todo.eat#L708), Api:51 | ×5 |
| `send_bytes` (длинное тело) | [Todo.eat:725](../../examples/http/todo/Todo.eat#L725), Api:68 | ×2 |
| `slurp`+`body_kind`+`body_len` | [Todo.eat:545](../../examples/http/todo/Todo.eat#L545), Api:247 | ×2, попарно идентичны |
| `port_arg` | все примеры | ×6 |
| Ответы-хелперы (`not_found`/`bad_request`/…) | [Todo.eat:651](../../examples/http/todo/Todo.eat#L651), Router, Api | заново в каждом |
| Самописный `struct Out` (сериализатор JSON) | [Todo.eat:58-172](../../examples/http/todo/Todo.eat#L58) | 115 строк, дублируют `JOut`+`Resp`, экранирование беднее (нет `\n\t\r`) |
| `for i in 0..65536 { d.put(body.at(i)) }` (Body→JDoc) | Todo ×2, Api ×1 | ×3, нет метода-моста |

Следствие: **«Hello world» по HTTP — 169 строк**, из них ~120 — механика.

### 3.2 JSON: `JOut` сериализует только готовое дерево

`JOut` умеет `ser(d, root)` — обход разобранного `JDoc`. Инкрементального
билдера **из прикладных данных** нет: публично только `byte/word/num/eb/qstr`,
причём `qstr` читает из `d.arena`, а не из `str<N>`. Нет
`begin_array/end_array/field/qval(str<N>)`. Поэтому `Todo.eat` завёл
свой `Out`. Моста `Body → JDoc` нет ни со стороны `Body`, ни `JDoc`
(`JDoc.load` берёт лишь `str<256>`, тело до 65536 не подать).

### 3.3 Общеязыковой шум (вне HTTP)

- **`requires true / ensures true`** — 2 строки-заглушки почти на каждой
  функции ([Functions.eat:2-6](../../examples/functions/Functions.eat#L2)
  фиксирует это как осознанную конвенцию). Пользователь считает
  постоянное дублирование нелогичным.
- **Option/Result → пирамиды `match`**: одно значение — 3-4 строки,
  вложенные — 6-8 ([json/Main.eat:50-64](../../examples/json/Main.eat#L50),
  [Todo.eat:451-464](../../examples/http/todo/Todo.eat#L451),
  [parse_todo_body:511-538](../../examples/http/todo/Todo.eat#L511)). Нет
  `?`, `if let`, `unwrap_or`.
- **Строки `str<N>`**: только интерполяция `"{a}{b}"`, аппенда нет.
  Копирование строки — `title = "{title}{char(u8(ts[i]))}"` в цикле
  ([Todo.eat:520](../../examples/http/todo/Todo.eat#L520)) — O(n²) и
  уродливо. Сравнение — хелпер+цикл.
- **Циклы**: `while` нет → `for _ in 0..ПОТОЛОК { if усл { break } }` +
  внешний флаг (из `break` не вернуть значение). Магические потолки
  (`65537`, `63488`, `8900`).
- **Sentinels**: `4294967295` (NONE/SENT/NO_CONN/NO_IDX/MISS) размазан.

## 4. Числа (пределы)

Трек L1 не вводит новых потолков — переиспользует существующие
([HTTP_PLAN](HTTP_PLAN.md)): `HTTP_REQ_CAP=8192`, `HTTP_RESP_CAP=16384`,
`HTTP_MAX_HEADERS=64`, `HTTP_MAX_BODY=65536`, буфер вывода `JOut/Out` —
`[u8; 65536]`. Бюджеты дренажа (`0..8900` витка запроса, `0..70000` тела)
переезжают в `lib/Server.eat` как именованные константы. `SERVE_CONNS`
(детерминированный останов для сверки транскриптом) остаётся у
пользователя — это часть его цикла, не библиотеки.

Трек L2 потолков не вводит вовсе (сахар над синтаксисом).

## 5. Аксиомы ОС

Новых нет. `lib/Server.eat` вызывает уже существующие сокет-аксиомы
([HTTP_PLAN §3](HTTP_PLAN.md)): `socket_listen/accept/avail/read_byte/
write_span/close`.

## 6. Архитектура / механизм

### 6.1 L1a — `lib/Server.eat`: драйвер соединения

`struct Server` владеет состоянием, которое сейчас руками разложено по
`main`: `lst/live/conn/served/req/st/eof`. Пользователь получает цикл в
~12 строк вместо ~60:

```
match server_listen(port_arg()) {   # port_arg тоже уезжает в lib
    Ok(s0) {
        let s: Server = s0
        loop {
            const k: u32 = s.tick()          # accept-or-drain, один виток
            if k == PARSE_DONE {
                s.reply(handle(s.req))      # диспетч — у пользователя
            } elif k != PARSE_MORE {
                s.reply(err_resp(k))        # k несёт 400/413/431
            }
            if s.served >= SERVE_CONNS { break }
        }
    }
    Err(_) { print("не открыть порт") }
}
```

- `func server_listen(port: u16) -> Result<Server, IoError>` — bind+listen.
- `.tick(let self) -> u32` — один виток: если нет соединения — `accept`;
  иначе дренаж готовых байт сквозь `req.push_byte`. Возвращает
  `PARSE_MORE` (ещё не готово / idle), `PARSE_DONE` (запрос собран,
  читай `s.req`), либо код ошибки парсера (`400/413/431`). Обрыв пира —
  внутренний `close`+advance, наружу `PARSE_MORE`.
- `.reply(let self, w: Resp)` — `send_resp` + `close` + `served++` +
  `conn=SENT`. Курсор записи внутри.
- Хелперы-ответы (`not_found/bad_request/too_large/store_full/
  method_not_allowed/err_resp`) и `port_arg` — сюда же.

**Узел дизайна (§10 L1-a) — ✅ РЕШЁН вариантом (ii).** Как отдать тело >
`Resp.buf` без дженериков над размером буфера. Тело живёт в
`JOut/Out.buf: [u8;65536]`; `socket_write_span` требует конкретный
буфер. Варианты были: (i) `Server` зависит от `lib/Json.eat` и стримит
`JOut` напрямую (lib→lib, прецедент Fixed→Fmt) — но не всякий сервер
JSON-ный; (ii) канонический `Bytes`-вид `[u8;65536]+n+over` в самой
`lib/Server.eat`, в который пишут и JSON, и текст. **Выбран (ii):** один
буфер-носитель, независимый от Json; `.reply_bytes(head, b)` стримит
заголовки+тело. L1b/L1c кладут в `Bytes` результат `JOut`.

### 6.2 L1b — `JOut` как инкрементальный билдер

Добавить в `lib/Json.eat` методы построения **из прикладных данных**
(не из `JDoc`): `begin_array/end_array/begin_obj/end_obj`,
`field(name: str<64>)`, `qval(s: str<N>)` (строка в кавычках с полным
экранированием `JOut.eb`, включая `\n\t\r\b\f`), `bval(b: bool)`.
Самописный `Out` из `Todo.eat` (115 строк) исчезает; экранирование
становится корректным.

### 6.3 L1c — мост `Body → JDoc`

Метод `d.load_body(b: Body)` (или `b.feed(d)`): bounded-цикл
`0..HTTP_MAX_BODY`, `d.put(b.at(i))` — ровно тот, что скопирован трижды.
Границы совпадают (`Body.buf`, `JDoc.src`, `HTTP_MAX_BODY` — все 65536),
семантических препятствий нет.

### 6.4 L2a — оператор `?` (сахар над `match`)

`expr?` в функции, возвращающей `Result<_, E>` / `Option<_>`:
`Ok(v)`/`Some(v)` → `v`; `Err(e)` → `return Err(e)`; `None` → `return
None`. Разворот — в существующий исчерпывающий `match`
([GENERICS_PLAN §7.1](GENERICS_PLAN.md)), IR идентичен. Схлопывает
`parse_todo_body`, JSON-извлечение в `json/Main.eat`, `slurp`.

### 6.5 L2b — дефолтные контракты (сахар: пер-клауза омиссия ≡ `true`)

**Дизайн (Флавор A, выбран).** Пропущенный `requires` ≡ `requires true`,
пропущенный `ensures` ≡ `ensures true`. Пишется только содержательная
клауза; обе тривиальны — не пишется ничего. Снимает **1873** строки-
заглушки по examples/lib/selfhost (`ensures true` 1210 + `requires true`
663; «обе тривиальны подряд» в примерах — лишь 184, потому доминирует
смешанный случай — это и отсекает Флавор B ниже).

**Почему это почти не машинерия.** Парсер **уже** опционален (нет
ключевого слова → `requires=None`, [parser.py:303-309](../../src/eatc/parser.py#L303));
кодоген **уже** трактует `None` как «ничего не эмитить», что тождественно
`true` ([codegen.py:559](../../src/eatc/codegen.py#L559)); дамп AST печатает
клаузу только при `not None` ([astdump.py:108](../../src/eatc/astdump.py#L108)).
Правка сводится к **снятию** проверки правила 5
([checks.py:118-132](../../src/eatc/checks.py#L118)) и её зеркала в
selfhost `Check`. Грамматика, IR, дампы — байт-в-байт прежние (пропуск
клаузы уже неотличим от `true` на всех нижних слоях).

```
# было (4 строки)              # стало
func not_found() -> Resp       func not_found() -> Resp {
    requires true                  ...
    ensures true               }
{ ... }
# смешанный случай — стало (одна содержательная клауза):
func clamp(v: i32) -> u32
    ensures result <= 65535
{ ... }
```

**Рамка — осознанное переопределение формы правила 5, не обход.** Сам
контракт остаётся частью сигнатуры и доказывается верификатором;
отменяется лишь *церемония* писать `true`. §5.2 SPEC переписывается:
«обязана иметь `requires`; отказ — явный `requires true`» → «контракт
опционален; **отсутствие клаузы ≡ `true`**». Что теряется — сигнал ревью
«забыл vs осознанно отказался»; реальную защиту не трогает (`requires
true` ничего не доказывал, границы/переполнение/деление ловит
верификатор+trap независимо). Опциональный возврат сигнала — строгий линт
`make check --strict-contracts` (не в грамматику), по спросу.

**Отвергнутые флаворы.** B — постфиксный маркер тотальности `func f() ->
T !`: сохраняет видимость отказа, но покрывает лишь «обе тривиальны»
(~10% шума) и требует правки лексера/грамматики/selfhost/tmLanguage.
C — `requires _` вместо `requires true`: строка остаётся, экономит 3
символа. Оба проигрывают A по охвату/цене.

### 6.6 L2c — прочий сахар (по спросу, отдельные решения)

Кандидаты, каждый — свой микро-решение пользователя (§10):
- **Вывод типа для `const`** от литерала/RHS (`const a = 10`). Касты между
  ширинами остаются явными — вывод только для очевидного RHS.
- **`else`-форма распаковки** для «дефолт вместо проброски»
  (`const v = parse_i32(s) else { PORT_DEFAULT }`) — закрывает `port_arg`,
  где `?` не подходит (нужен дефолт, а не возврат ошибки).
- Именованные sentinel-константы вместо `4294967295` в примерах
  (`lib/Const.NONE` уже есть — это дисциплина примеров, не язык).

## 7. Что реализуем НЕ (явные отказы)

- **Роутер-фреймворк с регистрацией колбэков** (`router.get(path, fn)`) —
  правило 9 (нет указателей на функции). Диспетч остаётся статическим
  `match` у пользователя; библиотека даёт только предикаты `Req.*`.
- **Цикл сервера в библиотеке** — единственный `loop` обязан быть в
  `main` (SPEC §209). Библиотека даёт `.tick()`, цикл — у пользователя.
- **`Vec`/динамические контейнеры для тел/коллекций** — нет кучи
  (правило 3). Всё bounded-пулами, как сейчас.
- **`unwrap()` без ветки ошибки** — молчаливо отброшенный `Err`/`None`
  запрещён (правило 7). `?` пробрасывает явно, `else` даёт дефолт явно.
- **Полные дженерики ради обобщённого билдера** — цена/польза плохие
  ([GENERICS_PLAN §5](GENERICS_PLAN.md)); `JOut`/`Bytes` фиксируем на
  `[u8;65536]`.
- **Дефолтные контракты как «контракт не нужен»** — контракт по-прежнему
  часть сигнатуры; меняется лишь то, что `true` можно не писать. Правило
  5 (наличие контракта) не отменяется.

## 9. Паритет

- **L1**: зеркала в `selfhost/*.eat` **нет** — трогается только `lib/`

## 10. Открытые вопросы (нужно решение пользователя до кода)

- **L1-a (буфер тела):** ✅ **РЕШЕНО 2026-07-18 — вариант ii.** Носитель
  `Bytes` (`[u8; 65536] + n + over`) живёт в самой `lib/Server.eat`, в
  него пишут и текст, и JSON; `.reply_bytes(head: Resp, b: Bytes)`
  стримит заголовки+тело. Server не тянет зависимость `Server`→`Json`
  (не всякий сервер JSON-ный), буфер один для тела любой длины.
- **L2-a (`?`):** синтаксис постфикса `?` (конфликтов с текущей
  грамматикой нет — `?` сейчас не оператор); поведение при несовпадении
  типа ошибки функции и выражения (запрет на компиляции). Выделять ли в
  отдельный `ERR_PROPAGATION_PLAN.md` ([GENERICS_PLAN §10](GENERICS_PLAN.md)).
- **L2-b (дефолтные контракты):** согласие снять конвенцию «отказ виден
  на ревью». Пользователь высказал аппетит — нужна финальная фиксация в
  SPEC §2/§7 (это и есть акт языкового решения). Альтернатива-компромисс:
  дефолт только для `requires true` (частый случай), `ensures` писать
  всегда, — обсудить.
- **L2-c:** нужен ли вывод типа `const` и `else`-форма, или ограничиться
  `?` + дефолтными контрактами.

## 11. Инварианты (какие обходы Power of 10 план НЕ открывает)

- **Кучи нет.** L1 — те же bounded-пулы/буферы; L2 — сахар без данных.
- **Рекурсии нет.** `.tick` — линейный автомат; `?` разворачивается в
  `match`, граф вызовов остаётся DAG.
- **Цикла без границы нет.** Единственный `loop` — по-прежнему в `main`
  пользователя; все библиотечные циклы — `for` со статическим потолком.
- **Глобалов нет.** `Server` — `let` в кадре `main`, как `Store` в Todo.
- **Молча отброшенного результата нет.** `?` пробрасывает `Err/None`
  явно; `unwrap` без ветки отвергнут (§7); дефолтный контракт — это
  `true`, а не отсутствие проверки.
