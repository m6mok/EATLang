## 1. Поверхность паритета (суть задачи)

У всех зеркалированных фаз есть эталонный дамп, который diff'ается
байт-в-байт. У верификатора наблюдаемых артефактов **два**, оба нужно
сделать сверяемыми:

1. **Результат анализа** — множество `(вид проверки, узел) → доказано?`
   и агрегат отчёта. Сейчас детерминированно нигде не печатается:
   `build` выводит только сводку `доказано N из M`, порядок обязательств
   — по `id(node)` (невоспроизводим). → **этап 0** вводит канон дампа.
2. **Элизия в IR** — доказанная проверка исчезает из бинарника
   ([codegen.py](../src/eatc/codegen.py): `if getattr(stmt, "proven",
   False): return`; атрибуты `no_overflow/in_bounds/div_safe/cast_ok/
   shift_ok`, `requires_proven/ensures_proven`). Но **канон selfhost —
   это `eatc ir`, где проверок нет — они все в рантайме**. Верификатор
   в selfhost создаёт **новый канон IR** — `eatc build`-IR (проверки
   сняты), — который SelfIr-с-верификатором обязан воспроизвести
   байт-в-байт, включая **новый фикспойнт bootstrap**. → **этап 5**.

## 2. Почему это самый инвазивный self-host трек

`verifier.py` держит структуры, дорогие в EATLang (нет куч, хеш-мап,
рекурсии; всё — фиксированные пулы):

- `State` (verifier.py:211) = **словари по строковым путям** (`ivs`,
  `rels`, `nz`, `holes`; ключи `имя` / `self.поле` / `var.поле`).
  Хеш-мап нет → **интернированный пул путей** + параллельные массивы,
  `copy`/`join`/`kill` над индексами.
- **Замыкание разностных ограничений** (`State.closure`, verifier.py:272,
  Флойд–Уоршелл) — O(n³) по пулу путей.
- **Итеративные пассы-фикспойнт** инвариантов пулов/полей (`run`,
  verifier.py:381, до 3 проходов) + «пробный» проход без следов
  (`_probe`, `_begin_pass`/`_finish_pass`).
- `Iv` (диапазоны), hull/inter, символический `result`, summaries
  функций (`hive`, дырки-сентинелы) — всё поверх пулов, без рекурсии.

Образцы recursion-free машин уже есть в [Check.eat](../selfhost/Check.eat)
(машины выражений/инструкций, resolve/const_eval — явные стеки кадров);
порт идёт по этим образцам. Объём — сопоставим с `Check.eat` (237 КБ).

- Новая команда `eatc verify <файл>` → детерминированный отчёт

  вид), строки вида `bounds 12:7 proven` / `overflow 30:5 runtime`,

  shift=… requires=… ensures=…`.
- Стабилизировать порядок (сейчас `id(node)` в `Verifier.checks`,
  verifier.py:351) сортировкой по позиции узла.
- Согласовать с `verify_suite` (он проверяет значения `#! expect:`,
  этот дамп — источник тех же чисел).
- Инвазивность нулевая: переиспользует существующую карту `checks`.

### Этап 1. Инфраструктура состояния в EATLang — СДЕЛАН 2026-07-15

Реализация: `selfhost/Verify.eat` (+ `VerifyMain.eat`), гейт
`make verify_selfhost_verify` — `SelfVerify == eatc verify` байт-в-байт
на курируемом списке (`VERIFY_GATE` в Makefile: кейсы 01–06, 09–10, 16 —
bounds/overflow/div/cast/requires/ensures/assert). Паритет и на
интерпретаторе, и на нативном бинарнике.

- `struct St` — состояние потока на параллельных массивах (`kn/ls/lm/
  hs/hm/nz` по индексу пути), копируется присваиванием структуры,
  join'ится `meet`. `Iv` — знаковая пара (sign, mag u64), как `ces/cem`
  в Check (единый диапазон i32…u64 без переполнения i64).
- Пул интернированных путей (этап 1 — только простые имена; `self.поле`
  и `var.поле` — этап 2). **Замер под пул** (риск §5): максимум
  различимых путей в одном `State` на bootstrap-входе — **64** (кадр
  `main`), суммарно интернированных за функцию — **146**; предел пула
  на `State` выставлен 64 (курируемые кейсы — единицы путей), при
  расширении до полного репозитория (этап 4) поднять до 256.
- Три машины по образцу Check (recursion-free, явные стек-кадры):
  выражения-интервалы (`iv_step`), трёхзначная логика (`eb_step`),
  поток (`flow_step`: блок/if/for). Ускорение циклов
  (`accelerate`/`accel_body_iv`/`accel_after_iv`) — линейный ход
  loop-carried путей и охраны из условий; `_refine` — уточнение по
  условиям/requires/assert.
- **Ключевое ограничение EATLang (правило 1, нет рекурсии):** граф
  вызовов обязан быть DAG — запрещён **любой** статический цикл, даже
  если во время исполнения он ограничен явными стеками. Поэтому машины
  не вызывают друг друга по кругу: `iv` не вызывает `eval_bool`
  (числовой контекст and/or/not в кейсах не встречается — возвращаем ⊥),
  `eval_bool` вызывает `iv` в одну сторону, а проверка requires вынесена
  из машины в **отложенный список** вызовов (`pend_n`), обрабатываемый
  на уровне потока (`flush_requires`/`check_requires_site`) после каждого
  выражения. Это зеркалит `_iv_call`→`_check_requires` эталона, не
  замыкая граф. **Порог параметров функции — 6 включая `self`** (≤5
  явных): крупные хелперы разбиты, часть операндов вынесена в поля.
- Упрощения этапа 1 (расширяются в 2–3, паритет на курируемом списке
  цел): без реляционных ограничений (`rels`/`closure`), дырок, summary,
  lockstep, пулов-инвариантов; интервал частного `/` — диапазон типа
  (обязательство div-safe отмечается точно); `loop`/`match` в машине
  потока — заглушки (в кейсах нет). Экспериментально подтверждено: на
  кейсах 01–06, 09–10, 16 эти механизмы не влияют на дамп.

### Этап 2. Отношения и контракты — ✅ СДЕЛАН

**Реляционно-структурное ядро — СДЕЛАНО** (кейсы 07,08,13,19 в гейте,
18 частично):

- Разностные ограничения `p <= q + d` — **разреженный список рёбер в
  `struct St`** (`rp/rq/rs/rm/nrel`, ёмкость 32), не матрица NP×NP
  (риск §5: St копируется на каждом кадре потока, матрица раздула бы
  копии/замыкание). `env_add`/`env_relate`/`rels_kill`; пересечение
  ветвей — `meet_rels` (макс. d). Замыкание Флойда–Уоршелла — во
  **временный плотный буфер `Verify.cl_*`** по индексу пути (не в St),
  `closure`/`closure_k`/`closure_ik`, `cl_le`/`cl_lt`.
- Решатели `decide_rel`/`decide_offset`/`decide_delta` (эталон
  verifier.py:1587–1665), `decompose` (p ± c). Подключены в `cmp_rel`
  (резерв сравнения после интервального `cmp`) и `sub_floor`
  (floor_zero вычитания: `a − b` при `b ≤ a` не уходит ниже нуля).
  Эмиссия рёбер — в `refine_rel` (хвост `_refine`), на `let`/`=`
  (`x = y + 1 → x == y+1`) и на `return` (`result ==` выражение).
- Структурное равенство `expr_equal` — **явный стек `eq_a/eq_b`**
  (рекурсии нет), раскрытие `result` в выражение return (`res_expr`).
  *Упрощение:* коммутативность `+`/`*` не раскрывается (под-сравнение
  перестановки замкнуло бы граф `eq_one→…→eq_one`, правило 1) —
  расхождение только при переставленных операндах.
- **Дотовые пути** `self.поле`/`var.поле` — пул расширен `pp_par/pp_sf`
  (цепочка компонент: имя | `self` | поле-от-родителя; `result` —
  отдельный вид sf=2), `node_path` собирает цепочку явным стеком,
  `rels_kill` снимает рёбра пути. `uncast` — прозрачные касты char/u8.
- **Починен баг захвата r-регистров**: `flow_let/assign/return`
  захватывали интервал значения ПОСЛЕ `flush_requires` (проверка
  requires переоценивает выражения и затирает регистры) — теперь до.
- **Генератор `tools/gen_verifymain.py`**: пересобирает гигантский
  инициализатор `Verify {…}` из полей struct (нуль/NONE/`p: p`),
  `--check` сверяет. Новое поле в St/Verify — правка + прогон скрипта.

**Контракты, битовые, match, write_span — СДЕЛАНО** (кейсы 11,12,21,22,
23,26,27,28,29,30,31,32,33,35 в гейте):

- `assume_ensures`/`map_side`/`assume_conj` (эталон verifier.py:1979–
  2093) — конъюнкт ensures в терминах вызывающего: `map_side` — явный
  стек пост-обхода `+/-` (без рекурсии), `result→bind`, `параметр→arg`,
  `self.поле→получатель`; ветви off-off/off-iv/iv-off дают разностный
  факт + подрезку (`refine_path_iv`). Подключено на `let x = f(...)`.
- `contract_iv` (SPARK-граница, verifier.py:2114) в `call_user` для
  вызова через границу модуля: диапазон типа, обрезанный ensures во
  временном `$result` (кейс 33 — `a[mask(7)]` в позиции выражения).
- **Разрыв цикла (правило 1):** `map_arg` берёт интервал через
  `const_of`, НЕ `iv_eval` — иначе граф `contract_iv→…→map_arg→iv_eval
  →call_user→contract_iv` замкнулся бы. Сложный интервал-аргумент даёт
  «неизвестно» (этап 3; гейт/цели — пути/литералы).
- Битовые `& | ^` (`bitwise`), сдвиги `<< >>` (`shift`, overflow/shift),
  `~` (`bnot` = маска−x) в машине интервалов (эталон `_bitwise`).
- **`match`-плечи** в машине потока (kind 20; арм — kind 21): кадр
  `fmatch`/`fmatch1`/`fmatch2`, subject оценивается (сайты requires),
  плечи с интервалом связки (тип нагрузки) join'ятся.
- `write_span(a,off,len)` — обязательство bounds `off+len<=N` (`call_write_span`).
- `has_call`/`refine_cmp`/`refine_neq` раскрывают прозрачные касты
  (`uncast`, эталон `_has_call`/`_path_of`) — кейс 22 (`u8(c)` в requires).
- if-условие ветви `eval_bool`-ится с отметками (эталон `_flow_if`) —
  кейс 27 (границы `a[i]` в `if a[i] > x`).

**Методы и lockstep — СДЕЛАНО** (кейсы 18,20,25 в гейте):

- **Вызовы методов** в машине выражений (`iv_method`/`iv_method_fin`,
  kind 42): разрешение по типу получателя (`resolve_method`→struct.method),
  сайт requires через отложенный список (как plain-call), `contract_iv`
  cross-module с obj_path получателя. `iv_field` (kind 43): чтение

- **Call-env метода** (`build_method_env`, эталон `_call_env` с obj_path):
  параметры ← дети 1.., `self`/`self.поле` ← снимок путей получателя
  (`snapshot_self`/`restore_self`) до сброса env.
- **Мутирующий `var self`-метод:** kill получателя (и поддерева `p.*`,
  `kill_subtree`) ОТЛОЖЕН до `flush_requires` — иначе факты о получателе

- **StructLit-поля** в `flow_let` (`seed_structlit`): `let c = S{f:e}` →

- **Lockstep-счётчики** (`lockstep`, эталон `_lockstep`): безусловный

  `p >= i + (v0_lo−start)`; условный — только верх (кейс 20).

### Этап 3. Пассы-фикспойнт и дырочный домен — ✅ СДЕЛАН

Паритет **всех 50 кейсов** (01–50) в `verify_selfhost_verify`.

- **Инварианты пулов/полей** (кейсы 39, 40): `verify` гоняет до 3
  пассов-фикспойнт (`begin_pass`/`finish_pass`/`commit_key`/`commit_pool`);
  инвариант пасса k−1 — факт пасса k. Пул — массив (локальный `(fi,имя)`
  или поле struct `(si,поле)`), ключи интернированы в `pki_*`; запись
  элемента — `note_pool`/`note_field`/`store_array`/`elem_iv`, чтение
  сужается `pool_read_refine`/`field_read_refine`. **Копии целых
  массивов НЕ трассируются** (в гейте их нет; источник-копия — мёртв).
  *Побочно:* точный интервал `%`/`/` (`arith_mod`/`arith_div` вместо
  диапазона типа — упрощение этапа 1 снято) — без него `%251`/`%64`
  не давали инварианта пула.
- **Дырочный домен** (кейсы 41, 42, 43): значение «плотное ∪ {сентинел}»
  — параллельные массивы `hk/hls..hsm` в `struct St` по индексу пути.
  Чтение пула-сентинела ставит регистр дырки `ho_*`; `flow_let`/
  `flow_assign` снимают в `env.h*`; guard `!= сент` схлопывает
  (`refine_neq`→`collapse_hole`); `meet` пересекает дырки.
- **Пробный проход тела цикла** (эталон `_probe`/`_exit_rec`): гоняется
  через ОСНОВНУЮ машину потока — кадр for в фазе 1 (`ffor0`→
  `ffor_probe_done`→`ffor1`), НЕ вложенным драйвером (иначе граф
  вызовов замкнулся бы, правило 1). Собирает дизъюнктивное объединение
  значений loop-carried не-accel путей (`collect_observed`/`er_*`/
  `er_finalize` = `_loop_exit`), применяет инвариант на входе тела и на
  выходе цикла (форма «плотное ∪ {NONE}» выживает — кейс 42).
- **Сводки функций** (кейс 42): `finalize_summary` (эталон `_summary`/
  `_hive_summary`) — hive «плотное ∪ {сент}» / интервал; `apply_summary`
  на одномодульном вызове (функция и метод) ставит дырку по hive.
  **Тождество параметру (tag 2) — диапазоном типа** (интервал аргумента
  сквозь требует переоценки → замкнул бы граф; в гейте таких вызовов нет;
  полное — этап 4).
- **Замеры пулов** (гейт — по одному кейсу-файлу): `pki` (ключи пулов/
  полей) и `er_*` (наблюдаемые пути) — единицы на кейс; предел `pki`
  512, `er`/`fov` 64/16, дырки в St — 64 (как пул путей). Для полного
  репо (этап 4) замерить пик и, при нужде, поднять `pki` (как `npp` 64→256).

### Этап 5. Элизия в IR — проход №2 оси `-O` (подчинён OPT_PLAN) — ✅ СДЕЛАН 2026-07-17

**Механизм задан [SELFHOST_OPT_PLAN.md](SELFHOST_OPT_PLAN.md), не этим
документом.** OPT_PLAN §2 запрещает бинарник-на-оптимизацию (`SelfBuild`

проходов единой оси** `eatc ir -O` ↔ `SelfIrOpt`.

**Решения пользователя (2026-07-16):**

- **Э1 — состав прохода: полное зеркало кодогена по аннотациям.**
  Кодоген у `ir` и `build` общий и уже читает аннотации; с вызовом
  `verify()` в `cmd_ir -O` в эмиссии автоматически появляются не только
  элизии trap-блоков, но и `add/sub/mul/shl` с `nsw`/`nuw`
  (codegen.py:1056/1127) и `llvm.assume` на доказанном requires
  (codegen.py:585). Python-кодоген не трогается; `SelfIrOpt` зеркалит
  **все** эти ветки.
- **Э2 — диагностика решений (урок OPT_PLAN §9) — постоянный канал.**
  `eatc verify -O` (fold перед verify, дамп обязательств) и argv-флаг
  `-O` у `SelfVerify`; сверка дампов под конвейером fold→verify —
  постоянная часть гейта (в цикле `verify_selfhost_verify_all`).
  Сверка множества решений идёт **до** сверки эмиссии.

**Механика (по контракту OPT_PLAN §4):**

- Порядок проходов фиксирован: `fold` → `verify` (как в
  [cmd_build](../../src/eatc/__main__.py) — точки `[v,v]` снимают
  проверки ниже); порядок — часть паритет-контракта. Синергия — чтение
  `folded`/`fold_value` в эталоне (verifier.py:2214) ↔ чтение
  `nfs`/`nfm` в selfhost-Verify (новая точка в VerifyExpr).
- Аннотации — **массивы по id узла на Check** (как `nty`/`nfs`):
  Verify держит Check полем (`v.c`), пишет прямо в него; `ir_new(v.c)`
  их видит. Виды: битовая маска узла (overflow/bounds/div/cast/shift +
  assert-proven) + пофункционные `requires_proven`/`ensures_proven`.
  Источник — существующие `ck_k/ck_n/ck_o` (+ атрибуты функций).
- Ir читает доп. флаг в местах, зеркальных codegen.py: `arith`
  (интринзик → plain c nsw/nuw), `shift` (обе trap-ветки, `shl nuw`),
  индексация 32/64 бит, `write_span`, cast-семейство, assert,
  пролог requires (элизия/`llvm.assume` + зеркало `assume_safe`),
  эпилог ensures. Дефолт (маска 0) оставляет канон нетронутым.
- `IrOptMain`: `run_typed` → `ct_fold_pass` → `verify` (аннотации) →
  `ir.emit()` с `opt: 1`. Канонные цели Verify-код не таскают
  (веса файлов — только в `SELFHOST_IR_OPT`, как CheckFold).
- **Bootstrap-фикспойнт на opt-режиме — необязателен** (OPT_PLAN D6):
  канон `eatc ir` ↔ `SelfIr` (без `-O`) остаётся жёстким фикспойнт-
  гейтом; `-O` сверяется своим diff-гейтом.
- Проверить взаимодействие с `--trap-codes` (`ir -O --trap-codes`
  легален в Python; отдельный selfhost-вехикл под комбинацию не
  заводится — OPT_PLAN §2).

  проверок (напр. `overflow` и `bounds`) — фиксировать вторичный

  правках верификатора смотреть на него, а не на пофайловую зелень.
