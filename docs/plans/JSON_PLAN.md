# План: разбор JSON в EATLang (библиотека `lib/Json.eat`)

- **Статус:** ✅ ВЫПОЛНЕН (2026-07-16, этапы 0–2; этап 3 «дроби» —
  осознанно не реализован, отдельное решение — [FIXED_POINT_PLAN](FIXED_POINT_PLAN.md))
- **Роль:** справка — дизайн-контракт библиотеки `lib/Json.eat` (§11 — итоги)
- **Правки языка/грамматики:** нет
- **Новые аксиомы ОС (SPEC §7):** нет
- **Зеркало в selfhost:** не требуется (`lib/Json.eat` на EATLang)
- **Опирается на:** [MODULES_PLAN](MODULES_PLAN.md) ✅ (`lib/`)
- **Блокеры:** нет
- **Блокирует:** —

Изменений языка не требует — JSON целиком
выражается наличными средствами (массивы, `str<N>`, enum, `match`,
`for` со статической границей, реактивный `loop`). Это **дизайн-документ
библиотеки**, а не правка компилятора; правила Power of 10 не ослабляются.

## 1. Мотивация и рамка

JSON показателен тем, что рекурсивен по определению: значение может
содержать объект из значений. Прямой рекурсии в EATLang нет (правило 1),
кучи нет (правило 3), `while` нет (правило 2). Поэтому «любой JSON»
невыразим — но **конкретный JSON** выразим полностью, ценой объявления
трёх чисел. Каждое свойство JSON бьётся ровно об одно правило и лечится
одним приёмом: назвать потолок.

| Свойство JSON | Правило | Потолок |
| --- | --- | --- |
| значение вложено в значение (рекурсия) | 1 (нет рекурсии) | `MAX_DEPTH` — глубина явного стека |
| дерево растёт в куче | 3 (нет кучи) | `MAX_NODES` — ёмкость пула узлов |
| строки/числа произвольной длины | 3 (нет кучи) | `ARENA` — байтовая арена под строки |
| «читать, пока не кончится» | 2 (нет `while`) | `MAX_INPUT` — граница цикла-драйвера |

Ни одно из превращений не открывает обход правил: превышение любого
потолка — **штатная ошибка-значение** (`Result::Err`), а не trap и не
segfault. Незавершаемость исключена по построению.

## 2. Числа (пределы библиотеки)

Стартовые значения; выносятся в `const` и подбираются под задачу
(self-host компилятора, конфиги МК, тест-фикстуры):

| Константа | Значение | Смысл |
| --- | --- | --- |
| `MAX_INPUT` | 65 536 | граница цикла-драйвера = верх числа токенов |
| `MAX_NODES` | 4 096 | ёмкость пула узлов (весь «heap» дерева) |
| `MAX_DEPTH` | 64 | глубина стека кадров (= «стек вызовов» разбора) |
| `ARENA` | 65 536 | байты под все строковые значения и ключи |

Все — в границах §6 (массив ≤ 65 536; строки живут в байтовой арене,
`str<N>` (N ≤ 256) — только на геттере `as_str`). При превышении —
`JErr.Capacity` / `JErr.Depth`, программа завершается штатно.

## 3. Модель данных: дерево → плоский пул с индексами

Указателей нет (правило 9): «ссылка на потомка» — `u32`-индекс в массив.
Тегированный узел без union — `kind` плюс поля, часть которых не
используется при данном виде. Дети связаны списком (`first_child` +
`next_sib`) — так вложенные контейнеры естественно чередуются в порядке
аллокации, не требуя непрерывности среза.

```text
const SENT: u32 = 4294967295   # «нет узла»

struct JNode {
    kind: u32       # 0 null · 1 bool · 2 num · 3 str · 4 array · 5 object
    ival: i32       # для num (см. §7 о дробях)
    bval: bool      # для bool
    s_off: u32      # для str — срез в арене
    s_len: u32
    first_child: u32   # array/object, иначе SENT
    next_sib: u32      # следующий брат, иначе SENT
    key_off: u32       # член объекта — ключ (срез в арене)
    key_len: u32
}

struct JDoc {
    nodes: [JNode; 4096]   # MAX_NODES — весь «heap» дерева
    n: u32                 # курсор пула
    arena: [u8; 65536]     # ARENA — все строки подряд
    a: u32                 # курсор арены
}
```

Аллокация узла — инкремент курсора с явной проверкой ёмкости; превышение
возвращается значением:

```text
func alloc(var self, k: u32) -> Result<u32, JErr>
    requires k <= 5
    ensures true
{
    if self.n >= 4096 {
        return Err(JErr.Capacity)
    }
    let id: u32 = self.n
    self.nodes[id] = JNode { kind: k, ival: 0, bval: false,
        s_off: 0, s_len: 0, first_child: SENT, next_sib: SENT,
        key_off: 0, key_len: 0 }
    self.n = id + 1
    return Ok(id)
}
```

## 4. Рекурсия → явный стек кадров

`parse_value` **не вызывает себя** — иначе граф вызовов перестанет быть
DAG и правило 1 упадёт на компиляции. Вместо стека вызовов — массив
кадров; его глубина и есть `MAX_DEPTH`. `push` требует наличия места
контрактом, а переполнение возвращает именованную ошибку вместо
segfault'а:

```text
struct Frame {
    container: u32   # индекс собираемого array/object
    last: u32        # последний присоединённый ребёнок (O(1) append), иначе SENT
    want_key: bool   # для объекта — ждём ключ или значение
}

struct PStack {
    fr: [Frame; 64]   # MAX_DEPTH
    d: u32            # вершина
}

func push(var self, c: u32) -> Result<u32, JErr>
    requires true
    ensures true
{
    if self.d >= 64 {
        return Err(JErr.Depth)     # «слишком глубокая вложенность»
    }
    self.fr[self.d] = Frame { container: c, last: SENT, want_key: true }
    self.d = self.d + 1
    return Ok(self.d)
}
```

## 5. Драйвер: один цикл со статической границей, `match` по токену

«Пока есть вход» → `for _ in 0..MAX_INPUT { … break }`. Один проход:
токен присоединяется к вершине стека; `{`/`[` делают `push`, `}`/`]` —
`pop`. Рекурсии нет — вся вложенность живёт в `PStack.d`. `match` по виду
токена обязан быть исчерпывающим (правило 7); результаты нельзя проглотить
молча — `?`-проброс или `discard` (правило 7).

```text
func parse(var doc: JDoc, var st: PStack, src: [u8; 65536], len: u32)
        -> Result<u32, JErr>
    requires len <= 65536
    ensures true
{
    var lx: Lexer = Lexer { src: src, len: len, pos: 0 }
    for _ in 0..65536 {                 # граница = MAX_INPUT, не while
        let t: Tok = lx.next()
        match t.kind {
            LBrace  { discard container(doc, st, 5)? }
            LBracket{ discard container(doc, st, 4)? }
            RBrace  { discard close(st, 5)? }
            RBracket{ discard close(st, 4)? }
            Scalar  { attach(doc, st, t)? }
            End     { break }           # единственный выход из цикла
        }
    }
    return top_result(st)
}
```

## 6. Верификация и артефакты

Каждое `self.nodes[id]` / `self.buf[off]` — потенциальный trap границы.
Проверки `if self.n >= 4096` и `if self.d >= 64` дают верификатору
инвариант «после guard индекс `< N`»; он это доказывает и **снимает
runtime-проверку** ([VERIFICATION_PLAN.md](VERIFICATION_PLAN.md)).
`eatc build` печатает точную верхнюю границу стека процесса: при
`MAX_DEPTH = 64` без рекурсии глубина считается точно (SPEC §8).

## 7. Числа без float

Плавающей точки нет (SPEC §1, осознанно). Варианты для `3.14` / `1e9`:

- **MVP**: принимать только целые (`i32`/`i64`); `.`/`e` — `JErr.BadNum`.
  Достаточно для конфигов и self-host-нужд.
- **Fixed-point**: хранить мантиссу + десятичную экспоненту (`ival` +
  масштаб) с объявленной точностью; парсинг — целочисленный.
- Отдельным решением пользователя — обсуждать `f32` (сейчас SPEC против).

Выбор фиксируется при реализации; по умолчанию — MVP.

## 8. Режимы

- **DOM** (дерево в памяти): §3–5, ценой `MAX_NODES` + `ARENA` +
  `MAX_DEPTH`.
- **SAX** (события без дерева): нужен только `MAX_DEPTH` (стек скобок для
  валидации), пул узлов не нужен. Чистый потоковый фильтр в реактивном
  `loop` — самая родная для EATLang форма; кандидат на первый этап.

## 9. Этапы

0. ✅ **SAX-валидатор** — токенайзер + стек скобок; проверяет корректность
   потока, ничего не строит. Минимум риска, обкатывает лексер и потолки.
1. ✅ **DOM-парсер** — пул `JNode` + арена + стек кадров (§3–5), целые числа
   (MVP §7). Геттеры: `get(obj, key)`, `at(arr, i)`, `as_int`, `as_str`.
2. ✅ **Сериализатор** — обход дерева итеративной машиной (без рекурсии,
   явный стек), вывод через `write_span`/RtStr.
3. **Дроби** — fixed-point по §7, если понадобится; отдельное решение
   (не реализовано, см. [FIXED_POINT_PLAN](FIXED_POINT_PLAN.md)).

Границы этапов и связь с `lib/` (import/export, снапшот `verify_sig`) —
по [MODULES_PLAN.md](MODULES_PLAN.md).

## 10. Итоги реализации (2026-07-16)

Всё — один модуль [lib/Json.eat](../../lib/Json.eat) (import `hex_val`
из `lib/Hex.eat`); витрина — [examples/json/](../../examples/json/)
(`make run_json`, в `make check` и `make verify`); интерфейс — в
снапшоте `tests/sig/lib.sig`. Решения, зафиксированные при реализации:

- **Одна машина на оба режима.** `validate()` (этап 0) — тот же драйвер
  и грамматика, что `parse()`, но в «сухом» режиме (`dry`): пул узлов и
  арена не трогаются, потолок SAX — только глубина 64 (§8). `Ok` —
  число принятых токенов.
- **Числа — MVP §7:** только целые `i32`; дробь, экспонента, длиннее
  11 цифр, вне диапазона, ведущий ноль — `Err(JErr.Num)`.
- **Строки:** escape `\" \\ \/ \b \f \n \r \t \uXXXX`; `\uXXXX`
  кодируется в UTF-8 (1..3 байта), суррогаты D800..DFFF вне MVP —
  `Err(JErr.Str)`; управляющие байты в строке без escape — ошибка.
- **Строгий JSON:** без хвостовых запятых, ключ — только строка,
  ровно одно значение верхнего уровня.
- **API:** `JDoc` (`put`/`load`/`clear` → `validate`/`parse` →
  `kind_of`/`count`/`at`/`get`/`as_int`/`as_bool`/`is_null`/`as_str`/
  `str_is`/`key_is`), геттеры — через `Option`; `JOut.ser` — этап 2,
  минифицированный вывод, клиент печатает `write_span(o.buf, 0, n)`.
  Ошибки — enum `JErr` (`Capacity`/`Depth`/`Input`/`Syntax`/`Num`/`Str`).
- **Состояния кадра вместо отдельной проверки скобки:** состояния
  0–4 бывают только у объекта, 10–12 — только у массива, поэтому
  сверка «`}` закрывает объект» бесплатна (см. `step`).
- **Находка — защитные копии немутирующего `self`:** эталон-интерпретатор
  копирует получателя у метода с обычным `self` (бинарник элидирует);
  на структуре в сотни КБ это делает горячие хелперы лексера
  неподъёмными (~50 мс/токен). Горячие `cb`/`word_is` объявлены
  `var self` без мутации — разбор в интерпретаторе ускорился на два
  порядка. Геттеры оставлены с честным `self`: они вне горячего пути.
- **Арена не переполняется разбором** (содержимое строк строго короче
  входа, вход ≤ 65 536), но проверка ёмкости остаётся — потолок честный,
  а не выведенный из инварианта соседнего пула.

## 11. Инварианты (не открывать обходов)

- Ни одной функции, вызывающей себя прямо или косвенно (граф — DAG).
- Ни одного цикла без статической границы; единственный `break` на цикл.
- Превышение любого потолка — `Result::Err`, не trap и не молчаливое
  усечение.
- `match` по видам узлов/токенов — исчерпывающий; результаты не
  отбрасываются молча.
- При реализации — паритет с self-hosted, если код попадёт в `selfhost/`
  ([MODIFYING.md](../MODIFYING.md)); чисто библиотечный `lib/Json.eat`
  паритетен через общий фронтенд.
