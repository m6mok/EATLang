# FAULTS — журнал ошибок исполнения работ

Сюда записываются ошибки и сбои, встреченные при исполнении этапов
планов (правило [docs/WORKFLOW.md](../WORKFLOW.md) §3.4). Индекс
планов — [README.md](README.md).

Формат записи: дата · план/этап · симптом · причина · как
исправлено/обойдено · урок.

## 2026-07-16 · SELFHOST_VERIFIER_PLAN · этап 4

1. **Вакуумный пофайловый паритет.** Симптом: пофайловая сверка
   «эталон против self-hosted верификатора» показывала зелёным все
   модули компилятора и библиотеки. Причина: файлы без функции `main`
   не проходят проверку типов — оба сравниваемых вывода пусты,
   сравнение ничего не проверяло. Исправлено: содержательное покрытие
   этих файлов даёт тест самоприменения (все модули склеиваются в один
   корректный вход); ограничение задокументировано в плане §5 и в
   описании гейта. Урок: «зелёный» отрицательный кейс (обе стороны
   упали) — не доказательство паритета; у каждого пункта сверки
   должен быть позитивный сигнал.

2. **Глобальные регистры против вложенных циклов.** Симптом: точечные
   расхождения дампов только на коде с вложенными циклами. Причина:
   несколько «регистров» состояния анализатора (список ускоряемых
   путей, снимок значений на входе цикла) были одиночными полями и
   затирались обработкой вложенного цикла до того, как внешний цикл
   их дочитает. Исправлено: состояние переведено на кадровые копии
   (своя копия на каждый уровень вложенности). Урок: в машине с
   явным стеком кадров любое состояние, живущее «через» вложенный
   кадр, обязано храниться в кадре, а не в глобальном поле.

3. **Сверка по устаревшему входу.** Симптом: анализ расхождения
   опирался на склеенный входной файл во временной папке, собранный
   до очередных правок исходников, — часть выводов не сходилась с
   кодом. Причина: временный артефакт не пересобирался. Исправлено:
   вход пересобирается перед каждой сверкой той же командой. Урок:
   временные артефакты сверок пересобирать всегда, привычкой, а не
   по подозрению.

4. **Правка занятого файла невозможна.** Симптом: индекс планов
   (docs/plans/README.md) требовал обновления строки верификатора, но
   файл содержал чужие незакоммиченные правки параллельной работы.
   Обойдено: правка отложена и передана в промт следующего этапа.
   Урок: закреплено правилом WORKFLOW §3.6 (не трогать чужие
   незакоммиченные файлы).

## 2026-07-16 · элизия защитных копий интерпретатора (JSON_PLAN §10, коммит e3ef4bf)

1. **Красный гейт на общем дереве чужим WIP.** Симптом: полный гейт
   этапа падал на verify_bootstrap — segfault `./build/SelfIr2`
   (stage1 при этом «BOOT OK»). Причина: незакоммиченные правки
   параллельного трека верификатора в selfhost/*.eat; собственный
   дифф этапа (interpreter.py, lib/Json.eat) кодоген не трогает.
   Обойдено атрибуцией в три прогона: (а) общее дерево без своих
   файлов — падает так же; (б) чистый HEAD в worktree — зелёный;
   (в) HEAD + свой дифф в worktree — полный гейт зелёный, коммит
   легитимен. Чужой segfault ушёл с финальным коммитом параллельного
   трека (4f8683e). Урок: красный гейт на общем дереве сначала
   атрибутируется, потом чинится; зелёный полный гейт на HEAD+дифф в
   worktree — достаточное определение готовности. Закреплено в
   WORKFLOW §4 (атрибуция + worktree-гейт, с нюансом `.venv`).

## 2026-07-16 · AGENTS_ORCHESTRATION_PLAN · конвенции §§1–3 → WORKFLOW

1. **Красный гейт на общем дереве при одновременных прогонах.**
   Симптом: полный гейт этапа (правки только docs) упал на
   verify_selfhost — `LEX DIFF examples/alias/Alias.eat`; файл давно
   в HEAD, кода в диффе этапа нет. Причина (атрибуция): параллельный
   агент (аудит документации) в тот же момент вёл работы в общем
   дереве — два одновременных make-прогона в одном каталоге затирают
   друг другу артефакты сверок (дампы). Обойдено: полный гейт в
   worktree на HEAD + свой дифф — зелёный целиком, коммит легитимен
   (WORKFLOW §4). Урок: гейт на общем дереве не запускать, пока в нём
   активен другой агент; при любом подозрении на соседа — сразу
   worktree-стенд, не пересчёт на общем дереве.

## 2026-07-16 · REFACTOR_SELFHOST_PLAN · этап 1 (потолок 100)

1. **`make bench` падал на замере размеров до начала правок.**
   Симптом: `bench_size` → трейсбек
   `ValueError: invalid literal for int(): ' 16 (zerofill)'`
   (tests/bench/bench.py:676). Причина: новый вывод `size -m` на
   macOS дописывает к числу пометку вида `(zerofill)`; парсер ждал
   голое число. Ошибка воспроизводилась на чистом состоянии до
   правок этапа — не следствие рефакторинга. Исправлено:
   `int(val.split()[0])` — берём первое поле. За падением size
   скрывался второй устаревший кейс: стресс токенов ждал отказа на
   ~140К при пределе 131072, но предел поднят до 262144 ещё этапом 4
   верификатора (4f8683e) — bench с тех пор целиком не добегал.
   Кейсы сдвинуты на ~260К/~270К. Урок: эталон метрик «до» снимается
   в начале этапа — это же выявляет сломанные инструменты замера до
   того, как они смешаются с эффектом правки; при подъёме предела §6
   обходить и стрессы bench, не только tests/.

2. **Автосплит elif-таблицы терял `elif` у таблиц без `else`.**
   Симптом: после скриптового переформатирования однострочных
   elif-таблиц (Ast.eat:130, Lexer.eat:58) сверка потоков токенов
   показала пропажу одного `ELIF`. Причина: запасная ветка скрипта
   для таблиц, заканчивающихся не `else`, а последним `elif`,
   склеивала хвост без ключевого слова. Поймано сверкой лекс-дампов
   до/после (позиции и NEWLINE отброшены — остальное байт-в-байт),
   исправлено точечно. Урок: у механических трансформаций исходников
   держать машинную инварианту (поток токенов не меняется, кроме
   заявленных вставок) и гонять её после каждого файла.
