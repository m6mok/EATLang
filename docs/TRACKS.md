Живой список направлений после спринта оптимизации 2026-07-13
(10 коммитов, детальные замеры и уроки — итерации 4–9 в

гибрид с порогом 32 байта.

## Трек 1. Строковая модель `str<256>` (слой 4, решение по SPEC)

**Сделано 2026-07-13**: направление «read-only по ссылке» реализовано
по плану [STR_MODEL_PLAN.md](STR_MODEL_PLAN.md) (решения пользователя
в §6 там же): defensive copy у вызываемого снята, перекрытие с
`var self`-получателем закрывает caller-копия, атрибуты
`noalias nocapture nofree nonnull [readonly]`; SPEC §5.4, пример
`examples/alias/`, уроки — десятая итерация FINDINGS. Направления
«копия по длине» (по замеру не нужна) и `str<N>` — отложены.

**Проблема (исходная).** Строка — фиксированные 260 байт (`{i32, [256 x i8]}`),
передаётся по значению: каждое связывание — memcpy 260 байт, даже
для трёхбайтового имени. Это доминирующая цена строкового кода
(StrBench ~88 M оп/с против 820 M у арифметики; получатель без
`var self` копирует агрегат при каждом вызове — `Rt.write(self)`
копирует 260 байт, чтобы вывести 5).

**Направления дизайна** (обсудить до кода, у каждого свои trade-off):

- копия по фактической длине (`memcpy ln + 4` вместо 260) — семантика
  не меняется, меняются оба кодогена; выигрыш ограничен: длина
  известна только в рантайме, LLVM теряет константный размер;
- read-only параметры по ссылке (передача `str` в функцию без
  `var` — без копии; язык уже передаёт указатель, копию делает
  вызываемый) — самый большой выигрыш, но надо доказать отсутствие
  алиасинг-ловушек (`s.append_str(s)`) и зафиксировать семантику
  в SPEC §5;
- `str<N>` с настоящей параметризацией ёмкости (сейчас рантайм
  фиксирован на 256) — отдельная большая тема, тянет за собой
  layout рантайма.

**Объём.** SPEC §5 + оба кодогена + Rt.eat + интерпретатор +
verifier; полный паритет и фикспойнт. Самый инвазивный трек.

## Трек 2. Микроконтроллеры: размер, trap-строки, cross-compile

Язык уже идеален для МК по построению (статическая память, точная
верхняя граница стека §8, нет кучи). Не хватает:

- **политика trap-строк**: IR раздувается ×79 в основном из-за
  уникальных сообщений `stdin:line:col: error: trap: ...`
  (на bootstrap-входе — десятки тысяч глобалов). Для МК нужен режим
  «коды вместо строк» (например, trap(u32 code) с таблицей вне
  флеша или вообще без текстов) — решение по SPEC, меняет аксиому
  trap или добавляет режим кодогена;
- **замер размеров секций** бинарника (text/rodata/данные) в bench —
  метрика для флеша; сейчас меряем только время;
- **cross-compile**: канонический `.ll` уже платформонезависим
  (пустой triple) — собрать пример под ARM Cortex-M (thumbv7m),
  шим аксиом поверх UART вместо stdin/stdout; выяснить историю
  со стеком (128 МБ хост-линковки против килобайт на МК — пулы
  компилятора на МК не нужны, но пример типа mos6502 должен влезать);
- `-Wl,-stack_size` и отчёт §8 — сверить с реальными лимитами МК.

## Трек 3. Верификатор: добивать «100 % доказанного»

- `write_span` доказан (кейсы 28/29). Остальные систематические
  остатки видно по `eatc build` (печатает «доказано X из Y»):
  собрать каталог недоказанных по selfhost и примерам, бить по
  самым частым классам;
- интервалы через поля struct (сейчас факты о полях умирают чаще,
  чем нужно; `var self`-вызов убивает всё состояние получателя);
- связка «ширина маски ↔ величина сдвига» (из
  [CRYPTO_SUITE_PLAN.md](CRYPTO_SUITE_PLAN.md) §1: общий `rotl`
  оставляет одну проверку — верификатор не связывает `32-n` с `n`);
- доказанные факты уже идут в оптимизатор (`nsw`/`nuw` в build-пути) —
  чем больше доказано, тем быстрее и меньше код.

## Трек 4. Хвосты производительности компилятора (низкий приоритет)

По остаточному профилю SelfIr (0.13 с) и FINDINGS:

- `lb_tp`/копии в банкованные пулы: скалярный масками-доказанный
  цикл; попытка аффинной по-банковой формы, которую LLVM распознаёт
  как memcpy, не делалась — потенциал ~десятки мс;
- `g_hash` считается по содержимому при каждом commit — можно
  инкрементально при записи в пул;
- экранированные спаны дампов (`write_span_esc`, значения токенов) —
  посимвольные; батчить между escape-символами;
- LTO на пути `clang .ll + runtime.c` (инлайн аксиом в вызывающих) —
  проверить цену линковки против выигрыша;
- F7 (FINDINGS): RSS emit_ir у Python-эталона — низкий приоритет;
- F2, вторая половина: `err:`-строки self-hosted компилятора всё ещё
  в stdout (код выхода уже честный; перенос в stderr — eprint-хелпер
  на write_err_byte в Rt.eat + правка fail-путей).

## Трек 5. Эргономика языка для крипто/МК (решения по SPEC)

Находки [CRYPTO_SUITE_PLAN.md](CRYPTO_SUITE_PLAN.md) §0 — каждая
требует решения пользователя:

- hex-литералы (`0xEDB88320` сейчас пишется decimal);
- битовое NOT `~x` (сейчас `x ^ 4294967295`);
- беззнаковое вычитание с заворотом / отрицание mod 2³²;
- тип шире u32 (u64) — SHA-512, длины в битах;
- сама криптосюита `examples/crypto/` — готовый план в файле,
  компилятор не трогает.
