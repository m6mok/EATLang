# Процессы языка

Как устроен конвейер компиляции EATLang и какие проверки образуют
«зелёный» статус репозитория. Формальная спецификация — в
[SPEC.md](../SPEC.md), учебник — в [GUIDE.md](GUIDE.md), правила
внесения изменений — в [MODIFYING.md](MODIFYING.md).

## 1. Два компилятора, один эталон

В репозитории живут две реализации компилятора:

| | Python-бутстрап | Self-hosted |
| --- | --- | --- |
| Где | `src/eatc/` | `selfhost/*.eat` |
| Роль | **эталон семантики** | **целевой продукт** |
| Вход | файлы-аргументы CLI | конкатенация модулей в stdin |
| Выход | дампы / бинарник | текстовый LLVM IR в stdout |

Python-бутстрап никуда не исчезает: каждая фаза self-hosted
компилятора дифференциально сверяется с ним **байт-в-байт**.
Целевая картина — использовать self-hosted компилятор (`build/eatc`,
`make eatc`); Python остаётся судьёй.

## 2. Конвейер компиляции

```text
.eat → лексер → парсер → структурные проверки (Power of 10)
     → тайпчекер (типизированный AST, DAG вызовов, глубина стека)
     → статический верификатор (интервалы, отношения, контракты;
       доказанные проверки удаляются из бинарника)
     → llvmlite → текстовый LLVM IR
     → clang + src/eatc/runtime.c (шим аксиом ОС) → нативный бинарник
```

Соответствие фаз файлам:

| Фаза | Python | Self-hosted | Эталонный дамп |
| --- | --- | --- | --- |
| Лексер | `lexer.py`, `tokens.py` | `Tok.eat`, `Lexer.eat` | `eatc lex` |
| Парсер | `parser.py`, `ast_nodes.py` | `Ast.eat`, `Parser.eat` | `eatc parse` |
| Декларации/сигнатуры | `checks.py`, `typechecker.py` | `Check.eat` | `eatc sig` |
| Тайпчекер тел | `typechecker.py` | `Check.eat` | `eatc typed` |
| Верификатор | `verifier.py` | — (только Python) | отчёт `eatc build` |
| Эмиссия LLVM IR | `codegen.py` | `Ir.eat` | `eatc ir` |
| Интерпретатор | `interpreter.py` | — (эталон исполнения) | вывод `eatc run` |
| Рантайм | `runtime.c` (аксиомы ОС) | `Rt.eat` (всё остальное) | — |

Дампы (`lex`, `parse`, `sig`, `typed`, `ir`) — это построчные
текстовые срезы каждой фазы; именно их diff'ает дифференциальная
сверка. `eatc ir` печатает IR **без верификатора** (все trap'ы на
месте) — так self-hosted эмиттер сверяется с llvmlite-текстом.

## 3. Рантайм и аксиомы

Аксиомы языка — пять примитивов ОС в `src/eatc/runtime.c`
(~30 строк C): `read_byte`, `write_byte`, `write_err_byte`, `exit`,
trap. Всё остальное (`print`, `write`, интерполяция, `==` строк)
реализовано на EATLang в `selfhost/Rt.eat`, который идёт **первым
модулем каждой собираемой программы** (в Makefile — переменная `RT`);
`read_line` и `parse_i32` — библиотечные функции `lib/Io.eat` и
`lib/Parse.eat` поверх методов RtStr (этап 2 модулей).

## 4. Команды CLI (`python -m eatc`, в Makefile — `$(EATC)`)

| Команда | Что делает |
| --- | --- |
| `check <файлы...>` | каждый файл отдельно: парсинг, Power of 10, типы, test-блоки |
| `run <файлы...>` | check + запуск `main` интерпретатором (последний файл — главный) |
| `build <файлы...> [-o out]` | check + верификатор + LLVM → бинарник + отчёт (стек, память, «доказано N из M») |
| `lex <файл>` | эталонный дамп токенов |
| `parse <файл>` | эталонный дамп AST |
| `sig <файл>` | эталонный дамп сигнатур |
| `typed <файл>` | эталонный типизированный дамп |
| `ir <файл>` | эталонный текстовый LLVM IR без верификатора |
| `stream <файл> [--lib DIR]` | поток драйвера: Rt + DAG модулей от главного файла, каждый за директивой `#module "путь"` |

Программа собирается двумя способами (docs/MODULES_PLAN.md):

- **низкоуровневый список**: несколько файлов с единым пространством
  имён; последний — главный. Self-hosted эквивалент — `cat` файлов
  в stdin. Так собирается сам компилятор (Makefile `SELFHOST_*`).
- **драйвер импортов**: единственный файл с `import`-блоками —
  `run`/`build`/`check` строят DAG модулей (циклы — ошибка),
  топологически сортируют и компилируют поток с `#module`-директивами;
  Rt.eat подставляется неявным нулевым модулем. Корни путей — `--lib`.

## 5. Процессы проверки (make-цели)

Полный «зелёный» статус — все цели ниже проходят. Порядок — от
быстрых к дорогим.

| Цель | Что проверяет |
| --- | --- |
| `make check` | компиляция всех примеров: парсинг, Power of 10, типы, test-блоки |
| `make run_<пример>` | пример интерпретатором (`run_math`, `run_all`, `run_mos6502`, ...) |
| `make verify` | нативные бинарники всех примеров; вывод бинарника == вывод интерпретатора |
| `make verify_suite` | регрессионный набор верификатора (`tests/verify/`, [план](VERIFICATION_PLAN.md)); каждый кейс объявляет `#! expect: вид=доказано/всего`, кейсы `0/N` защищают от ложных доказательств |
| `make verify_selfhost` | self-hosted лексер/парсер/sig/typed/IR против эталона на **каждом** `.eat` репозитория; самоприменение (тайпчекер типизирует себя, ≈37 000 строк байт-в-байт); interp == native |
| `make verify_bootstrap` | фикспойнт: stage1 (`build/SelfIr`) эмитит IR самого себя == `eatc ir`; clang собирает stage2; stage2 эмитит тот же IR байт-в-байт |
| `make verify_trapcodes` | режим trap-кодов (МК): `build/SelfIrCodes` == `eatc ir --trap-codes` байт-в-байт на bootstrap-входе |
| `make verify_sig` | дрейф интерфейса lib/: sig потока драйвера от `tests/sig/SigProbe.eat` == снапшот `tests/sig/lib.sig` (осознанное изменение — `make regen_sig`) |
| `make bench` / `bench_quick` | нагрузочное тестирование (`tests/bench/`): синтетика, стресс пределов SPEC §6, интерпретатор против бинарника |

## 6. Сборка языка и программ self-hosted компилятором

```sh
make eatc                              # build/eatc — язык из Python-бутстрапа
make eatc_self                         # build/eatc-self — язык, собранный самим собой
make compile SRC="Mod.eat Main.eat"    # .eat → build/Main.ll
make link    SRC="Mod.eat Main.eat"    # build/Main.ll → build/Main
make binary  SRC="Mod.eat Main.eat"    # компиляция + линковка
make run     SRC="Mod.eat Main.eat"    # запустить build/Main
```

Компилятор — фильтр stdin → stdout; `RT` подставляется первым модулем
автоматически. `COMPILER=build/eatc-self` собирает программы
компилятором, собранным самим собой (IR байт-в-байт тот же).

Бинарникам, собираемым из self-hosted IR, нужен стек 128 МБ
(`STACK_FLAGS` в Makefile): пулы компилятора живут в кадре `main`.

## 7. Структура репозитория

```text
src/eatc/        Python-бутстрап (эталон): лексер … codegen, runtime.c
selfhost/        self-hosted компилятор + Rt.eat + планы фаз
lib/             библиотека на EATLang (Ascii, Buf, Const, Fmt, Hex, Io, Num, Parse);
                 подключается списком файлов после Rt.eat
                 (docs/MODULES_PLAN.md, этап 0 — конкатенация)
examples/        эталонные примеры; крупные: mos6502, lexer (проба)
tests/lex|parse|ir/   стресс-кейсы дампов (входят в verify_selfhost)
tests/verify/    регрессионный набор верификатора (verify_suite)
tests/bench/     нагрузочное тестирование
docs/            документация процессов, руководств и планов
build/           артефакты сборки (не в git)
```
