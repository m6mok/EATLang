# Руководство по EATLang

Практический учебник языка. Формальная спецификация — в
[SPEC.md](../SPEC.md); здесь — как на языке писать.

## 1. Философия за одну минуту

EATLang построен вокруг одной идеи: **10 правил NASA (Power of 10)
нельзя нарушить, потому что в языке нет средств их нарушить**.

Отсюда следует всё остальное:

- нет кучи → размер массива и ёмкость строки — часть типа;
- нет рекурсии → граф вызовов — DAG, глубина стека известна точно;
- нет `while` → каждый цикл имеет границу, каждая программа завершается;
- нет исключений → ошибки — значения (`Result`), их нельзя «забыть»;
- нет warnings → всё либо корректно, либо ошибка компиляции.

Язык **тотальный**: любая программа доказуемо завершается (кроме
одного разрешённого `loop` в `main` — модель реактивной системы;
формально — завершается каждая *активация*: виток `loop` и каждый шаг
задачи-автомата в нём, SPEC §5.1; словарь идиомы — `lib/Async.eat`).
Компилятор печатает верхнюю границу памяти программы как артефакт
сборки — это возможно только благодаря перечисленному выше.

## 2. Быстрый старт

Нужны [uv](https://docs.astral.sh/uv/) и clang.

```sh
uv sync                                # окружение (единственная
                                       # зависимость — llvmlite)
make check                             # проверить все примеры
make run_hello_world                   # запустить интерпретатором

make eatc                              # компилятор EATLang из Python-бутстрапа
make eatc_self                         # компилятор, собранный самим собой
make binary SRC="examples/math/Math.eat"   # компиляция + линковка
make run    SRC="examples/math/Math.eat"   # запустить build/Math

# по шагам: .eat → LLVM IR → бинарник (модули через пробел, последний — главный)
make compile SRC="Mod.eat Main.eat"    # → build/Main.ll
make link    SRC="Mod.eat Main.eat"    # → build/Main
# рантайм-модуль selfhost/Rt.eat подставляется первым автоматически
# библиотека lib/ (Args, Ascii, Async, Buf, Const, Fixed, Fmt, Hex, Io,
# Json, Num, Parse, U128)
# подключается тем же списком:
make run SRC="lib/Ascii.eat lib/Num.eat examples/modules/Main.eat"

# либо импортами (docs/plans/MODULES_PLAN.md §2): драйвер сам строит DAG
# модулей и подставляет Rt.eat и lib/; пути — от корня (--lib .)
#
#   import {
#       is_digit,
#       digit_value as digit,
#   } from "lib/Ascii.eat"
#
# модуль публикует имена export-блоком (первым блоком файла):
#   export { fmt_u32, Dec }
# каждое имя — явно (wildcard нет); коллизия и неиспользуемый
# импорт — ошибки, переименование — as с любой стороны
PYTHONPATH=src uv run python -m eatc run --lib . examples/modules/Main.eat
```

Три команды компилятора:

| Команда | Что делает |
| --- | --- |
| `eatc check <файлы>` | парсинг, проверки Power of 10, типы, исполнение test-блоков |
| `eatc run <файл>` | check + запуск `main` интерпретатором |
| `eatc build selfhost/Rt.eat <файлы> [-o out]` | check + верификация + LLVM → нативный бинарник + отчёты |

Для микроконтроллеров у `build`/`ir` есть флаг `--trap-codes`:
вместо текстов trap-сообщений во флеш попадают числовые коды
(`trap N` в stderr), а таблица `; trap <код>: <сообщение>` уходит
комментариями в хвост `.ll` и в файл `<out>.trapmap` рядом с
бинарником (у Mos6502 это −98 % константных данных).

Первая программа:

```text
func main() {
    print("Hello, world!")
}
```

## 3. Переменные и типы

Переменная объявляется с типом и инициализатором — оба обязательны.
По умолчанию она неизменяема; мутабельность — осознанный выбор:

```text
func main() {
    let a: i32 = 10        # неизменяемая
    var b: i32 = 5         # изменяемая
    b = b + a
    print("b = {b}")
}
```

Типы:

| Тип | Описание |
| --- | --- |
| `i32`, `u32`, `u16`, `u8`, `u64`, `i64` | целые; переполнение — trap, а не молчаливый wrap |
| `bool` | `true` / `false` |
| `char` | один байт-символ |
| `[T; N]` | массив ровно из N элементов |
| `str<N>` | строка ёмкостью до N байт |
| `Result<T, E>`, `Option<T>` | встроенные sum-типы для ошибок |

Неявных преобразований **нет**: `i32 + u32` — ошибка компиляции.
Явные преобразования — `i32(x)`, `u32(x)`, `u16(x)`, `u8(x)`, `u64(x)`,
`i64(x)`; выход за диапазон — trap (или статически доказанная
безопасность, см. §10).

`char` — ровно один байт (литерал вне ASCII лексер отклоняет). Код
символа — `u8(c)`, символ по коду — `char(b)`; оба каста тотальны
(trap невозможен) и прозрачны для верификатора. Дальше код расширяется
явно — `u32(u8(c))`:

```text
let c: char = 'A'
let code: u8 = u8(c)          # 65
let next: char = char(code + 1)   # 'B'
```

Целые литералы — десятичные (до 2^64−1) и шестнадцатеричные
(до 16 hex-цифр):

```text
const CRC_POLY: u32 = 0xEDB88320
const SHA_K0: u64 = 0x428A2F98D728AE22
let masked: u32 = flags & ~0x0F    # ~ — битовое NOT (только беззнаковые)
```

Битовые операции (`& | ^ << >> ~`) определены только для беззнаковых
типов; у сдвигов величина ограничена шириной типа (иначе trap), `~`
из типа не выходит и trap'а не имеет. `u16` добавлен для МК
(ADC/DMA-буферы, таблицы регистров): `[u16; N]` вдвое плотнее `[u32; N]`.
`u64`/`i64` — для SHA-512, длин в битах и времени без заворота; на
32-битных МК их деление/умножение стоят вызова `__aeabi_*`
(mcu/common/eabi64.c) — в горячих циклах M0 предпочитайте u32.

Мутабельного глобального состояния не существует. На верхнем уровне
допустимы только `const`, `func`, `struct`, `enum`, `test`:

```text
const LIMIT: u32 = 5
```

Значение const может быть **вызовом comptime-годной функции** — она
вычисляется компилятором (тотальность делает это разрешимым, SPEC
§5.6), результат запекается литералом:

```text
const CRC7: u32 = crc_entry(7)     # вычислено при компиляции

func crc_entry(i: u32) -> u32
    requires i < 256
    ensures true
{
    var c: u32 = i
    for _ in 0..8 {
        if c % 2 == 1 { c = (c / 2) ^ 3988292384 } else { c = c / 2 }
    }
    return c
}
```

Так же вычисляется **массив-константа целиком** — функция, вернувшая
`[скаляр; N]`, запекается в readonly-таблицу (`__const`, для МК —
флеш; кода инициализации на старте нет):

```text
const CRC32: [u32; 256] = crc_table()   # 256 значений посчитаны на компиляции

func crc_table() -> [u32; 256]
    ensures true
{
    var t: [u32; 256] = [0; 256]
    for n in 0..256 {
        var c: u32 = n
        for _ in 0..8 {
            if c % 2 == 1 { c = 3988292384 ^ (c / 2) } else { c = c / 2 }
        }
        t[n] = c
    }
    return t
}
```

**Comptime-годность — это свойство функции, а не аннотация**: ключевого
слова `comptime` в языке нет, компилятор выводит годность сам, обходя
**весь подграф вызовов** функции (граф — DAG, поэтому обход конечен).
Функция годна, когда выполнены **оба** условия.

1. **Чистота.** Транзитивно по вызовам функция не достигает аксиом ОС
   (`print`, `write`, `read_byte`, `write_byte`/`write_span`/
   `write_err_byte`, `exit`, `arg_count`/`arg_len`/`arg_byte`,
   `in_avail`/`ticks`, `socket_*`) и
   `extern`-функций — то есть её результат не зависит от внешнего мира.
   Иначе — ошибка *«не comptime-годна: транзитивно вызывает аксиому ОС
   или extern»*.

2. **Скалярное подмножество (ярус A).** Параметры, возврат и всё тело —
   из целых (`i32`…`u64`), `bool` и одномерных массивов скаляров
   `[скаляр; N]` (ярус A2). Внутри разрешены арифметика, сравнения,
   `and`/`or`/`not`, битовые операции, **целочисленные касты** (`u8(x)`,
   `u32(x)`), `if`, `for` (с `break`), `assert`, индексация, литералы
   `[a, b, …]` и `[v; N]`, присваивание элемента `t[i] = v` и вызовы
   других годных функций. Вне подмножества — `char`/`str`,
   `struct`/`enum`, `match`, `loop`, конструкторы `Ok/Err/Some/None`,
   `len()`, многомерные массивы; при их наличии — ошибка *«не
   comptime-годна: тело вне скалярного подмножества»*.

У вычисления три исхода (следствие тотальности, §1): значение посчитано
и запечено литералом; либо внутри случился **trap** (переполнение,
нарушенный `requires`/`assert`, деление на ноль, индекс вне границ) —
это **ошибка компиляции** `comptime: trap: …` с координатами
const-декларации, а не молчаливое значение по умолчанию; либо превышен
бюджет (1 000 000 шагов или 64 уровня вложенности вызовов, SPEC §6) —
тоже ошибка компиляции. Вызывать годную функцию можно **только
в const-декларации**; чтобы подставить результат в размер массива
`[T; N]` или границу `for 0..N`, заведите промежуточный const.

Та же годность лежит в основе необязательной оптимизации сборки
`eatc build --fold` (ярус B, по умолчанию выключена): годный вызов со
скалярным результатом и константными аргументами компилятор сворачивает
в литерал прямо **в теле** обычной функции. Наблюдаемое поведение не
меняется — если вызов trap'нул бы или не уложился в бюджет, он не
сворачивается и trap'нет в рантайме на том же месте.

Строки поддерживают интерполяцию — выражение в `{}` типизируется
компилятором:

```text
print("итого {a + b}, флаг {flag}")
```

## 4. Функции и контракты

Контракты — часть сигнатуры. Правило 5 требует: у функции
с параметрами — `requires`, с возвращаемым значением — `ensures`.
Отказ от контракта только явный: `requires true` видно на ревью.

```text
func clamp(x: i32, lo: i32, hi: i32) -> i32
    requires lo <= hi
    ensures result >= lo and result <= hi
{
    if x < lo {
        return lo
    }
    if x > hi {
        return hi
    }
    return x
}
```

`requires` — предусловие над параметрами; `ensures` — постусловие над
параметрами **и** именем `result`. Тело контракта — обычное булево
выражение: сравнения, `and`/`or`/`not`, арифметика, обращение к полям
`self` у метода, вызовы функций. Несколько условий связываются `and`:

```text
func at(a: [i32; 8], i: u32) -> i32
    requires i < 8              # индекс в границах
    ensures true
{
    return a[i]
}
```

`requires` проверяется у **каждого вызова** (до входа), `ensures` — на
**каждом** `return`. Нарушение — trap с точными координатами:
у `requires` — места вызова, у `ensures` — возврата
(`error: trap: нарушен requires функции half`). Третья точка проверки —
`assert expr` в теле: та же семантика, trap при нарушении.

Но сперва верификатор пробует доказать каждую проверку статически (§10);
доказанная не попадает в бинарник. `requires` снимается из тела функции,
только когда доказаны **все** её вызовы, — иначе остаётся runtime-trap'ом.

Результат вызова **нельзя молча отбросить** (правило 7):

```text
let m: i32 = clamp(15, 0, 10)   # использовать
discard clamp(15, 0, 10)        # или отбросить явно
clamp(15, 0, 10)                # ошибка компиляции
```

Функция не длиннее 60 statements, не больше 6 параметров — иначе
ошибка компиляции (правило 4 и пределы §6 спецификации).

### extern: граница с C (микроконтроллеры)

Периферию МК (GPIO, таймеры, SPI) язык видит через `extern func` —
объявление C-функции без тела:

```text
extern func gpio_set(pin: u32, on: bool)
    requires pin <= 29

extern func systick_ms() -> u32
    ensures true
```

Реализация пишется на C рядом с проектом и линкуется по имени
(см. mcu/). Правила границы:

- аргументы и возврат — скаляры; в аргументах допустимы массивы
  `[u8|u16|u32|u64; N]` (C получает указатель read-only);
- `requires` проверяется у вызывающего как обычно; `ensures` и
  завершаемость C-кода — на вере (та же граница, что у аксиом ОС);
- интерпретатор extern не исполняет — программа с такими вызовами
  проверяется бинарником (для МК — QEMU).

## 5. Управление

`if` / `elif` / `else` — как везде. `while` в языке **нет** — цикл
обязан иметь границу, известную до запуска:

```text
for i in 0..10 {          # константный полуинтервал [0, 10)
    print("{i}")
}

let data: [i32; 3] = [1, 2, 3]
for x in data {           # число итераций — из типа массива
    print("{x}")
}
```

Границы диапазона — литералы или `const`. Переменная в границе —
ошибка компиляции («не константа времени компиляции, правило 2»).
Идиома «пока не найдём» выражается через `for` + ранний `return`.

Единственная бесконечная конструкция — `loop`, разрешён только
в `main` и не более одного (модель главного цикла реактивной
системы); внутри допустим `break`:

```text
import {
    read_line,
} from "lib/Io.eat"

func main() {
    loop {
        match read_line() {
            Ok(line) {
                print("echo: {line}")
            }
            Err(_) {
                break
            }
        }
    }
}
```

Досрочно завершить процесс с кодом статуса — `exit(code: u32)`.
Как и `loop`, он ограничен: только в `main` и **не более одного** на
программу (иначе граф завершения перестал бы быть предсказуемым); код
после `exit` в том же блоке недостижим:

```text
func main() {
    match read_line() {
        Ok(line) { print("echo: {line}") }
        Err(_)   { exit(1) }
    }
}
```

Недостижимый код после `return`/`break` — ошибка компиляции
(правило 10). Неиспользуемая переменная — тоже (заглушка — `_`).

## 6. Структуры и методы

Структуры живут на стеке и передаются **по значению** — испортить
данные вызывающего невозможно: мутирует только `var self`. Физическую
копию компилятор элидирует — параметры читаются по ссылке, а при
перекрытии аргумента с `var self`-получателем (`s.append_str(s)`)
копию делает вызывающий (SPEC §5.4):

```text
struct Point {
    x: i32
    y: i32

    func add(self, other: Point) -> Point
        requires true
        ensures true
    {
        return Point { x: self.x + other.x, y: self.y + other.y }
    }
}

func main() {
    let p: Point = Point { x: 1, y: 2 }
    let q: Point = p.add(Point { x: 3, y: 4 })
    print("({q.x}, {q.y})")
}
```

Обычный `self` неизменяем. Метод, который меняет получателя,
объявляет первый параметр `var self` и требует изменяемой переменной
(`var`-получатель) на месте вызова:

```text
struct Counter {
    n: i32

    func bump(var self)
        requires true
    {
        self.n = self.n + 1
    }
}

func main() {
    var c: Counter = Counter { n: 0 }
    c.bump()
    c.bump()
    print("{c.n}")     # 2
}
```

`var self` — единственный способ метода писать в чужую память, и он
виден в сигнатуре. Указателей и ссылок в языке нет; остальные
параметры всегда by-value — «алиасинг» невозможен по построению.

### extend: методы вне блока struct

Когда методов много (фазы self-hosted компилятора — сотни), один блок
`struct` разрастается на тысячи строк. Блок `extend ИМЯ { методы }`
дописывает методы к уже объявленному struct — например, из соседнего
файла той же программы:

```text
struct Counter {
    n: i32

    func bump(var self)
        requires true
    {
        self.n = self.n + 1
    }
}

extend Counter {
    func reset(var self)
        requires true
    {
        self.n = 0
    }
}
```

Семантика тождественна методам внутри блока `struct`: те же проверки,
контракты и дампы — после слияния компилятор различий не видит.
Правила:

- struct обязан быть объявлен **раньше по потоку** (порядку склейки
  файлов) и в том же модуле;
- **полей в extend нет** — раскладка структуры читается в одном месте;
- дубликат имени метода — ошибка компиляции;
- блоков extend может быть несколько, методы дописываются по порядку.

## 7. Enum, match и ошибки

```text
enum Mode {
    Idle,
    Active,
    Failed
}

func describe(m: Mode) -> str<16>
    requires true
    ensures true
{
    match m {
        Idle   { return "idle" }
        Active { return "active" }
        Failed { return "failed" }
    }
}
```

`match` обязан быть исчерпывающим — пропущенный вариант это ошибка
компиляции, а не сюрприз в рантайме.

Вариант enum может нести **одно** значение (`Num(i32)`; для нескольких
полей — struct). Конструируется как `Token.Num(42)`, разбирается
`match`'ем с привязкой имени, а `_` игнорирует нагрузку:

```text
enum Token {
    Plus,
    Num(i32),
}

func value(t: Token) -> i32
    requires true
    ensures true
{
    match t {
        Plus   { return 0 }
        Num(n) { return n }
    }
}
```

Enum с нагрузкой нельзя сравнивать `==`/`!=` — только `match` (плоский
enum без полей сравнивать можно).

Исключений нет. Ошибающиеся операции возвращают `Result` — и правило 7
не даст его проигнорировать (`read_line`/`parse_i32` — из lib/,
шапка import опущена):

```text
func main() {
    print("Введите число: ")
    match read_line() {
        Ok(line) {
            match parse_i32(line) {
                Ok(n) {
                    print("получено {n}")
                }
                Err(_) {
                    print("это не число")
                }
            }
        }
        Err(_) {
            print("ошибка ввода")
        }
    }
}
```

Создавать `Result`/`Option` умеют и ваши функции — конструкторами
`Ok(x)`, `Err(e)`, `Some(x)`, `None`. Они **контекстные**: пишутся
только там, где ожидаемый тип известен точно — в `return` функции
с типом возврата `Result`/`Option`, в `let` с такой аннотацией и в
payload вложенного конструктора (`return Ok(Some(x))`):

```text
enum DivError {
    ByZero,
}

func checked_div(a: u32, b: u32) -> Result<u32, DivError>
    requires true
    ensures true
{
    if b == 0 {
        return Err(DivError.ByZero)
    }
    return Ok(a / b)
}
```

В аргументе вызова или выражении без аннотации `Ok(1)` не
скомпилируется — свяжите значение `let r: Result<...> = Ok(1)`.

Встроенные функции: `print(s)` (строка + перевод строки), `write(s)`
(строка без перевода строки), `read_byte() -> Result<u8, IoError>`,
`len(x) -> u32`. Сами `read_line() -> Result<str<256>, IoError>` и
`parse_i32(s) -> Result<i32, ParseError>` — библиотечные
(`lib/Io.eat`, `lib/Parse.eat`): в программе с import-шапкой их
подключает драйвер (`eatc run --lib . Main.eat`), в cat-режиме —
файл в списке (`eatc run selfhost/Rt.eat lib/Parse.eat App.eat`).

Байтовый вывод: `write_byte(b: u8)` — один байт;
`write_span(a: [u8; N], off: u32, len: u32)` — диапазон
`a[off..off+len)` одним вызовом ОС (батч: на порядки дешевле цикла
`write_byte` для длинных диапазонов; `off + len > N` — trap).
Диагностика идёт отдельным каналом: `write_err_byte(b: u8)` пишет байт
в stderr — не смешивается с полезным выводом фильтра в stdout.

Аргументы командной строки (argv без имени программы) — байтовые
аксиомы `arg_count() -> u32`, `arg_len(i) -> u32`,
`arg_byte(i, j) -> u8` (выход за границы — trap, как индекс массива);
сборку `i`-го аргумента в `str<256>` даёт `get(i)` из `lib/Args.eat`.
Итерация — фиксированный цикл с `break` по `i >= arg_count()`.
Интерпретатор берёт argv из хвоста `eatc run FILE... -- ARG...`,
бинарник — из argv ОС.

### Кооперативная асинхронность: суперцикл на in_avail/ticks

`async/await` в языке нет — и не нужно: каноничный ответ мира
Power of 10 — **циклический исполнитель** (superloop) в единственном
`loop` в `main` ([ASYNC_PLAN](plans/ASYNC_PLAN.md), ярус 0). Кирпичи —
две аксиомы без trap-границ: `in_avail() -> u32` (сколько байт stdin
прочитается `read_byte` **без блокировки**; `0` — ждать нечего) и
`ticks() -> u64` (монотонные миллисекунды, первый вызов — 0).

Идиома задачи: кадр — `struct` (всё, что живёт через точку ожидания),
шаг — метод `step(var self)`, «ждать нечего» — ранний `return`.
Каждый шаг — обычная тотальная функция, поэтому задача не может
зажать исполнитель; вечен только сам `loop`. Диспетчеризация — явные
вызовы `step()` по задачам (указателей на функции нет, правило 9).

Словарь идиомы — `lib/Async.eat` (ярус 1): **завершаемая** задача
возвращает `Poll` (`Ready` — завершилась и дальше шаг идемпотентен,
`Pending` — ждёт или работает; расход — `if ready(t.step()) { break }`),
**вечная** — `step` без результата. Кирпичи: `Timer`
(перезаряжающийся дедлайн, `fire(period) -> Option<u64>` — момент
срабатывания) и `Debounce` (подтверждение двоичного уровня за N
выборок). Планировщик обобщённым не бывает — `match`/вызовы по
конкретным задачам остаются в вашем `main`.

Примеры (`examples/async/`): `Async.eat` — две независимые задачи
(`make run_async`); `Pipe.eat` — конвейер фильтров stdin с бюджетом на
виток, backpressure через кольца (`make run_async_pipe`);
`Debounce.eat` — кнопка с дребезгом плюс пульс по таймеру, витрина
`lib/Async.eat` (`make run_async_debounce`).

Сверка `make verify` детерминирована по построению: stdin — файл
(`in_avail` = остаток до EOF), `EAT_TICKS=virt` — виртуальные часы
(+1 на каждый вызов `ticks()`, одинаково в интерпретаторе и
бинарнике).

### Сокеты: HTTP-сервер как циклический исполнитель

Сеть входит в язык шестью аксиомами (SPEC §7,
[HTTP_PLAN](plans/HTTP_PLAN.md) §5): `socket_listen(port) ->
Result<u32, IoError>` (bind+listen; `Err(Fail)` — порт занят),
неблокирующий `socket_accept(fd) -> u32` (сентинел `0xFFFFFFFF` —
никто не ждёт), `socket_avail(fd) -> u32` (страж чтения: `0` — шагнуть
позже; закрытие пира тоже «готово» — чтение даст `Err(Eof)`),
`socket_read_byte(fd) -> Result<u8, IoError>`, неблокирующий
`socket_write_span(fd, buf, off, len) -> u32` (сколько принято ядром —
курсор досылки ведёт вызывающий, правило 7) и идемпотентный
`socket_close(fd)`. Встроенный `IoError` **не расширялся**: «данных
нет» — это `socket_avail`/сентинел (прецедент D1 ASYNC), а не
`WouldBlock`. Дескриптор — `u32`-«адрес» таблицы ядра, не указатель;
операция на чужом/закрытом fd — trap.

Сервер — тот же суперцикл, что и в асинхронности: accept-loop живёт в
единственном `loop` в `main`, обработка соединения — тотальный шаг,
внутренние циклы — `for` со статической границей. Превышение любого
потолка — HTTP-ошибка-значение, не trap (`docs/plans/HTTP_PLAN.md`
§3).

Сверка `make verify` детерминирована **транскриптом**: сокет-аксиомы
при `EAT_NET=<файл>` читают записанный сценарий — текстовые события
`accept` / `data <fd> <байты>` / `close <fd>` (эскейпы `\r` `\n` `\t`
`\\` `\xHH`; данные применяются до ближайшего `accept` — темп задаёт
сценарий), вывод `socket_write_span` идёт в stdout тем же потоком, что
`write`. Дескрипторы детерминированы: слушатель 3, соединения 4, 5, …
Чтение без готовых данных в сверке — trap (суррогат вечной
блокировки): зовите `socket_read_byte` только при `socket_avail > 0`.
Живой режим (реальные неблокирующие сокеты) — `make serve` (порт —
аргумент бинарника: `make serve PORT=9090`), в гейт сверки он не
входит. Пример: `examples/http/Echo.eat` (`make run_http_echo`).

Сам HTTP/1.1 — `lib/Http.eat` (этап 1 плана): `Req` — автомат
request-line + заголовков (`push_byte` на каждый принятый байт →
`PARSE_MORE` / `PARSE_DONE` / готовый статус ошибки: 400 —
синтаксис, 413 — переполнен `REQ_CAP` 8192, 431 — больше 64
заголовков; после `Done`/ошибки шаг идемпотентен), `Header` — срезы
в буфере запроса (не копии), матчеры `method_is`/`path_is`/
`find_header`/`header_val_is` (имена и токены — без учёта регистра);
`Resp` — сборка ответа в буфер `RESP_CAP` (`status_line`/
`header_line`/`body` с автоматическим `Content-Length`; переполнение
— флаг `of`, ошибка-значение, не усечение). Пример:
`examples/http/Hello.eat` (`make run_http_hello`, живой —
`make serve_hello` и `curl -i http://127.0.0.1:8080/`).

### Дроби без float: Q16.16 и Q32.32 из lib/Fixed.eat

Float'ов в языке нет — дроби считает fixed-point
([FIXED_POINT_PLAN](plans/FIXED_POINT_PLAN.md)): число хранится целым
raw `v`, реальное значение = `v / 65536` (`struct Q16`, диапазон
±32768, шаг 1/65536). Арифметика целочисленная: переполнение и деление
на ноль — trap с координатами, а не NaN/inf. Конструкторы `q16(n)` и
`q16_ratio(num, den)`; методы `add`/`sub`/`neg`/`mul`/`div` (mul/div —
через широкий `i64`-промежуток), `eq`/`lt`/`le`,
`abs`/`min`/`max`/`clamp`; округления по пину семантики языкового `/`
(усечение к нулю, знак `%` по делимому): `to_int` — усечение, `round`
— половина от нуля, `floor`/`ceil` — поправка по знаку остатка.
Печать `repr(digits)` усекает дробную часть: двоичная сетка не
совпадает с десятичной (`1/3` → `"0.333328"`). Витрина —
`examples/fixed/` (`make run_fixed`).

Шире и точнее — **Q32.32** (`Q32`, `q32`, `q32_ratio`, тот же API):
raw — весь `i64`, диапазон ±2^31, шаг ~2.3e-10, до 9 дробных цифр в
`repr`. Промежуток mul/div — программный 128-бит из `lib/U128.eat`
([U128_PLAN](plans/U128_PLAN.md)): `U128` на 32-битных лимбах в `u64`
(«маска — это доказательство») и знаковый `I128` (знак + магнитуда) —
сложение/умножение с trap за 128 бит и явными `add_wrap`/`mul_wrap`
mod 2^128, сдвиги, полное сдвиговое деление 128/128 (ровно 128
итераций), печать `repr_hex`/`repr_dec`. Точное произведение
`mul_64(a, b)` двух `u64` — кирпич для крипто/PRNG.

## 8. Модули и импорты

Программа — это упорядоченный список файлов-модулей. Есть два способа
их подать.

**Явный список** (низкий уровень, вход self-hosted компилятора):
файлы перечисляются вручную, последний — главный (с `main`). Драйвер
подставляет первым рантайм-модуль `selfhost/Rt.eat`, дальше идут
библиотечные и ваши:

```sh
make run SRC="lib/Ascii.eat lib/Num.eat examples/modules/Main.eat"
```

**Импорты** (надстройка): модуль объявляет зависимости `import`-шапкой,
драйвер сам строит граф, подставляет `Rt.eat` и нужные `lib/` и склеивает
поток. Пути — от корня проекта, корни задаёт `--lib`:

```text
import {
    is_digit,
    digit_value as digit,   # as — переименование у клиента
} from "lib/Ascii.eat"
```

```sh
PYTHONPATH=src uv run python -m eatc run --lib . examples/modules/Main.eat
```

Модуль публикует имена **export-блоком** — он идёт **первым блоком
файла** (педагогика Вирта: интерфейс проектируется раньше реализации),
не более одного; модуль без него не экспортирует ничего (таков `Main`).
`as` работает и здесь — публичное имя отличается от внутреннего, тело
рефакторится свободно:

```text
export {
    fmt_u32,
    dec_digits as digits,
}
```

Правила границы:

- **Wildcard нет.** Каждое имя — явно; `import *` не существует.
- **Экспортируются** `func`, `struct`, `enum`, `const` (но не `test`).
  Импортированный `const` — полноценная compile-time константа
  у клиента: годится в размер массива `[u8; N]`, границу `for 0..N` и
  выражения контрактов (для МК — главный механизм конфигурации ёмкостей).
- `struct`/`enum` импортируются **именем типа** — поля, методы и
  варианты приходят вместе с ним, отдельно не перечисляются.
- **Замыкание интерфейса**: тип в сигнатуре или контракте
  экспортированной функции обязан быть экспортирован сам — иначе клиент
  не смог бы ни объявить значение, ни проверить `requires`.
- **Коллизия и неиспользуемый импорт — ошибки** компиляции (как
  неиспользуемая переменная); коллизия лечится `as`.
- **DAG без циклов**: граф импортов обязан быть ациклическим — цикл это
  ошибка компиляции. Это правило «нет рекурсии», поднятое на уровень
  модулей; порядок склейки детерминирован (топосорт, tie-break —
  лексикографически), поэтому поток байт-в-байт воспроизводим.

Происхождение имени на месте вызова (`fmt_u32(...)`) не видно — Вирт
требовал `M.x`. Компенсируют это import-шапка как единственное
grep-место и `eatc sig`, печатающий источник каждого имени. Библиотека
`lib/` (Args, Ascii, Async, Buf, Const, Fixed, Fmt, Hex, Io, Json, Num,
Parse, U128) — обычные модули на этом же механизме;
`read_line`/`parse_i32` живут в ней (`lib/Io.eat`, `lib/Parse.eat`),
а не среди встроенных. Модуль `lib/` может импортировать другой модуль
`lib/` (`Fixed` → `Fmt`, `Fixed` → `U128` → `Hex`) — DAG общий на всех.

## 9. Тесты — часть языка

`test`-блоки исполняются **при каждой сборке**; упавший `assert` или
нарушенный контракт вызываемой функции = сборка не удалась. В бинарник
тесты не попадают.

```text
func double(x: i32) -> i32
    requires x > -1073741824 and x < 1073741824
    ensures result == x * 2
{
    return x * 2
}

test double {
    assert double(21) == 42
    assert double(0) == 0
}
```

Связка «контракт + тест» ловит целый класс багов до запуска: тест
вызывает функцию → рантайм-проверка `ensures` замечает расхождение
между обещанием и реализацией.

## 10. Статическая верификация

При `eatc build` верификатор (интервальный + реляционный анализ)
пытается доказать каждую проверку. Доказанное **удаляется из
бинарника**; отчёт показывает счёт:

```text
OK build/All — stack depth: 3, tests passed: 1
  верификация: доказано 9 из 16 проверок (7 остаётся в рантайме)
    assert: 0/4, cast: 1/1, деление: 1/1, ensures: 4/4, ...
```

Анализ консервативен: ложных доказательств не бывает, недоказанное
честно остаётся trap'ом.

Как писать код, который доказывается:

- **Давайте верификатору факты через `requires`.** `requires b != 0`
  перед делением на `b` доказывает «деление на ноль невозможно» — и
  проверка исчезает из кода.
- **Ограничивайте диапазоны.** `requires x >= 0 and x <= 1000`
  делает доказуемыми переполнения всей арифметики над `x`.
- **Индексируйте под контрактом.** `requires j < 3` доказывает
  границы `data[j]` для `[i32; 3]`.
- **Простые `ensures` доказываются реляционно**: `ensures result >= a`
  у функции, возвращающей `a` в ветке `a > b`, доказан — анализ
  связывает `result` с возвращаемым выражением и знает факты ветки.
- **`ensures result == x * 2` доказывается структурно**, если тело
  возвращает то же выражение.
- **Линейные циклы ускоряются**: переменная, меняющаяся на константу
  за итерацию (`count = count - 1`, `sum = sum + 7`), получает точный
  интервал в теле и после цикла — `assert sum == 700` после ста
  итераций `+7` доказан. Нелинейные обновления (`total = total + x`)
  сбрасывают знание до диапазона типа — такие проверки остаются
  в рантайме. Это нормально.

## 11. Отчёт о памяти

`eatc build` печатает верхнюю границу памяти — точный расчёт возможен,
потому что рекурсия запрещена и все размеры статичны:

```text
  память (§8): стек худшей цепочки ≤ 1628 Б, статические данные 479 Б
    кадр main: 844 Б
    кадр describe: 784 Б
```

Стек считается по худшему пути в DAG вызовов из фактических размеров
кадров (ABI-размеры LLVM). Куч нет, `malloc` в сгенерированном коде
не встречается.

## 12. Каталог «выстрелов в ногу», которые не скомпилируются

| Попытка | Ответ компилятора |
| --- | --- |
| глобальная переменная | «на верхнем уровне допустимы только func, struct, enum, const, test (правило 6)» |
| рекурсия, в т.ч. косвенная | «рекурсия запрещена (правило 1): цикл вызовов f -> g -> f» |
| `while`-подобный цикл с переменной границей | «не константа времени компиляции (правило 2)» |
| выбросить результат вызова | «результат отброшен молча; используйте значение или discard (правило 7)» |
| `match` без варианта | «match не исчерпывающий, пропущено: Err (правило 7)» |
| функция с параметрами без `requires` | «обязана иметь requires; осознанный отказ — `requires true` (правило 5)» |
| присваивание в `let` | «объявлена как let — неизменяема» |
| неиспользуемая переменная | «переменная не используется (правило 10)» |
| код после `return` | «недостижимый код (правило 10)» |
| `i32 + u32` | «разные типы (неявных преобразований нет)» |
| shadowing | «имя уже объявлено: shadowing запрещён (правило 6)» |
| функция длиннее 60 statements | «длиннее 60 (правило 4)» |

А то, что нельзя поймать статически, ловится trap'ом в рантайме
с точными координатами: переполнение, деление на ноль, выход за
границы, превышение ёмкости строки, нарушенный контракт.

## 13. Пределы

Компилятор построен по модели TeX: фиксированные ёмкости вместо кучи,
превышение — штатная ошибка `capacity exceeded`. Основные пределы:
60 statements на функцию, 6 параметров, вложенность блоков 8, глубина
выражения 32, 262 144 токена в файле. Полный список — SPEC.md §6.

Эти же пределы — фундамент долгосрочной цели: компилятор EATLang,
написанный на EATLang, без единой динамической аллокации.
