## 1. Верификатор: добить остаток (трек 3, продолжение)

### 1.1 Дизъюнктивный домен «интервал с дыркой» через циклы — СДЕЛАНО 2026-07-14

Реализовано (двадцать пятая итерация FINDINGS, только verifier.py):
дырка `State.holes` теперь проходит через циклы. `_loop_exit` —
объединение входа и всех записей тела (сборщик `_exit_rec`);
hive-summary (`_hive_summary`, `ret_holes`) — для аккумуляторов
`return f`; 2-проходный инвариант тела (проба без следов + real) —
для масочных счётчиков `slot = (slot+1) & M` и обходов `c = pool[...]`.
Bootstrap 5356 → 5456; TrapBench 133 → 138; кейсы 42–43. Риск
«проба оставляет следы» закрыт флагом `_probe`, глушащим `_mark`,
учёт requires и return (не снапшотом — дёшево).

**Остаток**: bounds 921 (Check.eat 471). Дальше — не дизъюнктивный
домен, а §1.2 (суммы полей) и семантические инварианты дерева
(`nns[nfc[c]...]` без guard'а на промежуточном узле — интервалам
недоступно). Кодогенный §2.1 независим.

### 1.2 Реляционные инварианты полей — ПЕРЕОЦЕНЕНО, ЗАКРЫТО 2026-07-14

Гипотеза «assert 0/103 = суммы полей `self.ln + o.ln <= 256`»
**опровергнута разбором** (двадцать восьмая итерация FINDINGS,
карта в VERIFICATION_PLAN): 103 непроказанных assert доминированы
`assert <вызов>(...)` в test-блоках (is_space/hex_val/same/
parse_status), которые верификатор бракует по `_has_call`; сумм
полей ~4 штуки. Доказать основную массу = comptime-вычисление
вызова (§5), а не разностные инварианты. Направление закрыто —
не окупается интервалами.

### 1.3 Мелочь: hive через границу модулей

`_contract_iv` (межмодульный вызов) не знает hive-summary —
find_*-функции, уехавшие в другие модули после этапа модульности,
теряют дырку на границе. Поправить симметрично _apply_summary.

## 2. Кодоген: факты верификатора → LLVM, копии

### 2.1 !range на чтениях пулов — ОТВЕРГНУТО замером 2026-07-14

Реализовано и откачено (двадцать шестая итерация FINDINGS).
Верификатор аннотировал Index-узлы `val_range=(lo,hi,sent)`
(build-only), кодоген эмитил `!range`, дырка → оборачивающийся
интервал; 562 сайта, канон чист, все ворота зелёные, семантически
корректно. Но замер: TrapBench x512 0.77 → 0.83 с (воспроизводимый
регресс ~8 %, не шум), выигрыша нигде. Политика §2.1 была «оставить
только если нейтрально» — не выполнена, откат. Канал «инварианты →
LLVM» уже закрыт `nsw`/`nuw` + `llvm.assume`; `!range` из пуловых
инвариантов на llvmlite=LLVM 22 / Apple Silicon вредит горячему
циклу. Не переоткрывать без нового стека LLVM.

### 2.2 sret прямо в слот назначения (изменение канона — оба эмиттера)

`let x: T = f(...)` сегодня: свежая alloca вызывающего → вызов →
memcpy в слот x (`emit_call` codegen.py:1104, `copy_into` :288).
Передавать слот x как sret-указатель напрямую. **RetBench показал:
при инлайне LLVM уже съедает копию (404M вызовов/с)** — выигрыш
только у неинлайнящихся функций (большие тела). Selfhost: машины
компилятора крупные — проверить профилем, сколько memcpy-возвратов
живёт в SelfIr до начала работ. **Цена высокая**: меняется
канонический IR → зеркало selfhost/Ir.eat байт-в-байт, фикспойнт,
все дампы. Делать только если профиль покажет ≥ единиц процентов.

### 2.3 Хвосты трека 4 (selfhost-код, по остаточному профилю SelfIr 0.13 с)

- `lb_tp`/копии в банкованные пулы: аффинная по-банковая форма,
  которую LLVM распознаёт как memcpy (метрики: CopyBench 220M,
  BankBench 250M, compiler-секция); потенциал ~десятки мс;
- `g_hash` инкрементально при записи в пул вместо пересчёта на
  каждый `g_commit` (Ir.eat);
- `write_span_esc`/экранированные спаны дампов — батчить между
  escape-символами (профиль дампов lex/parse: 1.59–2.45 МБ вывода).

Перед каждым — свежий `sample`-профиль нативного SelfIr: после
модульного этапа и инвариантов расклад мог сместиться.

**Оценка 2026-07-14: ниже пола измерения.** Фазы SelfIr: lex 0.01,
parse 0.02, sig 0.01, typed 0.09, ir 0.14 с — эмиссия IR (ir−typed)
≈ 0.05 с. Хвосты §2.3 — под-части этих 0.05 с, абсолютный выигрыш
единицы мс, а разброс bench ±10–20 мс. SelfIr (0.14 с, ×9 к Python)
слишком быстр для чистого `sample` (пул узлов ≤131072 не даёт
раздуть вход). Оптимизировать вслепую = риск нерегистрируемого
регресса. Трек 4 подтверждённо «низкий приоритет»: браться, только
если появится дешёвый способ надёжного профиля (TCG-плагин под МК
или инструментированный прогон).

## 3. Рантайм-шим и сборка

### 3.1 Аксиома read_span — Python-сторона ПРОВЕРЕНА, selfhost-зеркало отложено (2026-07-14)

Решение пользователя «давай всё» получено. **Python-сторона
реализована и доказана, но откачена** (паритет: «частично зелёное не
коммитится»): `read_span(a: [u8; N], off, len) -> u32` симметрична
write_span, но ПИШЕТ в массив, поэтому `a` обязан быть изменяемым
lvalue (`var`) — новая проверка мутабельности (переиспользует логику
`_check_mutable_receiver`). Сделано и проверено на ветке (не в
HEAD): runtime.c host+mcu (курсор общий с read_byte), codegen.py
(declare + gen_read_span + dispatch), typechecker (_read_span +
_check_mutable_array + сигнатура), interpreter (mutate массива —
`obj[off+i]=b`), verifier (bounds как write_span + **убийство пула
`a`**: read_span пишет произвольные u8, иначе чтение из
заполненного пула получило бы заниженный интервал — несостоятельно),
SPEC §7. **Тест `interp == binary` зелёный**, проверка мутабельности
отвергает `let`-массив.

**Блокер — selfhost-зеркало** (Check.eat + Ir.eat), интрикатное как
u64/extern: ДВЕ раздельные системы кодов (Check `xa` write_span=110,
Ir emit-код write_span=8) + typed-дамп, всё байт-в-байт, плюс
`xrspan` (bounds+gep как `xwspan` :3143, но результат i32 через
`rpre`/`e_rspan`, не void `vpre`/`e_wspan`), новый код в обеих
системах, ert/result-конвенция значащего вызова. Оценка — отдельный
фокус-этап (как каждый axiom-коммит в истории); риск — bootstrap
фикспойнт. Python-дизайн де-рискован, зеркало механическое.

**По выигрышу оговорка:** после слоя 0 read_byte уже 466M байт/с;
read_span экономит только цену вызова-аксиомы на байт, а обработку
(классификацию байта лексером) — нет. Реальный выигрыш на лексере
skromный; замерить ReadBench-вариантом до адаптации лексера.

### 3.2 LTO как release-режим (замерено, отложено)

−10 % времени самокомпиляции, −29 % размера, +2.75 с линковки
(шестнадцатая итерация). В build-путь не встраивать; кандидат —
флаг `--release`/Makefile-цель для финальных бинарников и МК
(в mcu-сборке уже живёт). Решение пользователя о форме флага.

### 3.3 Порог батчинга write_span — СДЕЛАНО 2026-07-15 (32 → 16)

Гибрид «< 32 байт побайтно, иначе fwrite» калибровался при -O0-шиме.
Спан-микробенч после -O2 (тридцатая итерация FINDINGS): кроссовер ~12
байт (fixed-cost fwrite ~22 нс/вызов против ~2 нс/байт), к 16 байтам
fwrite ×1.4. Порог опущен к измеренному оптимуму (32 → 16, риск-free —
C-шим, паритет не трогается). **На продакшн-workload'ах ниже пола**:
SelfIr-эмиссия/parse-дамп/typed-дамп compute-bound, спаны окна 12–31
тонут в вычислении; IoBench нечувствителен. Корректно для
write-доминированных путей, нейтрально на текущих. Закрыто.

### 3.4 Стоимость верификатора в build — O(n²) устранён 2026-07-15

cProfile verify() на SelfIr: `_func_by_key` — линейный скан по
`program.decls` на каждый межпроцедурный вызов (0.206 с self /
24642 вызова). Заменён индексом (dict, первое совпадение), verify()
351 → 314 мс (−10 %), доказательства идентичны. Побочный вывод:
2-проход тел циклов (§1.1) **не** бутылочное горло — гейт не нужен.
Остаток self-time — `_iv` (интервал-эвалюатор, ядро) и `_path_of`
(у пола) — не дешёвые сканы. Тайпчекер профилирован отдельно: плоский,
супер-линейности нет. Дальше по верификатору дёшево не выжать.

## 4. МК-профиль: где снятые проверки стоят по-настоящему

- **Цена проверок во флеше — измерено 2026-07-14** (двадцать седьмая
  итерация FINDINGS, таблица в [mcu/README.md](../mcu/README.md)):
  mos6502 на Cortex-M3, тот же `.ll` с верификатором и без —
  trap-сайтов 322 → 61 (−81 %), `.text` 6156 → 5624 Б (−8.6 %).
  Тезис замкнут: проверки нейтральны на суперскаляре, стоят флеша на
  МК. Самодостаточные константные программы (`work(7)`) непригодны —
  LLVM сворачивает их целиком; нужен вход из ROM (mos6502);
- **цикловый замер (QEMU `-icount`)** — отдельный шаг, нужен
  TCG-плагин счётчика инструкций (в homebrew-qemu по умолчанию нет);
  .text-метрика детерминирована и без него;
- прошивочные порты и периферийный шим — отдельный трек
  ([MCU_PLAN](MCU_PLAN.md)), не смешивать.

## 5. Comptime — ярус A СДЕЛАН 2026-07-15 ([COMPTIME_PLAN](COMPTIME_PLAN.md))

A1 (скаляры) реализован в обоих компиляторах одним паритет-этапом
(тридцать вторая итерация FINDINGS): const-из-вызовов, строгая
по-операционная семантика (= бинарник), бюджет 10⁶ шагов, глубина 64;
вычислитель selfhost — секция struct Check (Eval.eat отдельным файлом
невозможен из-за алиасинг-правила §5.4). Дампы байт-в-байт, фикспойнт
держится, кейсы 44–48. Синергия с треком 3 подтверждена: comptime-
const → точка [v, v] → доказанные границы. A2 (const-массивы в
`__const` — таблицы во флеш) — кейсы 49–50. **Ярус B реализован**
(COMPTIME_PLAN §11, `eatc build --fold`, по умолчанию выключен):
свёртка вызовов с константными аргументами в телах в литералы,
Python-эталон только (как снятие проверок; `eatc ir` не сворачивает →
паритет цел). Замер на пробе: 6 вызовов, call-инструкций 98 → 92.
