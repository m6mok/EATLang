# План полной верификации EATLang

Цель: доказуемость каждой конструкции, из которой будет собран
self-hosting компилятор. Метод: каталог кейсов → регрессионный набор
с ожидаемыми исходами → верификатор под каждый класс кейсов.

Набор: `tests/verify/*.eat`, запуск — `make verify_suite`. Каждый кейс
объявляет ожидание `#! expect: вид=доказано/всего`; кейсы с `0/N` —
обязательные «НЕ доказано» (защита от ложных доказательств).

## Ключевой принцип: модульные контракты

Вызывающий **всегда** вправе предполагать `ensures` вызванной функции,
даже если тот не доказан статически: недоказанный `ensures`
компилируется в runtime-проверку внутри функции, поэтому любой
завершившийся вызов ей удовлетворяет. Если функция с багом нарушает
контракт — она trap'ается до возврата, и предположение вызывающего
не наблюдается ложным. Это даёт SPARK-стиль модульного рассуждения:
функции доказываются по одной, композиция — через контракты.

Симметрично `requires`: внутри функции предпосылка — факт (проверена
вызывающим или trap'ом на входе).

## Арсенал верификатора (реализовано)

1. Интервалы путей (`имя`, `self.поле`, `var.поле`) с уточнением по
   requires, условиям ветвей, assert и `!= const` (подрезка границ,
   nz-факты для делителей).
2. Отношения путей `<`/`<=` с транзитивным замыканием; символический
   `result`; разложение `p ± c` (off-by-one); реляционный пол
   вычитания (`a - b >= 0` при `b <= a`).
3. Структурное равенство выражений (коммутативность `+`/`*`).
4. Ускорение циклов: единственное обновление `p = p ± const` даёт
   точный интервал `v0 + k·d` в теле и после цикла; охрана `p < C`
   ограничивает результат даже при избытке итераций.
5. Модульные контракты: ensures вызова — факты вызывающего;
   summary функций (интервал или тождество параметру).
6. Границы у целей присваивания (`pool[i] = ...`) — проверяются
   и доказываются наравне с чтением.

## Чего не хватает до self-host

Верификатор (по потребности, когда паттерн встретится в компиляторе):

- [ ] отношения через присваивание выражений (`let j = i + 1` →
  факт `j == i + 1`, сейчас только интервал);
- [ ] инвариант связи двух счётчиков в одном цикле (`i == count`);
- [ ] ensures с арифметикой в модульных фактах
  (`result <= pos + width`).

Язык (кирпичи, без которых компилятор не написать, — верификация
к ним готова):

- [ ] `char` ↔ `u8` (классификация символов лексером);
- [ ] `enum` с полезной нагрузкой (узлы AST);
- [ ] файловый I/O через `Result`;
- [ ] массивы struct'ов как пулы + `while`-эквивалент по префиксу
  (уже выражается `for` + охрана, кейсы 04/05);
- [ ] модули (компилятор в нескольких файлах).

Критерий готовности: каждый паттерн из будущего кода компилятора
имеет кейс в `tests/verify/` со статусом ✅, отрицательные кейсы
не деградируют. Набор обязателен к прогону перед каждым коммитом
в верификатор.
