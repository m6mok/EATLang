## 1. Мотивация и факты

- Функционал дублируется по репозиторию (пять копий десятичного
  форматтера, четыре — hex, шесть span-копировщиков; §4) — нужен `lib/`.
- Прошивку МК общая библиотека **не раздувает** — проверено 2026-07-14:
  в IR программы `print("hi")` попадает весь Rt.eat (18 функций), но
  LTO-конвейер МК (`-flto` + `ld.lld`, Makefile) вычищает все 14
  неиспользуемых функций RtStr; text Blinky.elf — 2574 Б; `check_mem.py`
  считает RAM по реальному elf. Реальная цена whole-program —
  **время компиляции/верификации**: tiny-программа тащит 138 проверок
  верификатора и 4 теста Rt.eat.
- Следствие: смысл импортов — «платишь только за подключённое»:
  в сборку попадает только DAG модулей от Main; неимпортированное
  не парсится, не верифицируется, не гоняет тесты.

## 2. Синтаксис (решение пользователя, 2026-07-14)

```text
import {
    fn1, fn2, fn3,
} from "lib/Fmt.eat"

export {
    foo1 as fn1,
    foo2 as fn2,
    fn3,
}
```

EBNF (три продукции, парсер без магии):

```ebnf
import_block = "import" "{" bind {"," bind} [","] "}" "from" str_lit .
export_block = "export" "{" bind {"," bind} [","] "}" .
bind         = ident ["as" ident] .
```

- `as` разрешён с обеих сторон: в export — публичное имя отличается от
  внутреннего (интерфейс заморожен, внутренности рефакторятся свободно);
  в import — разрешение коллизий у клиента.
- import-блоки — только в шапке файла, до первого объявления;
  export-блок — не более одного на модуль, первым блоком файла
  (педагогика Вирта «интерфейс проектируется первым»). Модуль без
  export ничего не экспортирует (Main).
- Хвостовая запятая допустима (как в примерах пользователя).

## 3. Семантика (предложения)

1. **Никакого wildcard.** `import *` не существует. Каждое имя — явно.
2. **Коллизия — ошибка компиляции**: импортированное имя совпало с
   другим импортированным или с локальным объявлением → ошибка;
   лечится `as`.
3. **Неиспользуемый импорт — ошибка** (практика Go; консистентно с
   существующей проверкой неиспользуемых переменных).
4. **Экспортируются**: `func`, `struct`, `enum`, `const` (решение
   пользователя 2026-07-14: const и enum — полноправные единицы
   импорта); `test`-блоки — нет. Глобальных переменных в языке нет —
   вопрос read-only-экспорта (Oberon-07) снят самим Power of 10.
   - `struct`/`enum` импортируются именем типа; поля, методы и
     варианты приходят вместе с типом и отдельно не перечисляются:
     варианты нужны match-исчерпываемости, layout — статической
     памяти клиента (без кучи клиент обязан знать размер).
   - Импортированный `const` — полноценная compile-time константа
     у клиента: годится в размер массива `[u8; N]`, границу цикла
     `0..N` и выражения контрактов. Для МК это главный механизм
     конфигурации: ёмкости буферов проекта задаются одним
     экспортируемым const-модулем.
   - Тип, встречающийся в сигнатуре или контракте экспортированной
     функции, обязан быть экспортирован сам — иначе клиент не может
     ни объявить значение, ни проверить `requires`; нарушение —
     ошибка компиляции у модуля-владельца.
5. **DAG без циклов.** Цикл импортов — ошибка компиляции (прецедент —
   Dafny: модули обязаны допускать линейный порядок; та же связка
   «тотальность ⇒ ацикличность», что у запрета рекурсии). Это правило
   «нет рекурсии», поднятое на уровень модулей.
6. **Пути и детерминизм**: порядок разрешения фиксирован —
   (1) относительно импортирующего файла, (2) корни из `--lib`-флагов
   CLI в порядке задания. Никакого неявного cwd; абсолютные пути
   разрешены, но в репозитории не используются. В дампах и
   trap-сообщениях путь канонизирован (относительный, `/`).
7. **Отступление от Вирта — осознанное**: на месте использования
   `fn1(...)` происхождение имени не видно (Вирт требовал `M.x`).
   Компенсации — пп. 1–3 + import-шапка как единственное grep-место +
   `eatc sig`, показывающий происхождение каждого имени.

## 4. Конкатенация остаётся; импорты разрешает драйвер

У языка нет файловой системы — self-hosted компилятор читает один
поток stdin и не может открыть `"lib/Fmt.eat"`. Поэтому:

- **Драйвер** (Python-`eatc` и обвязка selfhost-сборки) читает
  import-шапки, строит DAG, проверяет циклы, топологически сортирует
  (при равенстве — лексикографически по каноническому пути: порядок
  детерминирован → байт-в-байт) и конкатенирует модули в поток.
- **Компилятор** по-прежнему потребляет конкатенацию; import/export —
  обычные декларации, по которым он проверяет видимость и то, что
  каждый import ссылается на модуль **раньше** по потоку (нарушение —
  ошибка: драйвер и компилятор перепроверяют друг друга).
- **Границы модулей в потоке** — строка-директива драйвера
  `#module "lib/Fmt.eat"` перед каждым модулем: лексер сегодня трактует
  `#` как комментарий, значит директива обратно совместима; она же
  даёт пофайловую атрибуцию trap-сообщений (сейчас — только stdin).
  Ручной `cat a.eat b.eat | SelfIr` без директив продолжает работать
  как один безымянный модуль.
- `eatc build Rt.eat App.eat` (явный список) остаётся низкоуровневым
  режимом; `eatc build App.eat` с импортами — надстройка над ним.

## 5. Видимость, IR и DCE

- **Сделано (этап 2, 2026-07-14)**: неэкспортированные функции
  эмитятся с `internal`-линковкой LLVM (метод — по линковке
  структуры-владельца; модуль 0 не экспортирует ничего, поэтому весь
  Rt и cat-склейки internal, наружу торчат только настоящий `@main`
  и extern-декларации): мёртвый код удаляется даже хостовым
  `clang -O2` без LTO; для МК LTO остаётся вторым эшелоном.
  Зеркало: `Check.is_exported` + `Ir.p_fn` (паритет байт-в-байт).
- Манглинг модульного префикса НЕ получил (решение 2026-07-14:
  отложено вместе со снятием глобальной уникальности имён — internal
  работает и без префикса, пока имена глобально уникальны).
- Rt.eat остаётся неявным нулевым модулем (кодоген требует RtStr);
  его импортировать не нужно.

## 6. Верификация на границе (этап 2, предложение)

Модель SPARK: клиент модуля верифицируется только по
`requires`/`ensures` экспортированных функций, тела модуля
доказываются один раз. Контракты экспорта — машинно-проверяемый
интерфейс; снапшот `eatc sig` коммитится и diff'ается — контроль
дрейфа интерфейса без второго файла (идея ульмского Оберона ценой
нуля синтаксиса).

## 9. Итоги этапа 0 (2026-07-14)

Сделано (модули подключаются явным списком после `selfhost/Rt.eat`;
в Makefile — `LIB_FRONT` для фронтенда self-hosted компилятора):

- `lib/Fmt.eat` — `Dec` (цифры в порядке печати) + `fmt_u32/fmt_u64/
  fmt_i32`: свёл **пять** копий генерации десятичных цифр в `Ir.eat`
  (`lb_dec`, `lb_dec64`, `nb_dec`, `tp_dec`, `g_dec` — остались
  тонкими копиями «Dec → свой синк»).
- `lib/Hex.eat` — `hex_digit`, `write_hex8/16`, `hex_val`: свёл
  mos6502/Main.eat (три функции), `p_hexd` (Ir.eat, удалён —
  `p_esc` пишет `write_hex8`), `hex_val` (Lexer.eat).
- `lib/Ascii.eat` — `is_digit`, `is_alpha`, `is_space`,
  `digit_value`: свёл `is_digit`/`is_ws` (Lexer.eat), ByteUtil.eat
  (examples/modules, файл удалён), магические диапазоны
  в examples/lexer/LexUtil.eat.
- `lib/Buf.eat` — `same` (перенесён из Tok.eat) и `pool16` (первые
  16 байт спана вложенного пула 2×65536): свёл копирующие циклы
  `Parser.tok_is16` и `Check.span_is`.
- `lib/Num.eat` — `min`/`max`/`clamp` (u32): клиентов в selfhost пока
  нет — модуль для МК-конфигурации; подключён в examples/modules.

Находки:

- **Синки не дедупятся**: `*_sb`-методы (Ir.eat) и `write_span2`
  (Check.eat) пишут в буферы разной формы через методы с проверкой
  ёмкости; у свободных функций нет `var`-параметров (мутирует только
  `var self`), поэтому «копия внутрь чужого пула» остаётся методом
  владельца пула. Свести их могли бы var-параметры или дженерики —
  языковое решение, не принимаем молча.
- `RtStr.append_u32/u64` остаются в Rt.eat: Rt — первый (нулевой)
  модуль каждой программы и не может зависеть от `lib/`,
  идущего в потоке после него.
- `eatc check <файл>` требует `main`, поэтому lib-модули проверяются
  не поштучно, а тестами внутри программ, которые их подключают,
  и пофайловыми дампами `verify_selfhost` (каталог `lib` добавлен
  в перебор).

Сделано (язык и оба компилятора):

- **Лексика**: ключевые слова `import`/`export`/`from`/`as` (токены

  импортирующего файла, (2) `--lib`-корни; канонический путь —
  относительный к cwd. `eatc stream` печатает поток (вход self-hosted
  компилятора и сверок Makefile).
- **Selfhost-зеркало**: FSM-состояния 17–19 в Lexer.eat, разбор
  блоков в Parser.eat, таблицы модулей/экспортов/импортов и
  канонизация линейным сканом пула узлов в Check.eat, sig-дамп
  `module/import/export`, trap-префикс по `fmod` в Ir.eat. Дампы
  и IR — байт-в-байт; негативные проверки видимости (коллизии,
  unused-import, замыкание интерфейса) живут в Python-эталоне,
  как и верификатор.
- **Витрина**: examples/modules/Main.eat собирается драйвером
  (`--lib .`), `lib/*.eat` получили export-блоки, стресс —
  tests/parse/ModulesStress.eat (дампы всех фаз в verify_selfhost),
  плюс сверка драйверного потока в verify_selfhost.

Попутно вскрыта и починена латентная мина Check.eat: слот `xa`
машины вызовов хранил либо код встроенного (100..112), либо индекс
пользовательской функции — при fi ∈ 100..112 вызов диспатчился как
встроенный («тип значения не совпадает» на ровном месте; до сих пор
эти fi доставались методам, идущим другим путём). Теперь fi хранится
со сдвигом +4096; Ir.eat не был подвержен (маркер + отдельное поле).

Ограничение этапа 1 (задокументировано): путь в `from "..."` сверяется
компилятором с каноническим путём модуля в потоке, поэтому пути пишутся
от корня проекта (`--lib .`); запись относительно импортирующего файла
разрешается драйвером, но канонику не меняет. Этап 2 — по плану §8.

  интерполяция в цикле, O(n²) при n ≤ 256. Кандидат на языковое
  решение, молча не принимаем.
- Ёмкость str-параметра фактически стирается (`compatible` равняет
  любые `str<N>`, lowering один — `{i32, [256 x i8]}`): `parse_i32`
  принимает любую строку при сигнатуре `str<256>`.
- Контракты над sum-типами не выражают «Ok ⇔ вход корректен» —
  у Result-функций `ensures true` (явный отказ, как `requires true`);
  выразительность контрактов над enum — открытый вопрос.
- Латентное расхождение исчезло: Python-встроенный `parse_i32`
  трогал `str.strip()` (все пробельные), Rt — только `' '`/`'\t'`;
  теперь эталон один — Rt.eat.
- Контракты в sig-дамп не входят (снапшот ловит дрейф
  сигнатур/экспортов/типов): печать выражений в Check.eat — второй
  зеркалируемый принтер, ~60–100 строк — отдельное решение.
