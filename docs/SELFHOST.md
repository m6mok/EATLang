# Self-hosting: компилятор EATLang на EATLang

Цель — компилятор EATLang, написанный на EATLang и собирающий сам себя.
Python-бутстрап (`src/eatc/`) остаётся эталоном: каждая фаза self-hosted
компилятора сверяется с ним дифференциально, байт-в-байт.

Архитектура self-hosted компилятора — фильтр `stdin → stdout`
(файловых дескрипторов в языке нет): модули программы конкатенируются
в stdin, результат печатается в stdout.

## Дорожная карта

| Фаза | Модули | Статус |
| --- | --- | --- |
| 1. Лексер | `selfhost/Tok.eat`, `selfhost/Lexer.eat`, `selfhost/LexMain.eat` | ✅ |
| 2. Парсер (пул AST, явный стек вместо рекурсии) | `selfhost/Ast.eat`, `selfhost/Parser.eat`, `selfhost/ParseMain.eat` | ✅ |
| 3a. Декларации и сигнатуры (таблицы, типы, const, правила 4–5) | `selfhost/Check.eat`, `selfhost/SigMain.eat` | ✅ |
| 3b. Тайпчекер тел (statements, выражения, DAG вызовов) | `selfhost/Check.eat`, `selfhost/TypedMain.eat` | ✅ |
| 4. Эмиссия текстового LLVM IR | `selfhost/Ir.eat`, `selfhost/IrMain.eat` | ✅ |
| 5. Bootstrap: self-hosted компилятор собирает себя (фикспойнт) | рост пулов по всем модулям | ✅ |
| 6. Рантайм на EATLang (в C — шим из трёх аксиом ОС) | `selfhost/Rt.eat` | ✅ |

## Фаза 1: лексер (готово)

Стриминговый конечный автомат: `main` читает байты через
`read_byte()` в единственном `loop`, каждый байт проходит через
`Lexer.step(b)` (15 состояний, включая двухсимвольные операторы
`-> .. == != <= >=`, строки с escape'ами, char-литералы,
комментарии), `Lexer.finish()` добивает незавершённый токен,
финальный NEWLINE и EOF.

Свойства, совпадающие с эталонным лексером `src/eatc/lexer.py`:

- NEWLINE — значимый токен; подавляется внутри `( )` и `[ ]`,
  в начале файла и после другого NEWLINE;
- колонки считаются в символах: continuation-байты UTF-8
  (`0x80..0xBF`) колонку не двигают — позиции токенов после
  кириллических строк совпадают с эталоном;
- байты ≥ 128 — буквы идентификаторов (эталон принимает юникодные
  буквы через `str.isalpha`);
- escape-последовательности строк и char-литералов декодируются
  лексером (`\n \t \\ \" \' \0`);
- ошибки — те же тексты и позиции, что у эталона (в формате
  `err: СТРОКА:КОЛОНКА: сообщение`; в дифф не входят — эталон
  печатает ошибки в stderr).

### Дифференциальная сверка

Формат дампа — построчно `СТРОКА:КОЛОНКА ВИД [значение]`; значение
печатается у INT, STRING, CHAR, IDENT с экранированием
`\\ \n \t \r \0`. Эталонный дамп — команда `eatc lex <файл>`.

```sh
make verify_selfhost
```

1. собирает `build/SelfLex` (LLVM);
2. на **каждом** `.eat` репозитория (`examples/`, `selfhost/`,
   `tests/`, включая стресс-кейсы `tests/lex/`) сверяет
   `eatc lex f` с `./build/SelfLex < f`;
3. сверяет интерпретатор с бинарником на исходниках самого лексера
   (лексер лексирует себя).

Стресс-кейсы `tests/lex/`: пустой файл, файл без завершающего
перевода строки, незавершённые двухсимвольные операторы на EOF,
кириллические идентификаторы, `{{`-экранирование, многострочные
скобки.

### Следствие для бутстрапа: интерпретатор — байтовая машина

Дифф-сверка выявила расхождение бутстрапа: интерпретатор хранил
строки как юникодные `str` Python — `write("{char(b)}")` для
`b >= 128` печатал 2 байта UTF-8 вместо одного, а `len` считал
символы, а не байты (нативный рантайм — байты). Исправлено:
строковые значения интерпретатора — latin-1 (1 символ == 1 байт),
вывод — через `sys.stdout.buffer`. Теперь `print`/`write`/`len`/
индексация байт-точны и совпадают с нативным рантаймом.

## Фаза 2: парсер (готово)

Рекурсия в языке запрещена (правило 1), поэтому парсер — автомат
с **явным стеком кадров** (`selfhost/Parser.eat`): кадр = позиция
в грамматике (func, type, block, expr, if, match, ...) + фаза
(точка возврата после дочернего кадра). Насос `pump()` делает один
ограниченный шаг за итерацию; весь разбор — один `for` с константной
границей шагов. AST — пул узлов (`kind, line, col, payload,
first_child/last_child/next_sibling`), строки узлов — ссылки в пул
байтов лексера, без копирования.

Выражения — **шунтинг-ярд**: стек операндов (индексы узлов) + стек
операторов с сентинелями `( вызов метод индекс массив заполнение
struct-литерал сегмент-интерполяции`. Тонкости, выровненные
с эталоном (рекурсивным спуском):

- сравнения неассоциативны: перед `<` сворачиваются только операторы
  строго выше приоритетом, и если наверху осталось сравнение —
  выражение окончено (второе `<` не съедается, как в `parse_cmp`);
- `not` допустим только там, где его принял бы `parse_not`
  (начало выражения, после `or`/`and`, после сентинеля);
- `-литерал` сворачивается в одно число с позицией минуса
  (`-2147483648` выразим), `-0` остаётся `0`, `- -5` = `5`;
- `allow_struct_lit` (запрет `Name {` в заголовках if/for/match) —
  бит на каждой записи стека: скобки/аргументы/литералы возвращают
  разрешение, индекс и массив наследуют;
- const-выражения (`[T; N]`, `str<N>`) — уровень `parse_add`:
  бит уровня на записи стека, сравнение/логика завершают выражение;
- struct-литерал: поля разделяются запятой ИЛИ переводом строки
  (внутри `{ }` лексер переводы строк не подавляет — как эталон).

Интерполяция строк разбирается **без рекурсии в два приёма**:

1. проход в лексере (`sublex_pass`): значение каждого STRING-токена
   со скобками режется на сегменты; литеральные байты декодируются
   (`{{`/`}}`) в пул, выражения под-лексируются тем же автоматом
   в хвост пула токенов (позиции — от начала STRING-токена, как у
   эталона: под-лексер стартует с `tok.line:tok.col`); под-токены
   сами проходят этот же цикл — вложенная интерполяция обходится
   итеративно;
2. в парсере STRING-операнд с сегментами ставит SEG-сентинель и
   прыгает курсором в под-регион; по EOF региона выражение
   прикрепляется к узлу строки, курсор возвращается.

Дамп AST — préorder-обход с явным стеком; канон формата — `eatc
parse` (`src/eatc/astdump.py`): `kind [line:col] [name=|val=|op=…]`,
отступ 2 пробела, обёртки `ret/requires/ensures/elif/else` вместо
позиционной неоднозначности, поля struct печатаются раньше методов.

Сверка — в той же цели `make verify_selfhost`: `eatc parse f` против
`./build/SelfParse < f` на каждом парсящемся `.eat` (включая
стресс-кейс `tests/parse/Torture.eat`), плюс интерпретатор ==
бинарник на конкатенации собственных пяти модулей — парсер разбирает
сам себя (≈12 000 строк дампа байт-в-байт).

## Фаза 3a: декларации и сигнатуры (готово)

`selfhost/Check.eat` — первая половина семантического анализа:

- **пул интернированных типов**: равные типы имеют равный индекс,
  совместимость — сравнение индексов (у эталона — структурное
  равенство dataclass'ов); виды: целые/bool/char/void, `str<N>`,
  массив, struct, enum, `Result`, `Option`;
- **таблицы деклараций** (два прохода, как эталон): имена struct/enum,
  затем сигнатуры функций и методов (параметры без `self`,
  `var self`-флаг), поля struct, нагрузка вариантов enum, константы;
  коллизии имён, включая встроенные (`print`, `len`, `i32`, ...);
- **resolve и const_eval** — итеративные машины с явными стеками
  кадров (рекурсии нет); константы — знаковые пары (sign, mag)
  с семантикой floor для `/` и `%` (эталон считает Python'ом);
  переполнение 2^32 — штатная ошибка;
- **циклы типов** (struct/enum, содержащий себя по значению) —
  итеративный DFS с трёхцветной раскраской;
- **правила 4–5**: statements считаются по пулу узлов (подсчёт видов
  инструкций в поддереве функции), контракты — по наличию обёрток
  `requires`/`ensures`; наличие и сигнатура `main`.

Сверяемый артефакт — дамп сигнатур, канон — `eatc sig`
(`src/eatc/sigdump.py`): константы со значениями, сигнатуры функций
и методов с разрешёнными типами, поля, варианты с нагрузкой, строка
`stats funcs= structs= stmts=`. Сверка в `make verify_selfhost`:
каждый `.eat`, проходящий `eatc sig` (одиночные модули без `main`
пропускаются), плюс конкатенация собственных шести модулей —
чекер проверяет сам себя.

## Фаза 3b: тайпчекер тел (готово)

Вторая половина `selfhost/Check.eat` — полный семантический анализ:

- **машина выражений**: итеративный обход с явным стеком кадров;
  ожидаемый тип течёт вниз (литералы берут тип контекста, как у
  эталона), выведенный — вверх (`tres`); каждый узел-выражение
  получает аннотацию типа в `nty`;
- **машина инструкций**: кадры блоков/if/for/match/loop; области
  видимости (shadowing — правило 6, неиспользуемые переменные —
  правило 10), недостижимый код после return/break, «все пути
  возвращают значение» — снизу вверх по флагам `nrt`;
- **match**: домены Result/Option/enum (включая встроенные IoError и
  ParseError), привязка нагрузки, повторы и исчерпываемость;
- **правило 1**: рёбра графа вызовов собираются при типизации
  вызовов, DAG проверяется итеративным трёхцветным DFS, глубина
  стека — длина максимальной цепочки от `main`;
- получатель `var self`-метода обязан быть изменяемым lvalue;
  `loop` — только в `main`; `break` — только в `loop`; discard
  и «результат нельзя отбросить» — правило 7.

Сверяемый артефакт — типизированный дамп, канон — `eatc typed`
(`src/eatc/typeddump.py`): дамп AST фазы 2, где каждое выражение
несёт суффикс `:: тип`, let/for/arm — типы переменной/элемента/
нагрузки (у диапазонов — `bounds=a..b`), плюс футер `stack N` и
сортированные рёбра `edge caller -> callee`. Сверка в
`make verify_selfhost` на каждом типизируемом `.eat`; главный тест —
**тайпчекер типизирует сам себя**: дамп конкатенации семи модулей
(≈29 000 строк) совпадает с эталоном байт-в-байт, включая граф
вызовов и глубину стека.

### Следствия для бутстрапа

Фаза 2 выявила и починила два предела бутстрап-кодогена:

- копирование агрегатов шло `load`/`store` целого массива — SSA-узел
  на все элементы; на пуле `[u8; 65536]` LLVM SelectionDAG падал
  (NumValues — 16 бит). Теперь агрегаты копируются `llvm.memcpy`;
- кадр `main` с пулами компилятора ≈ 7 МБ упирался в умолчание ОС
  (8 МБ): бинарники линкуются с увеличенным стеком
  (`-Wl,-stack_size`; 64 МБ, с фазы 5 — 128 МБ) — риск из
  §«остаточные риски» фазы 1 закрыт.

## Фаза 4: эмиссия текстового LLVM IR (готово)

`selfhost/Ir.eat` — кодоген поверх таблиц и аннотаций тайпчекера,
печатающий текстовый модуль LLVM **байт-в-байт** как эталон: новая
команда `eatc ir` — тот же `src/eatc/codegen.py`, но без верификатора
(все проверки остаются в рантайме), имя модуля/файла в trap-сообщениях
канонизировано в `stdin`, triple пуст — вывод не зависит от платформы
и пути файла, а `clang` собирает такой `.ll` под хост.

Чтобы дамп совпадал побайтово, в EATLang воспроизведён принтер
llvmlite:

- **модель имён**: аргументы функции — `%".1"..%".n"`, возвращаемое
  значение съедает ещё один номер, дальше **каждая** инструкция
  (включая безрезультатные `store`/`br`) инкрементирует счётчик;
  именованные `alloca` и блоки дедуплицируются по базовому имени
  (`str`, `str.1`, ...) — хранятся только базовые имена: суффиксованные
  кандидаты доказуемо ни с чем не сталкиваются;
- **буферизация функции**: блоки печатаются в порядке создания, а
  заполняются вразнобой (тело `exit` генерируется последним, печатается
  вторым) — строки-инструкции копятся в связные списки по блокам;
  `switch` печатается при сборке блока, когда имена всех арм-блоков
  уже известны;
- **строки типов** — инлайн, как у llvmlite (struct раскрывается в
  `{i32, [256 x i8]}`-литералы) — итеративный принтер с мемоизацией
  по интернированному пулу типов;
- глобалы `@"str.N"` (сегменты строк-литералов и trap-сообщения
  `stdin:СТРОКА:КОЛОНКА: error: trap: ...`) дедуплицируются по
  содержимому (хэш + байты), хвост модуля — глобалы и `declare`
  интринзиков в порядке первого появления;
- пулы больше 64 КБ (текст функции — 4 МБ, содержимое глобалов —
  2 МБ) выражены вложенными массивами `[[u8; 65536]; N]` — предел
  элементов массива соблюдён;
- причуды llvmlite воспроизведены: двойной пробел после `phi` и
  `select` (пустые флаги + `rstrip`), `getelementptr` без
  `inbounds` в sizeof-трюке `gep(null, 1)`, `xor i1 %x, -1` для
  `not`, экранирование `c"..."`-констант (`\xx` для всего вне
  печатаемого ASCII, кавычки и бэкслэша).

Сверка в `make verify_selfhost`:

1. на каждом `.eat`, проходящем `eatc ir`, дифф с `./build/SelfIr < f`;
2. **самоприменение**: IR конкатенации семи модулей фронтенда
   (лексер+парсер+тайпчекер, ≈51 000 строк) совпадает байт-в-байт;
3. **end-to-end**: `.ll`, который эмитит SelfIr для всего фронтенда,
   собирается `clang` с `runtime.c` — полученный бинарник-тайпчекер
   выдаёт дамп, равный `eatc typed` (self-hosted компилятор породил
   работающий self-hosted тайпчекер);
4. интерпретатор == бинарник на SelfIr.

### Следствие для бутстрапа: alloca поднимаются в entry-блок

Дифф-прогон на `selfhost/Ir.eat` уронил `build/SelfLex` (Bus error):
бутстрап-кодоген эмитил `alloca` по месту использования — в том числе
в теле циклов, где LLVM выделяет стек **на каждой итерации**; цикл
печати токенов на 32 000 токенов съедал 64 МБ стека. Теперь каждая
функция начинается с entry-блока, который держит только `alloca` и
`br` на блок `body` (как в clang) — кадр функции статичен независимо
от числа итераций. Self-hosted эмиттер зеркалит это: инструкции
`alloca` дописываются в цепочку строк блока 0.

## Фаза 5: bootstrap (готово)

Самоприменение к самому себе: конкатенация всех восьми модулей
(`Tok Lexer Ast Parser Check Ir IrMain`, ≈348 КБ, 81 022 токена,
≈47 000 узлов AST, 498 функций, 1 789 рёбер графа вызовов) проходит
через self-hosted компилятор целиком. Цикл замыкает
`make verify_bootstrap`:

1. **stage1**: `build/SelfIr` (собран Python-бутстрапом) эмитит IR
   собственных восьми модулей — дифф с `eatc ir` байт-в-байт
   (26,85 МБ, 84 000 строк);
2. **stage2**: `clang` собирает из этого IR новый бинарник — и тот
   эмитит для тех же исходников **тот же IR байт-в-байт** (фикспойнт:
   компилятор, собранный самим собой, эквивалентен собранному
   бутстрапом).

Что потребовалось для роста:

- **предел языка**: «токенов в файле» поднят 65 536 → 131 072
  (SPEC.md §6, `limits.py`, self-hosted лексер — синхронно);
- **пулы лексера** — вложенные массивы (предел элементов массива
  65 536 остался): семь массивов токенов и пул байтов значений стали
  `[[_; 65536]; 2]`, элемент i — `p[i / 65536][i % 65536]`; доступ
  снаружи — методы `tok_*`/`sb_at`/`sb_set` (все — `var self`:
  получатель по указателю, без копии агрегата);
- **пул узлов парсера** 32 768 → 65 536 (плоский), бюджеты насосов
  удвоены (разбор 262 144 шага, дампы 131 072);
- **таблицы тайпчекера**: функции 512 → 1 024 (кодировка ключей
  тестов сдвинута: тест — 1 024 + ti, ребро — 2 048 + ti), рёбра
  графа вызовов 1 024 → 4 096, for-диапазоны 128 → 512, аннотации
  `nty`/`nrt` — 65 536;
- **обход полей struct-литерала** — граница 128 → 512 (у `Ir`
  131 поле — «в литерале не хватает полей» ловилось на собственном
  `IrMain`);
- **стек 128 МБ** (`-Wl,-stack_size,0x8000000`): кадр `main`
  self-hosted компилятора с выросшими пулами — ≈85 МБ.

## Фаза 6: рантайм на EATLang (готово)

Логика рантайма переписана с C на EATLang — `selfhost/Rt.eat`,
**первый модуль каждой программы** (`eatc build selfhost/Rt.eat App.eat`;
для self-host — `cat selfhost/Rt.eat App.eat | ./build/SelfIr`).
В `runtime.c` остался шим из трёх аксиом ОС, невыразимых в себе:
`eat_read_byte`, `eat_write_byte`, `eat_trap` (граница «язык ↔ ОС»,
как Pascal/web2c-прослойка у TeX).

- **новый builtin `write_byte(b: u8)`** (SPEC §7) — примитив вывода,
  симметричный `read_byte()`: через него `Rt.eat` выражает
  `print`/`write`;
- **struct RtStr { ln, buf }** повторяет layout встроенной строки
  `{i32, [256 x i8]}` — кодоген передаёт указатели на `str<256>`
  прямо в методы: `print`/`write`, сегменты интерполяции
  (`append_str/char/bool/i32/u32`), `==` (`eq`), `read_line`,
  `parse_i32` (пара `parse_status`/`parse_value` — конструкторов
  `Ok`/`Err` в языке нет, Result собирает кодоген из статуса
  и значения, как раньше из C-статуса);
- **оба кодогена** лаверят builtins в `@"eat_RtStr__*"`; литеральные
  сегменты строк — глобалы в layout'е `str<256>`
  (`{i32 len, [256 x i8]}` с дополнением нулями), trap-сообщения —
  по-прежнему сырые `c"..."` для шима; в `declare` остались три
  аксиомы;
- внутри методов вывода и разбора нет ни `print`, ни интерполяции —
  они сами реализуют эти механизмы (иначе рекурсия, невидимая
  правилу 1); краевые случаи (−2147483648, переполнения `parse_i32`,
  границы ёмкости) — `test`-блоки в самом `Rt.eat`, исполняются при
  каждой сборке;
- рантайм теперь проходит все проверки языка: контракты, верификатор,
  правила Power of 10 — раньше это был единственный код бинарника
  вне досягаемости компилятора; заодно интерпретатор научился трапать
  переполнение ёмкости строки при интерполяции (расхождение
  с нативной семантикой, скрытое с фазы 1);
- сверка та же: `make verify` (interp == native на примерах,
  включая `read_line`/`parse_i32`), `verify_selfhost` (IR байт-в-байт
  на каждом файле), `verify_bootstrap` — **фикспойнт девяти модулей**:
  компилятор компилирует собственный рантайм внутри себя (27,4 МБ IR).

Попутно закрыт хвост фазы 5: четыре скана таблицы функций и два
контракта в `Check.eat` остались с границей 512 — при 524 функциях
(с `Rt.eat`) self-хост не находил `main`; ловилось дифф-прогоном
и штатными trap'ами контрактов.

## Ограничения фазы 5 (осознанные)

- `eatc ir`/SelfIr не запускают верификатор: все проверки (границы,
  переполнение, контракты) эмитятся в рантайм — бинарник корректный,
  но медленнее собранного `eatc build`;
- один `.ll`-файл — один модуль со `stdin`-атрибуцией трапов:
  позиции ошибок указывают на строки конкатенации, а не исходных
  файлов;
- stage2 доказуемо равен stage1 только на текущих исходниках
  (фикспойнт по конструкции); на прочих входах равенство следует из
  побайтового равенства IR обоих бинарников.

## Ограничения фаз 1–2 (осознанные)

- лексема (значение одного токена) ≤ 256 байт — предел `str<256>`
  рантайма; превышение — штатная ошибка «лексема длиннее 256 байт»
  (модель TeX: фиксированные пулы);
- целочисленный литерал ≤ 2^32−1 (эталон держит произвольную
  точность до тайпчекера; вне диапазона у self-host — штатная
  ошибка);
- пределы пулов: 131 072 токена, 65 536 узлов AST, 131 072 байта
  значений, 2 048 сегментов интерполяции — превышение даёт штатную
  ошибку `err:`, не UB;
- при ошибке разбор останавливается на первой (как эталон);
  тексты ошибок совпадают с эталоном по смыслу, но не сверяются
  (эталон пишет их в stderr).
