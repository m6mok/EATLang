# Self-hosting: компилятор EATLang на EATLang

Цель — компилятор EATLang, написанный на EATLang и собирающий сам себя.
Python-бутстрап (`src/eatc/`) остаётся эталоном: каждая фаза self-hosted
компилятора сверяется с ним дифференциально, байт-в-байт.

Архитектура self-hosted компилятора — фильтр `stdin → stdout`
(файловых дескрипторов в языке нет): модули программы конкатенируются
в stdin, результат печатается в stdout.

## Дорожная карта

| Фаза | Модули | Статус |
| --- | --- | --- |
| 1. Лексер | `selfhost/Tok.eat`, `selfhost/Lexer.eat`, `selfhost/Main.eat` | ✅ |
| 2. Парсер (пул AST, явный стек вместо рекурсии) | — | планируется |
| 3. Структурные проверки + тайпчекер | — | — |
| 4. Эмиссия текстового LLVM IR | — | — |
| 5. Bootstrap: self-hosted компилятор собирает себя | — | — |

## Фаза 1: лексер (готово)

Стриминговый конечный автомат: `main` читает байты через
`read_byte()` в единственном `loop`, каждый байт проходит через
`Lexer.step(b)` (15 состояний, включая двухсимвольные операторы
`-> .. == != <= >=`, строки с escape'ами, char-литералы,
комментарии), `Lexer.finish()` добивает незавершённый токен,
финальный NEWLINE и EOF.

Свойства, совпадающие с эталонным лексером `src/eatc/lexer.py`:

- NEWLINE — значимый токен; подавляется внутри `( )` и `[ ]`,
  в начале файла и после другого NEWLINE;
- колонки считаются в символах: continuation-байты UTF-8
  (`0x80..0xBF`) колонку не двигают — позиции токенов после
  кириллических строк совпадают с эталоном;
- байты ≥ 128 — буквы идентификаторов (эталон принимает юникодные
  буквы через `str.isalpha`);
- escape-последовательности строк и char-литералов декодируются
  лексером (`\n \t \\ \" \' \0`);
- ошибки — те же тексты и позиции, что у эталона (в формате
  `err: СТРОКА:КОЛОНКА: сообщение`; в дифф не входят — эталон
  печатает ошибки в stderr).

### Дифференциальная сверка

Формат дампа — построчно `СТРОКА:КОЛОНКА ВИД [значение]`; значение
печатается у INT, STRING, CHAR, IDENT с экранированием
`\\ \n \t \r \0`. Эталонный дамп — команда `eatc lex <файл>`.

```
make verify_selfhost
```

1. собирает `build/SelfLex` (LLVM);
2. на **каждом** `.eat` репозитория (`examples/`, `selfhost/`,
   `tests/`, включая стресс-кейсы `tests/lex/`) сверяет
   `eatc lex f` с `./build/SelfLex < f`;
3. сверяет интерпретатор с бинарником на исходниках самого лексера
   (лексер лексирует себя).

Стресс-кейсы `tests/lex/`: пустой файл, файл без завершающего
перевода строки, незавершённые двухсимвольные операторы на EOF,
кириллические идентификаторы, `{{`-экранирование, многострочные
скобки.

### Следствие для бутстрапа: интерпретатор — байтовая машина

Дифф-сверка выявила расхождение бутстрапа: интерпретатор хранил
строки как юникодные `str` Python — `write("{char(b)}")` для
`b >= 128` печатал 2 байта UTF-8 вместо одного, а `len` считал
символы, а не байты (нативный рантайм — байты). Исправлено:
строковые значения интерпретатора — latin-1 (1 символ == 1 байт),
вывод — через `sys.stdout.buffer`. Теперь `print`/`write`/`len`/
индексация байт-точны и совпадают с нативным рантаймом.

## Ограничения фазы 1 (осознанные)

- лексема (значение одного токена) ≤ 256 байт — предел `str<256>`
  рантайма; превышение — штатная ошибка «лексема длиннее 256 байт»
  (модель TeX: фиксированные пулы);
- токены печатаются, а не копятся в пуле — пул токенов появится
  в фазе 2 вместе с парсером (изменение локально: `emit_*` вместо
  `write` будут писать в пул);
- при ошибке лексер останавливается на первой (как эталон).
