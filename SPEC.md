# Спецификация EATLang v0.0.1 (черновик)

## 1. Цель

EATLang — учебно-исследовательский язык программирования: компилируемый (через LLVM),
статически типизируемый, **тотальный** (каждая программа доказуемо завершается).

**Исследовательский вопрос:** каков минимальный набор конструкций языка, при котором
нарушить [10 правил NASA (Power of 10)](https://en.wikipedia.org/wiki/The_Power_of_10:_Rules_for_Developing_Safety-Critical_Code)
*синтаксически невозможно*, а писать на языке всё ещё удобно?

Долгосрочная цель — **self-hosting**: компилятор EATLang, написанный на EATLang.
Компиляция — тотальная функция `текст → текст`, поэтому она выразима в языке
(в отличие от self-интерпретатора, который запрещён диагонализацией).

## 2. Правила NASA → механизмы языка

| № | Правило NASA | Механизм EATLang | Когда проверяется |
| --- | --- | --- | --- |
| 1 | Простой control flow, без goto и рекурсии | `goto` нет в грамматике. Граф вызовов обязан быть DAG | компиляция |
| 2 | Все циклы с фиксированной границей | `while` нет в грамматике. Только `for` со статически известной границей; один корневой `loop` в `main` | компиляция |
| 3 | Нет динамической аллокации | Кучи нет. Размер массива и ёмкость строки — часть типа | по построению |
| 4 | Функция ≤ 60 строк | Функция > 60 statements — ошибка компиляции | компиляция |
| 5 | ≥ 2 assertion на функцию | Обязательные контракты: у функции с параметрами — `requires`, с возвращаемым значением — `ensures`. Отказ — только явный (`requires true`) | компиляция + рантайм (trap) |
| 6 | Минимальная область видимости | Мутабельных глобалов нет. Только `const` на верхнем уровне. Shadowing запрещён | компиляция |
| 7 | Проверять все возвращаемые значения | Результат вызова нельзя отбросить молча — только явный `discard`. Ошибки — `Result`, `match` исчерпывающий | компиляция |
| 8 | Ограничить препроцессор | Препроцессора нет | по построению |
| 9 | Ограничить указатели | Указателей нет. Передача параметров by-value; указателей на функции нет → граф вызовов статичен | по построению |
| 10 | Все warnings — errors | Класса warning не существует: неиспользуемая переменная, недостижимый код — ошибки компиляции | компиляция |

Следствие правил 1+2+3+9: язык **не Turing-полный**. Для каждой программы компилятор
вычисляет верхнюю границу памяти и глубины стека и печатает их как артефакт сборки.

## 3. Система типов

Замкнутое множество типов, без кучи и указателей:

| Тип | Описание |
| --- | --- |
| `i32`, `u32`, `u16`, `u8`, `u64`, `i64` | целые; переполнение — **trap** (аварийная остановка), не wrap |
| `bool` | `true` / `false` |
| `char` | один байт-символ |
| `[T; N]` | массив фиксированного размера, `N` — константа времени компиляции |
| `str<N>` | ограниченная строка: ёмкость `N` байт, текущая длина ≤ `N` |
| `struct` | пользовательская запись, может содержать методы |
| `enum` | перечисление; вариант несёт 0 или 1 значение (`Num(i32)`; несколько значений — struct) |
| `Result<T, E>`, `Option<T>` | **встроенные** sum-типы; пользовательских дженериков в 0.0.1 нет |

- Деление и остаток: делитель равный нулю — trap (позже — статическое доказательство через контракты).
- Индексирование массива/строки: выход за границы — trap (позже — статическое доказательство).
- Чисел с плавающей точкой в 0.0.1 **нет** (осознанно: MISRA/полётные стандарты ограничивают float).

## 4. Грамматика (EBNF)

Инструкция заканчивается концом строки. Точек с запятой нет. Одна инструкция — одна строка.
Комментарии: `#` до конца строки.

```ebnf
program        = { top_decl } ;
top_decl       = func_decl | extern_decl | struct_decl | enum_decl | test_block | const_decl
               | import_block | export_block | module_mark ;

(* Модули (docs/MODULES_PLAN.md §2–4). В именованном модуле порядок
   блоков строгий: export? import* decl*; вне именованных модулей
   (склейка cat, отдельный файл) позиции не проверяются, export-блоки
   инертны. module_mark — строка-директива драйвера `#module "путь"`:
   лексер выдаёт токен MODULE и начинает счёт координат заново —
   атрибуция ошибок и trap-сообщений пофайловая. Декларации до первой
   директивы — неявный нулевой модуль (Rt.eat), его имена видимы всем.
   import обязан ссылаться на модуль раньше по потоку; поток строит
   драйвер: DAG импортов, циклы — ошибка, топологическая сортировка
   с лексикографическим tie-break. Импортированное имя канонизируется
   во внутреннее имя экспортёра (`as` — переименование с обеих сторон). *)
import_block   = "import" "{" bind { "," bind } [ "," ] "}" "from" STRING ;
export_block   = "export" "{" bind { "," bind } [ "," ] "}" ;
bind           = IDENT [ "as" IDENT ] ;

const_decl     = "const" IDENT ":" type "=" const_expr ;

extern_decl    = "extern" "func" IDENT "(" [ params ] ")" [ "->" type ]
                 [ "requires" expr ] [ "ensures" expr ] ;
                 (* без тела: C-реализация линкуется по имени, SPEC §7 *)

struct_decl    = "struct" IDENT "{" { field_decl | func_decl } "}" ;
field_decl     = IDENT ":" type ;

enum_decl      = "enum" IDENT "{" variant { "," variant } "}" ;
variant        = IDENT [ "(" type ")" ] ;

func_decl      = "func" IDENT "(" [ params ] ")" [ "->" type ]
                 (* первый параметр метода: self | var self *)
                 [ "requires" expr ]
                 [ "ensures" expr ]      (* в ensures доступно имя result *)
                 block ;
params         = param { "," param } ;
param          = "self" | IDENT ":" type ;   (* self — только у методов struct *)

test_block     = "test" IDENT block ;

block          = "{" { stmt } "}" ;
stmt           = let_stmt | var_stmt | assign_stmt | if_stmt | for_stmt
               | loop_stmt | match_stmt | return_stmt | break_stmt
               | assert_stmt | call_stmt | discard_stmt ;

let_stmt       = "let" IDENT ":" type "=" expr ;          (* иммутабельная *)
var_stmt       = "var" IDENT ":" type "=" expr ;          (* мутабельная *)
assign_stmt    = lvalue "=" expr ;
lvalue         = IDENT { "." IDENT | "[" expr "]" } ;

if_stmt        = "if" expr block { "elif" expr block } [ "else" block ] ;

for_stmt       = "for" ( IDENT | "_" ) "in" iterable block ;
iterable       = const_expr ".." const_expr               (* полуинтервал [a, b) *)
               | expr ;                                   (* массив или str: по элементам *)

loop_stmt      = "loop" block ;    (* только в теле main, не более одного *)

match_stmt     = "match" expr "{" { match_arm } "}" ;
match_arm      = pattern block ;
pattern        = IDENT [ "(" ( IDENT | "_" ) ")" ] ;      (* Ok(x), Err(_), None, вариант enum *)

return_stmt    = "return" [ expr ] ;
break_stmt     = "break" ;                     (* в цикле, не более одного на цикл *)
assert_stmt    = "assert" expr ;
call_stmt      = call_expr ;                              (* только если функция ничего не возвращает *)
discard_stmt   = "discard" call_expr ;                    (* осознанный сброс результата *)

type           = "i32" | "u32" | "u8" | "bool" | "char"
               | "[" type ";" const_expr "]"
               | "str" "<" const_expr ">"
               | "Result" "<" type "," type ">"
               | "Option" "<" type ">"
               | IDENT ;                                  (* struct или enum *)

expr           = or_expr ;
or_expr        = and_expr { "or" and_expr } ;
and_expr       = not_expr { "and" not_expr } ;
not_expr       = [ "not" ] cmp_expr ;
cmp_expr       = bitor_expr [ ( "==" | "!=" | "<" | "<=" | ">" | ">=" ) bitor_expr ] ;
bitor_expr     = bitxor_expr { "|" bitxor_expr } ;
bitxor_expr    = bitand_expr { "^" bitand_expr } ;
bitand_expr    = shift_expr { "&" shift_expr } ;
shift_expr     = add_expr { ( "<<" | ">>" ) add_expr } ;
add_expr       = mul_expr { ( "+" | "-" ) mul_expr } ;
mul_expr       = unary_expr { ( "*" | "/" | "%" ) unary_expr } ;
unary_expr     = [ "-" | "~" ] unary_expr | postfix_expr ;
postfix_expr   = primary { "." IDENT [ "(" [ args ] ")" ] | "[" expr "]" } ;
primary        = INT | CHAR_LIT | STRING | "true" | "false"
               | IDENT | call_expr | struct_lit | array_lit
               | enum_lit | "(" expr ")" ;
call_expr      = IDENT "(" [ args ] ")" ;
args           = expr { "," expr } ;
struct_lit     = IDENT "{" IDENT ":" expr { "," IDENT ":" expr } "}" ;
array_lit      = "[" expr { "," expr } "]"      (* длина обязана совпадать с типом *)
               | "[" expr ";" const_expr "]" ;  (* заполнение: [0; 4096] — пулы *)
enum_lit       = IDENT "." IDENT [ "(" expr ")" ] ;
                 (* Sign.Positive | Node.Num(42) — конструктор нагрузки *)
```

Строковые литералы поддерживают интерполяцию: `"x = {expr}"` — выражение внутри `{}`
типизируется на этапе компиляции. Экранирование фигурной скобки: `{{`.

Целые литералы — десятичные или шестнадцатеричные (`0xEDB88320`,
регистр hex-цифр любой, префикс — строго `0x`). Hex-литерал ограничен
16 цифрами (всегда представим в u64); лексер нормализует его в
десятичное значение — далее по конвейеру hex не существует.
Предел десятичного литерала — 2^64−1 (модуль; `-9223372036854775808`
выразим: минус сворачивается в литерал).

Битовое NOT `~x` — унарный оператор (приоритет унарного минуса),
только для беззнаковых типов, как и битовые бинарные; результат —
тип операнда: `~x == x ^ 0xFFFFFFFF` для u32, `x ^ 0xFF` для u8.
Из типа не выходит — trap невозможен; верификатор считает интервал
точно (`~x = маска - x`).

Неявных преобразований типов нет. Явные — вызовы вида `i32(x)`, `u32(x)`,
`u16(x)`, `u8(x)`, `u64(x)`, `i64(x)`; сужающее преобразование вне
диапазона — trap (или доказывается контрактом вызывающего).
Знаковые (`i32`, `i64`) не участвуют в битовых операциях; унарный
минус применим к обоим.

`char` — ровно один байт (литерал вне ASCII отклоняется лексером).
Код символа — `u8(c)`, символ по коду — `char(b: u8)`; оба каста
тотальны (trap невозможен) и прозрачны для верификатора: факты о
`u8(c)` — это факты о `c`. Расширение кода дальше — явно: `u32(u8(c))`.

Битовые операторы `& | ^ << >>` определены **только для беззнаковых**
(`u32`, `u8`; у `i32` смысл зависел бы от представления знака) и требуют
одинаковых типов операндов. Приоритет — слабее арифметики и сильнее
сравнений (как в Rust): `x & 128 != 0` читается `(x & 128) != 0`,
`1 << i + 1` — `1 << (i + 1)`. `& | ^` тотальны. Сдвиг на ширину типа
и больше — trap; `x << n` эквивалентен умножению на 2^n (trap при
переполнении), `x >> n` — делению (тотален при допустимом n).
В константных выражениях типов (`[T; N]`, `str<N>`) битовых операторов
нет — как и сравнений. `>>` на месте двух закрывающих `>` вложенных
дженериков (`Result<u32, str<8>>`) парсер расщепляет на два `>`.

## 5. Семантика

### 5.1 Тотальность

- `for a..b`: границы — константы времени компиляции (литералы, `const`, размеры типов).
  Число итераций ограничено сверху границей, известной до запуска.
- `for x in arr`: число итераций = размер массива из типа.
- `break` допустим в любом цикле — это ранний выход, статическая граница
  остаётся (правило 2 не нарушается). Не более **одного** `break` на цикл
  (стиль MISRA: единственная точка раннего выхода, правило 1); `break`
  привязан к внутреннему объемлющему циклу.
- `loop` — единственная бесконечная конструкция; разрешён только непосредственно в теле
  `main`, не более одного. Это модель реактивной программы:
  бесконечный главный цикл, каждая итерация которого тотальна; `break` —
  единственный способ его завершить.
- Рекурсия (прямая и косвенная) запрещена: компилятор строит граф вызовов и требует DAG.

### 5.2 Контракты (правило 5)

- Функция **с параметрами** обязана иметь `requires`.
- Функция **с возвращаемым значением** обязана иметь `ensures` (доступно имя `result`).
- Осознанный отказ — явная запись `requires true` / `ensures true`: он виден на ревью.
- Этап 1: контракты компилируются в проверки на входе/выходе функции, нарушение — trap.
- Этап 2 (реализован частично): интервальный анализ целых доказывает контракты,
  переполнения, деление на ноль и границы; доказанные проверки удаляются из
  бинарника. `requires` снимается, только если доказаны **все** вызовы функции.
  Анализ консервативен: недоказанное остаётся runtime-trap'ом.
- `assert` в теле — дополнительная точка проверки, та же семантика (trap при нарушении).

### 5.3 Ошибки (правило 7)

- Исключений нет. Ошибающиеся операции возвращают `Result<T, E>`.
- `match` обязан быть исчерпывающим — пропущенный вариант это ошибка компиляции.
- enum с полезной нагрузкой не сравнивается `==`/`!=` — только `match`.
- Значимый результат вызова обязан быть использован: присвоен, передан, возвращён,
  разобран `match` — либо явно отброшен `discard`.
- **Конструкторы** `Ok(x)`/`Err(e)`/`Some(x)`/`None` создают значения
  `Result`/`Option` (решение 2026-07-14). Они **контекстные**: допустимы только
  там, где ожидаемый тип известен точно — в выражении `return` (по типу возврата
  функции), в `let` с аннотацией `Result`/`Option` и в payload вложенного
  конструктора (`Ok(Some(x))`). Общего вывода типов нет; вне контекста — ошибка
  компиляции. Имена `Ok`, `Err`, `Some`, `None` зарезервированы. Новых продукций
  в грамматике нет: `Ok(x)` — синтаксис вызова, `None` — идентификатор.

### 5.4 Область видимости (правило 6)

- Верхний уровень: только `const`, `func`, `struct`, `enum`, `test`. Мутабельного
  глобального состояния не существует.
- Модули: программа — упорядоченный список файлов
  (`eatc run/build a.eat b.eat main.eat`, последний — главный) с единым
  глобальным пространством имён; повтор имени между файлами — ошибка.
  Директив импорта нет: зависимость выражается порядком файлов, а для
  self-host компилятора — конкатенацией в stdin.
- `let` — по умолчанию; `var` — осознанная мутабельность.
- `self` неизменяем; мутирующий метод объявляется `func f(var self, ...)`
  и требует изменяемого получателя (`var`-переменной). Остальные
  параметры всегда by-value — других способов разделить состояние нет,
  поэтому алиасы невозможны.
- Копия параметра — деталь реализации и элидируется: не-`var`
  параметры и `self` передаются по ссылке (вызываемый их не пишет —
  единственный писатель в чужую память это `var self`). Если аргумент
  метода с `var self` может перекрываться с получателем — общий корень
  lvalue-пути, как в `s.append_str(s)`, — копию аргумента делает
  вызывающий. Наблюдаемое поведение неотличимо от by-value.
- Shadowing (повторное объявление имени во вложенной области) запрещён.
- Неиспользуемая переменная / параметр — ошибка компиляции (заглушка: `_`).

### 5.5 Тесты

`test`-блоки — часть языка. Компилятор собирает и исполняет их при сборке;
падение `assert` в тесте = сборка не удалась. Тесты не попадают в итоговый бинарник.

## 6. Пределы (capacity limits)

Константы компилятора; их превышение — штатная ошибка `capacity exceeded`
(модель TeX: фиксированные пулы вместо кучи). Эти же пределы делают возможным
self-hosting компилятор без динамической памяти.

| Предел | Значение 0.0.1 |
| --- | --- |
| statements в функции | 60 |
| параметров функции | 6 |
| глубина вложенности блоков | 8 |
| глубина выражения (AST) | 32 |
| токенов в файле | 131 072 |
| узлов AST в файле | 131 072 |
| функций в программе | 1 024 |
| ёмкость `str<N>` | N ≤ 4 096 |
| элементов массива | ≤ 65 536 |
| именованных модулей в потоке | 64 |
| импортированных имён на программу | 512 |
| длина канонического пути модуля | 128 байт |

## 7. Встроенные функции

| Сигнатура | Описание |
| --- | --- |
| `print(s: str)` | вывод строки (с интерполяцией) в stdout, с переводом строки |
| `write(s: str)` | вывод строки в stdout без перевода строки |
| `read_line() -> Result<str<256>, IoError>` | чтение строки из stdin |
| `read_byte() -> Result<u8, IoError>` | байт из stdin; `Err(Eof)` в конце потока |
| `write_byte(b: u8)` | один байт в stdout — примитив, через который выражается вывод |
| `write_span(a: [u8; N], off: u32, len: u32)` | байты `a[off..off+len)` в stdout одним вызовом ОС (батч-вывод); `off + len > N` — trap |
| `write_err_byte(b: u8)` | один байт в stderr — канал диагностики, не смешивается с выводом фильтра |
| `exit(code: u32)` | штатное завершение процесса с кодом; допустим **только в `main`** и **не более одного** вызова на программу (по аналогии с `loop`); код после `exit` в том же блоке недостижим (правило 10) |
| `parse_i32(s: str) -> Result<i32, ParseError>` | разбор целого |
| `len(x) -> u32` | текущая длина строки / размер массива |

Встроенные enum: `IoError { Eof, Fail }`, `ParseError { Empty, BadChar, Overflow }`.

Аксиомы языка — `read_byte`, `write_byte`, `write_span`,
`write_err_byte`, `exit`
и trap (примитивы ОС, шим `runtime.c` из семи функций: у trap две
формы). Остальные
встроенные — `print`, `write`,
интерполяция, `==` строк, `read_line`, `parse_i32` — реализованы на
EATLang в рантайм-модуле `selfhost/Rt.eat`, который идёт первым модулем
каждой собираемой программы (`eatc build selfhost/Rt.eat App.eat`).

**extern-функции** — расширяемые «аксиомы» проекта (решение
2026-07-13, [docs/MCU_PLAN.md](docs/MCU_PLAN.md) §1): `extern func`
объявляет C-функцию без тела; символ — имя как написано (без
`eat_`-манглинга). Граница доверия та же, что у аксиом: тотальность
и `ensures` extern-функции принимаются на веру, `requires`
проверяется у вызывающего (верификатор/рантайм как обычно).
На границе разрешены скаляры (`i32/u32/u16/u8/u64/i64/bool/char`) в
аргументах и возврате и массивы `[u8|u16|u32|u64; N]` в аргументах
(передаются указателем, для C — read-only; C не должен ни писать
в них, ни сохранять указатель). Интерпретатор extern не исполняет
(нет линковки с C) — вызов даёт ошибку; сверка extern-программ —
бинарником (для МК — QEMU, mcu/). main не может быть extern.

Режим trap-кодов (`--trap-codes` у `eatc ir`/`build`; self-hosted —
`SelfIrCodes`, модуль `IrCodesMain.eat`): вместо уникальной
строки-сообщения каждая точка проверки вызывает `eat_trap_code(u32)`
с числовым кодом («trap N» в stderr), а таблица
`; trap <код>: <сообщение>` уходит комментариями в хвост `.ll` — во
флеш попадает только число (метрика МК). Канонический режим по
умолчанию не меняется; паритет режима — `make verify_trapcodes`.

## 8. Артефакты компиляции

Компилятор выдаёт вместе с бинарником отчёт:

- суммарная статическая память программы (байт);
- максимальная глубина стека (граф вызовов — DAG, поэтому вычислима точно);
- на функцию: statements, глубина вложенности, статус контрактов.

## 9. Архитектура компилятора

```text
.eat → лексер → парсер (рекурсивный спуск, глубина ограничена)
     → типизированный AST
     → семантические проверки: DAG вызовов, границы циклов, длина функций,
       использование результатов, scope-правила, контракты
     → llvmlite → LLVM IR → clang → бинарник + отчёт
```

Bootstrap-компилятор — на Python (`src/eatc/`). После стабилизации языка компилятор
переписывается на EATLang и собирает сам себя; Python-версия остаётся эталоном.

## 10. Открытые вопросы

- Битовые операции (`& | ^ << >>`) — понадобятся для self-hosting; отложены до запроса.
- Операторный сахар для методов struct (`+` → `add`) — после стабилизации ядра.
- Модульная система (несколько файлов) — в 0.0.1 программа = один файл.
- `isqrt` и другая целочисленная математика — библиотечно, с контрактами.
