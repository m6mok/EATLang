# EATLang

Учебно-исследовательский язык программирования. Текущая версия: **0.0.1**
(черновик спецификации + эталонные примеры).

## Описание

- **Компилируемый** — через LLVM (llvmlite → LLVM IR → clang)

- **Статически типизируемый** — без неявных преобразований

- **Тотальный** — каждая программа доказуемо завершается; память и глубина стека
  известны на этапе компиляции

- **10 правил NASA (Power of 10) зашиты в сам язык** — нарушить их синтаксически
  невозможно: нет `while` без границы, нет рекурсии, нет кучи, нет указателей,
  контракты `requires`/`ensures` обязательны, результат вызова нельзя отбросить молча

Документация:

- [Руководство по языку](docs/GUIDE.md) — учебник: синтаксис, контракты,
  верификация, каталог «выстрелов в ногу», которые не скомпилируются
- [SPEC.md](SPEC.md) — формальная спецификация: грамматика EBNF,
  семантика, пределы
- [SELFHOST.md](docs/plans/SELFHOST.md) — self-hosting: компилятор EATLang
  на EATLang, дорожная карта и дифференциальная сверка с эталоном

**Исследовательский вопрос:** каков минимальный набор конструкций языка, при котором
нарушить Power of 10 невозможно, а писать на языке всё ещё удобно?
Долгосрочная цель — self-hosting компилятор.

## Как выглядит язык

```text
func max(a: i32, b: i32) -> i32
    requires true
    ensures result >= a and result >= b
{
    if a > b {
        return a
    }
    return b
}

test max {
    assert max(10, 20) == 20
}

func main() {
    print("The maximum is {max(10, 20)}")
}
```

Эталонные примеры — в [examples/](examples/): `hello_world`, `math`, `functions`,
`if_statement`, `iterator`, `struct` и витрина всех конструкций `all`.

## Запуск

```sh
make check           # компиляция всех примеров: парсинг, Power of 10,
                     # типы, исполнение test-блоков
make run_all         # запустить пример интерпретатором (run_math, ...)
make verify          # нативные бинарники + сверка с интерпретатором
make verify_suite    # регрессионный набор верификатора (tests/verify/)
make verify_selfhost # self-hosted компилятор против эталона на всех .eat
make verify_bootstrap# фикспойнт: компилятор собирает сам себя
make verify_selfhost_verify_all # self-hosted верификатор: весь
                     # репозиторий + самоприменение
```

Полный набор проверок — таблица make-целей в
[docs/PROCESSES.md](docs/PROCESSES.md) §5.

Требуются [uv](https://docs.astral.sh/uv/) и clang (для линковки бинарников).

Тот же полный гейт воспроизводим в контейнере на пиннутом
Linux-тулчейне (Podman, [docs/CONTAINERS.md](docs/CONTAINERS.md)):
`containers/run-gate.sh <дерево>` — разовый прогон; `make task-up
NAME=имя` → `task-merge` — изолированная задача с переносом в master
только после зелёного гейта.
