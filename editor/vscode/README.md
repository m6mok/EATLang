# EATLang для VS Code

Расширение даёт **подсветку синтаксиса** и **языковой сервер (LSP)** для
языка EAT (файлы `.eat`).

Подсветка (декларативная, работает всегда): ключевые слова, контракты
(`requires`/`ensures`/`assert`), модули (`import`/`export`/`from`/`as`,
`extern`), типы (`u8`..`u64`, `i32`/`i64`, `Result`/`Option`), встроенные
функции и аксиомы, битовые операторы, hex-литералы `0x…`, строки с
интерполяцией `{expr}`, char-литералы, комментарии `#`.

Языковой сервер написан **на самом EATLang** (`editor/lsp/`,
[docs/plans/LSP_PLAN.md](../../docs/plans/LSP_PLAN.md)) и общается с
редактором по JSON-RPC поверх stdio. Готово: рукопожатие
(`initialize`/`shutdown`/`exit`), **живые диагностики** — сервер линкует
фазы `lex+parse+check` настоящего компилятора и подчёркивает красным
лексические, синтаксические и типовые ошибки на лету
(`didOpen`/`didChange`/`didClose` → `publishDiagnostics`, полная
синхронизация текста), **переход к определению** (same-file и через
импортированные модули) и **verifier-inlays** — на каждой проверке
верификатора (`bounds`/`overflow`/`div`/`cast`/`shift`/`requires`/
`ensures`/`assert`) значок ✓ «доказано на компиляции» или ⚠ «останется
trap в рантайме» (`textDocument/inlayHint`; фаза `verify` слинкована в
сервер). Настройка `eatlang.optimize` включает ось `-O` (свёртка вызовов
перед верификацией, как `eatc ir -O`) — доказанных ✓ становится больше;
переключение обновляет значки на лету. Hover, автодополнение и CodeLens
бюджета памяти — следующие этапы.

## Установка

Подсветка работает без всякой сборки — симлинк в каталог расширений и
перезагрузка окна:

```sh
ln -s "$(pwd)/editor/vscode" ~/.vscode/extensions/eatlang.eatlang-0.1.0
```

Затем в VS Code: `Cmd+Shift+P` → «Developer: Reload Window».

## Включить языковой сервер

Серверу нужна одна npm-зависимость (`vscode-languageclient`); TS-сборки
нет — точка входа `extension.js` на чистом JavaScript.

```sh
cd editor/vscode
npm install
```

Перезагрузите окно. При открытии `.eat` расширение запускает сервер
командой из настроек `eatlang.server.command` / `eatlang.server.args`
(по умолчанию `bash editor/lsp/serve.sh`). Скрипт собирает **нативный
бинарник** сервера (`build/eat-lsp`) и запускает его: первый старт ждёт
~10 с сборки, дальше бинарник закеширован (пересборка — только при правке
источников сервера). Интерпретатор `eatc run` анализировал бы замыкание
модулей десятки секунд на правку — бинарник делает это за ~0,01 с.

Требования:

- воркспейс открыт на **корне репозитория EATLang** (сервер грузит
  `editor/lsp/` и `lib/` путями относительно корня);
- в `PATH` есть `uv`, окружение синхронизировано (`uv sync`).

Если `vscode-languageclient` не установлен, расширение это сообщит и
продолжит работать в режиме одной подсветки.

## Проверка сервера без редактора

Сервер тестируется транскриптом сессии (рукопожатие + диагностики) из
корня репозитория:

```sh
make verify_lsp
```

Ручная проверка подчёркиваний: открыть `.eat` с ошибкой (например
`let x: u32 = true`) — редактор красит её красным; исправить — подчёркивание
исчезает.

Ручная проверка inlay-хинтов: открыть `.eat` с недоказанной границей
(например `a[i]` при произвольном `i: u32`) — у проверки появляется ⚠
(tooltip объясняет вид и вердикт); при доказуемой (`a[i % 4]`, литеральный
индекс) — ✓. Включить `eatlang.optimize` в настройках — вердикты
пересчитываются с учётом свёртки вызовов (✓ может стать больше) без
правки текста. Если значков не видно — проверьте настройку VS Code
`editor.inlayHints.enabled`.

```sh
npm install
npx @vscode/vsce package
code --install-extension eatlang-0.1.0.vsix
```
